## 要約

GitHub Copilot の changelog に **2026-06-05 付で2件**掲載されました。

1つ目は、旧モデル **GPT-5.2 および GPT-5.2-Codex の退役（deprecated）** です。これらを明示的に指定していたワークフローは、後継モデルへの切り替えが必要になります。

2つ目は、**Visual Studio Code でエンタープライズが Copilot プラグインを直接管理できる機能の公開プレビュー**です。組織が許可するプラグインを VS Code 内で集中管理できるようになります。

## 何が変わったか

- **GPT-5.2 / GPT-5.2-Codex の退役**: 旧モデルが Copilot から退役。明示指定していたワークフローは後継モデルへ移行が必要。
- **エンタープライズ管理プラグイン（VS Code 公開プレビュー）**: 組織が Copilot プラグインを VS Code 内で直接管理可能に（プレビュー）。

## 業務インパクト（一般企業向け）

モデル退役は、Copilot をモデル指定込みで運用しているチームに移行作業を発生させます。CI・エージェント設定・社内ツールで GPT-5.2 / GPT-5.2-Codex を固定指定していないかを棚卸しし、後継モデルへ切り替える必要があります。指定を放置すると、退役後にエラーや想定外のフォールバックが起きる可能性があります。

エンタープライズ管理プラグイン（プレビュー）は、情シス・プラットフォームチームが「どのプラグインを社内で許可するか」を VS Code レベルで統制できる点で、ガバナンス強化につながります。プラグイン経由の情報持ち出しや未検証ツールの混入を抑えたい組織には有用です。

## 副業・個人活用視点

個人開発者は、GPT-5.2 / GPT-5.2-Codex を明示指定している設定やスクリプトがあれば、後継モデルへ更新しておくと安全です。普段モデルを指定せず使っている場合の影響は限定的です。エンタープライズ管理プラグインはプレビューかつ組織向けのため、個人利用では当面は直接の影響はありません。
