## 要約

GitHub MCP Serverが、2026-07-28リリース予定の**次期MCP仕様（ステートレス化）**に先行対応しました。主な変更点は、Redisセッションの廃止、リクエストペイロードのディープパケットインスペクション廃止（HTTPヘッダーからログ・シークレットスキャン用の値を取得する方式へ変更）、URLベースのユーザーログインをHTTPリクエスト単位で処理する「elicitation」の強化です。

新仕様のステートレスアーキテクチャにより、Redisセッションを介さずに初期化できるためレイテンシが低減し、スケーリングや並列クライアントハンドシェイクも容易になります。新旧クライアントの両方をGo SDKラッパー経由でサポートしており、既存のMCPクライアントとの後方互換性も維持されます。公式のconformanceテストでエージェント実装の準拠を検証できる点も示されています。

## 何が変わったか

- MCPの次期仕様（ステートレス化、2026-07-28リリース予定）にGitHub MCP Serverが先行対応
- Redisセッションを廃止し、初期化レイテンシを低減
- リクエストペイロードのディープパケットインスペクションを廃止、HTTPヘッダーからログ・シークレットスキャン用の値を取得する方式に変更
- URLベースのユーザーログインをHTTPリクエスト単位で処理するelicitationを強化
- Go SDKラッパー経由で新旧クライアントの両方をサポート（後方互換あり）

## 業務インパクト（一般企業向け）

自社でMCPサーバーを構築・運用している組織にとって、GitHubのような大規模事業者が次期仕様に先行対応した事例は、いつ・どのように自社実装を追従させるかの判断材料になります。ステートレス化はスケーラビリティに直結する変更であり、Redisのようなセッションストアの運用コストを削減できる可能性があります。MCPエコシステム全体が今後この方向に収束していく可能性を踏まえ、新規にMCPサーバーを設計する場合はステートレス前提のアーキテクチャを検討する価値があります。

## 副業・個人活用視点

個人でMCPサーバーを開発・公開している開発者にとって、GitHubの先行対応は仕様追従の具体的な実装参考になります。Redisセッションを使わずに済む設計は、小規模な個人プロジェクトの運用コストを下げる方向に働くため、自作MCPサーバーの改修を検討する際の判断材料として押さえておくとよいでしょう。
