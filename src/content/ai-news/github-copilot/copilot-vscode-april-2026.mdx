## 要約

GitHub Copilot in Visual Studio Codeの2026年4月リリースまとめが公開されました。

今回の更新は、文脈検索、エージェント体験、セッション継続性が中心です。Copilotがコードベースを理解し、ブラウザやterminalと連携し、長めの作業を継続できる方向に進んでいます。

## 何が変わったか

- すべてのworkspaceでsemantic indexingを利用可能に
- Copilot agentがGitHub repo / orgをgrep-style検索できる `githubTextSearch` を追加
- ローカルchat履歴を検索する実験的 `/chronicle`
- prompt caching、deferred tool loading、agentic toolsでtoken使用量を削減
- chat内でinline diffを確認可能に
- 自然言語からcustom agents、skills、instructionsを生成可能に
- open terminalへのread / writeアクセス
- browser tab sharingでブラウザ状態をagentに共有
- Copilot CLI sessionをGitHub.comやmobileから監視・操作可能に
- Copilot Business / EnterpriseでもBYOKを拡張

## 業務インパクト

Copilotは、コード補完ツールというより「開発作業を進めるエージェント」に近づいています。

特に重要なのは、ブラウザ、terminal、diff、履歴、GitHub横断検索がつながってきたことです。実務の開発は、コードを書く以外に、ログを見る、画面で確認する、過去の作業を思い出す、PRやissueを追う、といった周辺作業が多い。今回の更新は、その周辺作業をCopilot agentが扱いやすくするものです。

## 教材化視点

教材としては、「Copilotを補完からエージェント運用へ移す」テーマに向いています。

演習にするなら、次の流れがよさそうです。

- repoをsemantic searchで調査する
- GitHub横断検索で関連実装を探す
- agentに修正させる
- inline diffでレビューする
- browser tabを共有してUI確認する
- terminal出力を読ませてテスト失敗を直す
- セッションを継続して次タスクへ進める

開発者向けAI教材では、かなり実務に近い題材になります。
