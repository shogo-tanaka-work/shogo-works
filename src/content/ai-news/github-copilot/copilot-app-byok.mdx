## アップデート概要

GitHub Copilot アプリが BYOK（Bring Your Own Key）に対応した。組織が自社で契約している OpenAI・Azure・Anthropic などの API キーを Settings から登録し、Copilot エージェントセッションをそのプロバイダーで実行できるようになる。Copilot Business および Enterprise プランが対象。

## 主な変更点

### 自社 API キーを Copilot に接続

「Settings → Model Providers」からエンドポイントと API キーを登録するだけで設定が完了する。

対応プロバイダーは OpenAI・Azure OpenAI・Microsoft Foundry・Anthropic に加え、LM Studio と Ollama（ローカル LLM）、そして任意の OpenAI 互換エンドポイントまで対応している。LM Studio と Ollama はホスト名だけで設定でき、API キーの登録は不要だ。

### API キーのローカルキーチェーン保存

登録した API キーはローカル OS のキーチェーンに保存され、Copilot の UI から読み返されることはない。GitHub サーバーにキーが送信されないため、セキュリティポリシーの観点から BYOK に懸念を持つ組織でも採用しやすい設計だ。

### 内部ゲートウェイ経由のルーティング

データ境界要件が厳しい組織向けに、自社クラウドアカウント・テナント・内部ゲートウェイを経由して Copilot のリクエストをルーティングする設定が可能。金融・医療・公共機関のように「データが自社管理インフラ外に出てはならない」という要件を持つ組織でも Copilot を使えるようになる。

## 業務インパクト（一般企業向け）

Copilot の導入を検討してきたが「自社のデータがどこに行くかわからない」という理由で止まっていた企業にとって、BYOK は突破口になりうる。

特に実務的なのは「すでに Anthropic と契約しており、Claude を使っている組織が Copilot のエージェント機能をその API キーで動かす」というケースだ。AI 調達をプロバイダーごとに最適化しながら、フロンティアモデルとローカルモデルを混在させるコスト管理も可能になる。

## 副業・個人活用視点

個人での活用は Copilot Business / Enterprise 契約が前提になるため、フリーランス・個人開発者が即座に使えるわけではない。ただし「自社モデルを Copilot に繋ぐ」という設計思想は、組織の AI インフラ提案や見積もりを行う際の一つの選択肢として理解しておく価値がある。
