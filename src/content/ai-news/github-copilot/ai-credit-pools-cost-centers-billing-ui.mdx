## 要約

GitHub Copilotの管理機能として、**コストセンター単位のAIクレジットプール管理が課金UIから直接設定できる**ようになりました。これまではREST API経由でしか操作できなかった機能で、コストセンターの作成・編集画面上でAIクレジットプールのON/OFFを切り替えられます。プールの上限は、そのコストセンターに割り当てられたライセンス数から自動計算され、ライセンスの増減に応じて自動的に再調整されます。

上限に到達した際の挙動も選択可能で、それ以上の利用をブロックするか、超過分を追加課金として許可するかを設定できます。既存の「コストセンター予算」機能とは独立して動作するため、同一コストセンターに対してAIクレジットプールと予算の両方を併用設定することも可能です。

## 何が変わったか

- 課金UIからコストセンターごとのAIクレジットプールをON/OFF・上限設定可能に（従来はREST API限定）
- ライセンス数の増減に応じてプール上限を自動再計算
- 上限到達時の挙動（利用ブロック／超過分の追加課金許可）を選択可能
- 既存のコストセンター予算機能と独立して併用設定できる
- 対象はCopilot Business・Copilot Enterprise（GitHub Enterprise Cloudのみ）

## 業務インパクト（一般企業向け）

部門・チーム単位でCopilotの利用コストを分配・管理したい組織にとって、これまでAPI実装が必要だった設定がUI完結するのは運用負荷の低下に直結します。情シス・コーポレートエンジニアリング部門がAPIスクリプトを自前で用意しなくても、課金画面から直接コストセンターごとの上限とオーバーエイジ方針を設定できるようになるため、Copilot導入企業でコスト管理体制を整備する際のハードルが下がります。

特に「上限到達時にブロックするか、追加課金を許容するか」を明示的に選べる点は、部門予算の厳格な運用と、業務停止を避けたい部門とでポリシーを使い分けたい企業に有用です。ライセンス数変動に連動した自動再計算により、異動や増員時の再設定漏れも防げます。

## 副業・個人活用視点

個人利用や小規模チームでは直接関係しにくい機能ですが、複数クライアントの開発を並行して請け負うフリーランス・受託開発者がGitHub Enterprise Cloud環境で作業する場合、クライアントごとのコストセンター設定やAI利用上限の考え方を理解しておくと、契約先の管理者とのコスト面のやり取りがスムーズになります。
