## 要約

Claude Code（Team / Enterprise プラン）でアーティファクト機能が使えるようになりました。セッション中の作業進捗を、インタラクティブなウェブページとしてリアルタイムで生成・共有できる機能です。

具体的な用途として公式が挙げているのは、インシデント調査の進捗可視化、ダッシュボード生成、リリースチェックリストなどです。Claude Code が持っているコードベース・コネクター・会話コンテキストをもとに自動生成します。

URL は更新後も変わらず、バージョン履歴と復元機能も備えます。組織レベルのアクセス制御と保持期間設定も可能です。

## 何が変わったか

- セッションの進捗をライブのウェブページ（アーティファクト）として生成
- コードベース・コネクター・会話内容のコンテキストをもとに自動生成
- 組織内認証ユーザーへのリアルタイム共有が可能
- URL は更新時も不変（固定リンクとして送付・ブックマーク可能）
- バージョン履歴管理と以前バージョンへの復元
- 管理者が保持期間とアクセス制御を組織レベルで設定可能

## 業務インパクト（一般企業向け）

「今 Claude Code が何をやっているか」をチームに伝える方法がアーティファクトで増えました。これまでは Claude Code のセッション状況を共有するには画面共有やスクリーンショットに頼るしかなかったのが、URL を渡すだけでライブページを見てもらえます。

インシデント対応のような「今まさに調査中」という状況で、上司や別チームのメンバーに進捗を見せながら作業を続けられます。管理者には保持期間設定（データが残る期間）とアクセス制御（誰が見られるか）の設定権限があり、コンプライアンス要件に合わせた運用ができます。

## 副業・個人活用視点

フリーランス案件でのプロトタイプ提案や進捗報告に使えます。「Claude Code でどこまでできたか」を都度スクリーンショットで送る代わりに、URL 1本を送って「今ここまでです」と伝えられます。

ただし Team / Enterprise プランが前提なのでコスト計算は必要です。個人の Free / Pro プランでは現時点では利用できません。
