## 何が変わったか

`/mcp`で接続サーバーのツール数や0ツール状態を確認しやすくなり、`.zip`形式のプラグインアーカイブも`--plugin-dir`で扱えるようになりました。

worktree作成、MCPツール結果、巨大stdin、長いURL、並列shell tool callなど、実務で詰まりやすい失敗ケースにも修正が入っています。

## 教材化メモ

Claude Codeの実務教材では、MCPサーバー診断、チーム配布用プラグイン、worktree運用、並列ツール実行時の安定性を扱う回に接続しやすい更新です。
