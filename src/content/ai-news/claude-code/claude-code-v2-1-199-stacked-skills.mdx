## 要約

Claude Code v2.1.199（2026-07-02T23:35Z）は、**スタック型スキル呼び出し**という新しい使い方と、バックグラウンドエージェント運用の信頼性改善をあわせて届けるリリースです。

これまでスキルは `/skill-name` のように1つずつ呼び出すのが基本でしたが、v2.1.199からは `/skill-a /skill-b do XYZ` のように複数のスキル名をプロンプト冒頭にスタックして書けるようになりました。先頭から最大5個までのスキルがまとめて全ロードされ、それぞれの手順やコンテキストを踏まえた上で1回のリクエストとして実行されます。

残りは主に信頼性まわりの改善です。サブエージェントがレート制限や実行時エラーで打ち切られても、それまでの部分結果を親エージェントへ返すようになり、ストリーミング中にエラーが起きても取得済みのpartial応答を保持します。SSL証明書エラー発生時は原因と対処法を即座に画面へ表示し、Linux上でバックグラウンドデーモンが自滅ループに陥る不具合も修正されました。

## 何が変わったか

- **スタック型スキル呼び出し**: `/skill-a /skill-b <指示>` の形式で、先頭最大5個のスキルを1コマンドでまとめて起動可能
- **サブエージェントの部分結果保持**: レート制限・エラーで打ち切られても、それまでの成果を親エージェントに返す
- **ストリーミング中エラー時のpartial保持**: 応答が完了する前にエラーが起きても、取得済み部分は失われない
- **SSL証明書エラーの即時ガイド**: 発生時に原因・対処法をその場で表示
- **Linuxバックグラウンドデーモンの自滅ループ修正**: 特定条件で再起動を繰り返す不具合を解消

## 業務インパクト（一般企業向け）

複数のスキルを日常的に組み合わせて使っているチームにとって、スタック型呼び出しは「毎回スキル名を1つずつ指定して結果を待つ」という手間を減らします。例えば「調査スキル」と「記事化スキル」を1コマンドでつなげるような社内ワークフローがある場合、定型プロンプトをスタック形式に書き換えることで運用が簡潔になります。ただし先頭最大5個までという制限があるため、6個以上のスキルを連鎖させたい場合は分割呼び出しが必要です。

バックグラウンドエージェントの信頼性改善は、CI的にエージェントへ長時間タスクを任せている組織ほど恩恵が大きく、レート制限に達しても結果がゼロにならない点は運用上の安心材料になります。

## 副業・個人活用視点

自分用のスキルを複数作っている人は、よく組み合わせて使う手順をスタック形式のプロンプトとしてメモしておくと、毎回スキル名を打ち込む手間が減ります。副業で複数クライアントのタスクを並行して回す際も、部分結果が保持されるようになったことで「エラーで全部やり直し」のリスクが下がり、バックグラウンド実行を安心して活用しやすくなりました。
