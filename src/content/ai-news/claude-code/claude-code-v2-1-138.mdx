## 要約

Claude Code v2.1.138 が 2026-05-09 15:33 JST に公開されました。公式リリースノートで明示されている変更は `Internal fixes` のみです。

直前の v2.1.136 / v2.1.137 では、MCP 設定の消失、OAuth リフレッシュ競合、Windows の VSCode 拡張起動失敗など、実務運用に響く修正がまとまっていました。今回の v2.1.138 は、その流れを受けた安定化パッチとして捉えるのがよさそうです。

## 何が変わったか

- Claude Code v2.1.138 が正式リリースされた
- 公式に公開されている変更内容は内部修正のみ
- 新機能、UI変更、管理者向け設定変更は明示されていない

公開情報だけを見ると、利用者が設定や運用手順を変えるタイプの更新ではありません。一方で、Claude Code は開発フローの中心に入りやすいツールなので、内部修正だけのパッチでも、直前バージョンで挙動が不安定だった場合は確認する価値があります。

## 業務インパクト

今回の更新単体で「すぐ設定を変える」必要はありません。ただし、v2.1.136 / v2.1.137 に上げる予定があるなら、v2.1.138 まで含めて更新するのが自然です。

特に、Claude Code を次のように使っているチームは、バージョン固定の見直し対象になります。

- MCP サーバーを複数つないでいる
- VSCode / JetBrains 拡張と併用している
- Windows / WSL2 を含む混在環境で使っている
- 社内向けに Claude Code の導入手順を固定している

内部修正の詳細が出ていないため、更新後は「困っていた現象が消えたか」を実地で見るのが大事です。リリースノートだけで改善範囲を断定しない方が安全です。

## 副業・個人活用視点

個人利用では、今回のニュースだけで急いで検証記事を書くほどの材料は少なめです。むしろ、5/9に出た v2.1.136 / 137 の大きな修正とセットで、「Claude Code は細かいパッチが速く出るので、トラブル時はまず最新版との差分を見る」という運用知識として覚えておくと役に立ちます。

クライアントワークで Claude Code を使っている場合は、作業前にバージョンを揃えておくと、MCP や拡張機能まわりの再現性が上がります。
