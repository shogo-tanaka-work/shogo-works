## 要約

Claude Code v2.1.198 のリリース（2026-07-01T20:45Z）で、2つの大きな変化が起きました。

1つ目は **Claude in Chrome の GA（一般提供開始）**。以前はベータ段階だったブラウザ自動化ツールが正式リリースされ、Claude がブラウザを直接操作してWebタスクを処理できるようになりました。UIのスクリーンショット取得、フォーム入力、ページナビゲーションなどをCLIのコードと同じセッション内で扱えます。

2つ目は **バックグラウンドエージェントの自動化**。`claude agents` で起動したエージェントが、コード作業を終えると自動でコミット・プッシュ・ドラフトPR作成まで実行するようになりました。これにより「エージェントに指示して後から確認する」という非同期開発スタイルが一段と現実的になります。

このリリースにはほかにも、エージェント状態変化を外部に通知する Hook（`agent_needs_input` / `agent_completed`）、ネットワーク一時エラーの自動リトライ、AWS STS トークンの自動リフレッシュ、Explore エージェントがセッションモデルを継承するようになった改善なども含まれます。

## 何が変わったか

- **Claude in Chrome GA**: ブラウザ自動化ツールが正式一般提供
- **バックグラウンドエージェント自動化**: 作業完了時に自動コミット・プッシュ・ドラフトPR作成
- **エージェント通知Hook**: `agent_needs_input` / `agent_completed` の2イベントをHookで受信可能
- **`/dataviz` スキル追加**: チャート・ダッシュボード設計ガイダンスとカラーパレット検証
- **ネットワーク安定性向上**: ECONNRESET 等の一時エラーをバックオフ付き自動リトライに変更
- **AWS STS トークン自動リフレッシュ**: セッション期限切れによるdead-endを防止
- **Explore エージェント改善**: セッションのモデル設定を継承（Opus上限）、従来はHaikuがデフォルトだった
- **Extended thinking**: subagent と context compaction にカスケード

## 業務インパクト（一般企業向け）

バックグラウンドエージェントの自動コミット/PR作成機能は、開発チームのレビューフローに影響します。エージェントが自動で作ったドラフトPRをレビューするワークフローを設計しておくと、AI活用の恩恵を最大化できます。

Claude in Chrome の GA により、社内ツールの操作記録・自動テスト・ブラウザベースのRPA代替としての活用が広がります。ただしアクセス権限の管理（どのサイトをClaude Codeに操作させるか）を社内ルールとして整備することも検討してください。

エージェント通知Hook（`agent_needs_input`）は、エージェントが人間の確認を必要とした際に Slack や社内システムへ通知を飛ばす仕組みを作る基盤として活用できます。

## 副業・個人活用視点

バックグラウンドエージェントが「コードを書いて → コミット → PRまで作る」一連を自動化してくれるため、副業案件でのフィーチャー開発を非同期で進めやすくなります。本業の隙間時間に「タスクを投げて、後から確認する」スタイルで作業量を増やすことができます。

Claude in Chrome の活用例としては、ポートフォリオサイトのビジュアル確認、競合サービスの定点観測、フォームを使ったデータ収集の自動化などが考えられます。
