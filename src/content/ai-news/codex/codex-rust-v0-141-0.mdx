## 要約

OpenAI Codex CLI の安定版 **rust-v0.141.0** がリリースされました。今回の主な変更はリモート実行環境のセキュリティと信頼性の強化です。

リモート実行環境に「認証付き Noise リレーチャネル」が採用されました。Noise プロトコルは Signal や WireGuard でも使われている暗号化プロトコルで、通信の安全性が高まります。同時に、クロスプラットフォームでネイティブのワーキングディレクトリとシェル環境を維持したままリモート実行できるようになっています。

あわせてエグゼキュータープラグインがスレッドごとに stdio MCP サーバーを起動できるようになり、並行実行の柔軟性も向上しています。

## 何が変わったか

- リモート実行環境が認証付き Noise リレーチャネルを採用（暗号化通信でセキュリティ向上）
- クロスプラットフォームでネイティブのワーキングディレクトリとシェルを保持
- エグゼキュータープラグインがスレッドごとに stdio MCP サーバーを起動可能
- Windows サンドボックス実行の認証情報修正・PowerShell コマンド処理時間延長
- アイドル状態の exec-server リレーが接続を維持するよう修正

**alpha 動向（参考）:** 期間内に 0.141.0-alpha.7、0.142.0-alpha.1〜3 計4件のアルファ版も公開。

## 業務インパクト（一般企業向け）

Noise プロトコルによる暗号化リレーは、リモート実行環境の通信セキュリティを体系的に強化するものです。セキュリティポリシーが厳しい企業でも「通信が暗号化されているか」という観点でクリアしやすくなります。

エグゼキュータープラグインの stdio MCP 並行起動対応は、複数ツールを同時に使う複雑なエージェント構成でのパフォーマンス改善に繋がります。MCP を活用した社内ツール連携構成を組んでいる場合は注目です。

Windows サンドボックスの修正は、Windows 環境で Codex を試験的に導入している組織に直接影響します。

## 副業・個人活用視点

Codex をリモート実行で使っているユーザーは、今バージョンへのアップデートでセキュリティと安定性が改善されます。特に「リモートで実行させているが時々切れる」という経験があった場合、アイドル時の接続維持修正が効いてくる可能性があります。

Noise プロトコルの採用は、技術ブログや副業の提案書で「Codex の通信セキュリティ」を説明する際の具体的な根拠になります。
