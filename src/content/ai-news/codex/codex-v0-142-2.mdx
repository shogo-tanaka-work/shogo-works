## アップデート概要

OpenAI Codex v0.142.2 が 2026-06-25 に stable リリースされた。MCP ツールのデフォルト動作の変更、macOS プロキシサポート、プラグインのダークモード対応、safety UI 改善が主な変更点。

## 主な変更点

### MCP ツールがデフォルトでツール検索を使用

MCP（Model Context Protocol）ツールが、デフォルトでツール検索（tool search）を使用するようになった。

これまでは MCP に登録された全ツールのリストを参照してツールを選ぶ方式だったが、tool search を使うことで意味的に関連するツールを検索して選択できるようになる。MCP ツールの登録数が多い環境でも、より正確なツール選択が期待できる。

### macOS のシステムプロキシ自動対応

macOS の認証クライアントがシステムプロキシ設定（PAC ファイル、WPAD）を適切に処理するようになった。

企業の IT 環境ではインターネット通信をプロキシサーバー経由で行うケースが多い。macOS での proxy auto-configuration（PAC）や Web Proxy Auto-Discovery（WPAD）の設定を Codex が自動で尊重するようになったことで、企業 Mac ユーザーが追加設定なしで Codex を利用できる場面が増える。

### プラグインのダークモードロゴ対応

プラグインのロゴがダークモードに対応した。ダークモードで使用している場合のビジュアルが改善される。

### safety-buffering UI の改善

安全性バッファリングの UI が改善された。危険な操作の前に確認を挟む仕組みのユーザー体験が向上する。

### Bedrock 認証情報エラーの改善

AWS Bedrock を使っている環境での認証情報エラーメッセージが改善された。Bedrock 経由で Codex を利用している場合、問題発生時の原因特定がしやすくなる。

## 業務インパクト（一般企業向け）

MCP ツールのデフォルト変更は、社内ツールを MCP で接続している環境に影響する。ツール選択がより賢くなることで、「意図したツールが選ばれない」問題が減る。ただし動作が変わることで既存のワークフローに影響が出る場合はリリースノートを確認のうえ動作テストを推奨する。

企業 Mac での利用は、プロキシ設定の問題で使えなかったケースが解消される可能性がある。

## 副業・個人活用視点

MCP ツールを多数登録している個人開発者にとって、ツール検索のデフォルト化は実際の補完精度に影響する。大量のツールを接続していて「関係ないツールが選ばれる」という状況があれば、このアップデートで改善が期待できる。
