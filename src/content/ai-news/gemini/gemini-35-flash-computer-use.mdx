## アップデート概要

Google が 2026-06-24、Gemini 3.5 Flash への Computer Use 機能統合を発表した。ブラウザ・モバイル・デスクトップの 3 環境を「見て・推論して・操作する」エージェント機能が Gemini 3.5 Flash 単一モデルに組み込まれた形で提供される。Gemini API および Gemini Enterprise Agent Platform から利用でき、リファレンス実装コードは GitHub で公開されている。

## 主な変更点

### Gemini 3.5 Flash に Computer Use が統合

これまで Computer Use は独立したモデルや追加設定が必要だったが、今回の変更で Gemini 3.5 Flash のビルトインツールとして提供される。1 つのモデルで推論からブラウザ操作・デスクトップ操作まで完結する。

対応環境はブラウザ・モバイル・デスクトップの 3 種類。ソフトウェアテストの自動化、業務アプリケーションのワークフロー自動実行を主なユースケースとして想定している。

### セキュリティ対策が標準搭載

AI がコンピューターを操作するという性質上、セキュリティへの配慮が重要になる。Gemini 3.5 Flash の Computer Use には以下の対策が組み込まれている。

- プロンプトインジェクション対策の敵対的トレーニング済み
- センシティブな操作への人間承認確認オプション
- 間接的プロンプトインジェクション検出オプション

Google はこれらに加えて、サンドボックス環境・人間によるベリフィケーション・厳格なアクセス制御との組み合わせを推奨している。

### デモ環境とリファレンス実装

Browserbase がデモ環境を提供しており、すぐに試せる環境が用意されている。リファレンス実装コードは GitHub で公開されているため、自社システムへの組み込みも容易に始められる。

## 業務インパクト（一般企業向け）

「エージェントに業務操作をさせたい」というニーズに直接答える機能だ。特にソフトウェアの QA テスト自動化や、ブラウザ経由の定型入力作業の自動化に使えるケースが増える。Gemini API を既に使っている場合は、同じモデルで PC 操作まで拡張できるため、新たなモデル選定や費用の検討が不要になる。

導入時はサンドボックス環境での先行テストを強く推奨する。AI エージェントがシステムを操作する際のリスク（誤操作・意図しないデータ変更）は人間がレビューする仕組みと組み合わせて使うのが現状のベストプラクティスだ。

## 副業・個人活用視点

個人開発でよくある「Web サイトのスクレイピング・フォーム自動入力・繰り返しクリック操作」をブラウザ自動化ツール（Selenium・Playwright）の代わりに AI エージェントで処理するユースケースに使えるようになる。Gemini API のアクセスがあれば試せるため、まず Browserbase のデモから動作を確認してみると取り組みやすい。
