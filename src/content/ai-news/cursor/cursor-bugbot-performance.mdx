## 要約

**Cursor の Bugbot が大幅に改善**されました。Composer 2.5 のモデル訓練により、レビュー速度が**約5分から90秒に短縮（約3倍）**、バグ検出精度が**0.56 → 0.62 bugs/review（10%向上）**、実行コストは**22%削減**されています。

新機能として **`/review` コマンド**が追加され、コードを push する前にローカルでレビューを実行できるようになりました。GitHub・GitLab との統合では、**最後のレビュー以降の変更だけを対象にする差分レビュー**も可能です。

## 何が変わったか

- **レビュー速度**: 約5分 → 約90秒（約3倍）。
- **バグ検出精度**: 0.56 → 0.62 bugs/review（10%向上）。
- **コスト**: 実行あたり約22%削減。
- **`/review` コマンド追加**: push 前にローカルでレビューを実行できる。
- **GitHub・GitLab 統合強化**: 最後のレビュー以降の変更のみを対象に自動スキップ（重複防止）。
- **有効化**: Cursor 3.7 以上・cursor.com/agents から利用可能。自動有効化。CLI 対応は近日予定。

## 業務インパクト（一般企業向け）

PR レビューフローに Bugbot を組み込んでいるチームでは、**レビュー結果が届くまでの待ち時間が5分から90秒に短縮**されることで、開発者の文脈切り替えコストが下がります。バグが見つかっても次のタスクに移る前に対処できる時間内に結果が返ってくる、という感覚です。

`/review` コマンドにより、**PR 作成前のローカルチェックが手軽になりました**。CI でしか確認できなかったバグを、push 前の段階で開発者自身が把握できます。GitHub・GitLab の連携で差分レビューが機能するため、大規模リポジトリでも毎回全体を再分析するコストをかけずに運用できます。

## 副業・個人活用視点

個人・フリーランスでは、レビューコストの削減幅が体感に直結します。以前は「Bugbot を動かして結果が返るまで別作業」という待ちが必要でしたが、90秒なら当該コードを見ながら結果を受け取れます。`/review` をコミット前の習慣にすることで、クライアントへのコード品質担保にも使えます。
