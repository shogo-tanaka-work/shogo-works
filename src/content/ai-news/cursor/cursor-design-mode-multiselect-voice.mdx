## 要約

**Cursor 3.7 で導入されたブラウザ内 Design Mode に、2つの改善**が追加されました。

1つ目は**複数要素のマルチセレクト**です。2つ以上の要素を選ぶと、Cursor が各要素のコード・周囲のレイアウト・ページ上の視覚的関係をまとめて認識します。これにより、要素を揃える・重複コンテンツを削除する・複数コンポーネントを一括で調整する、といった指示をエージェントへまとめて出せます。

2つ目は**音声入力**です。Design Mode のオーバーレイから変更を声で指示でき、エージェント実行中でもマイクで次の変更をキューイングできます。前処理の完了を待たずに、次の指示を積んでいけます。

## 何が変わったか

- **マルチセレクト**: 2つ以上の要素を同時選択。Cursor が選択要素・そのコード・周囲レイアウト・視覚的関係を横断認識し、一致・重複削除・複数コンポーネントの一括調整を依頼可能。
- **音声入力**: Design Mode オーバーレイから音声で変更指示。エージェント実行中もマイクが使え、前処理の完了を待たずに次の指示をキューイング。
- 対象バージョン: **Cursor 3.7 以上**。

## 業務インパクト（一般企業向け）

フロントエンドの UI 調整は「1要素ずつ直す」と手数がかさみがちですが、マルチセレクトで複数要素を文脈付きでまとめて扱えると、レイアウトの整合性合わせやデザインの一括修正を短い指示で回せます。デザイナーとエンジニアの間で「この一帯をまとめて揃えたい」という依頼を、コードと視覚関係の両方を踏まえてエージェントに渡せる点が実務的です。音声入力は、実行中に次の修正を積めるため、レビューしながら指示を出す反復作業のテンポを上げます。

## 副業・個人活用視点

個人開発や受託の UI 制作では、Design Mode のマルチセレクトが「見た目を見ながらまとめて直す」作業に効きます。クライアントとの画面確認中に、その場で複数箇所をまとめて調整指示できると、修正のリードタイムを短縮できます。音声入力は、手を動かしながら次の指示を声で積めるため、画面共有しながらのライブ修正やプロトタイピングと相性が良い機能です。
