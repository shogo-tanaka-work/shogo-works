## 要約

xAIは2026年5月14日、Grok Build Early Betaを発表しました。Grok Buildはterminalで動くcoding agent / CLIで、professional software engineeringとcomplex coding work向けに設計されています。現時点ではSuperGrok Heavy subscribers向けのearly betaです。

発表内容を見ると、Grok Buildは単なるコード生成CLIではありません。plan mode、diff review、AGENTS.md、plugins、hooks、skills、MCP servers、parallel subagents、worktree integration、headless mode、ACP supportを前面に出しています。これはCodexやClaude Codeが進めてきたagentic coding環境に、xAIがかなり近い思想で参入したことを意味します。

## 何が変わったか

- Grok Build Early BetaがSuperGrok Heavy subscribers向けに公開
- terminalからinstallしてcoding agentとして利用
- plan modeで実行前に計画を承認・修正できる
- 変更はdiffとして確認できる
- AGENTS.md、plugins、hooks、skills、MCP serversに対応
- parallel subagentsとworktree integrationをサポート
- headless modeでscriptやautomationに組み込み可能
- ACP supportでagent orchestration appとの連携を想定

## 業務インパクト（一般企業向け）

Grok Buildの登場で、AI coding agent市場はさらに競争が激しくなります。注目すべきは、xAIが最初からAGENTS.md、plugins、hooks、skills、MCP、subagentsを掲げている点です。これは、開発チームがagentに期待する基本機能がかなり固まってきたことを示しています。

企業が比較する軸も変わります。モデル性能だけでなく、既存リポジトリ規約の読み込み、承認前提のplan mode、diff review、MCP連携、headless実行、subagent分割、worktree運用が重要になります。Codex、Claude Code、GitHub Copilot app、Grok Buildを横並びに見ると、「どのagentが自社の開発統制に合うか」が選定ポイントになります。

## 副業・個人活用視点

個人開発者にとっては、Grok Buildは新しい選択肢です。現時点ではSuperGrok Heavy subscribers向けなので誰でもすぐ使えるわけではありませんが、plan modeやdiff reviewが最初からあるのは安心材料です。

副業エンジニアは、今後複数のcoding agentを案件ごとに使い分ける場面が増えます。Grok BuildがAGENTS.mdやMCPに対応しているなら、既存のrepo規約やツール設定を使い回せる可能性があります。Codex / Claude Code向けに整えたAGENTS.mdやhooks設計が、Grok Buildでも資産になるかもしれません。
