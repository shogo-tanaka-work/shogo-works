## 要約

**Claude Managed Agents に、スケジュール実行（cron）と Vault による環境変数管理の2機能がパブリックベータで追加**されました。

スケジュール実行により、Managed Agents を**決めた時刻に自動で動かせます**。夜間のデータ同期、週次のコンプライアンス確認、定期レポートの生成といった繰り返しタスクを、人手のトリガーなしで回せるようになります。

Vault による環境変数管理では、**Vault に格納した環境変数はサンドボックス内には保持されず、ネットワーク境界でのみ添付**されます。API キーなどの秘密情報をエージェントの実行環境に常駐させない設計です。導入事例として Rakuten・Notion・Browserbase が紹介されています。

## 何が変わったか

- **スケジュール実行（cron）**: Managed Agents を定期実行可能に。夜間データ同期・週次レポート・定期コンプライアンス確認などを自動化。
- **Vault による環境変数管理**: Vault 格納の環境変数はサンドボックス内に保持されず、ネットワーク境界でのみ添付。秘密情報を実行環境に常駐させない。
- いずれも**パブリックベータ**。

## 業務インパクト（一般企業向け）

これまでエージェントは「リクエストされたら動く」使い方が中心でしたが、スケジュール実行によって**定期業務そのものをエージェントに委ねられる**ようになります。夜間バッチ・週次レポート・定期監査のような、人がやると属人化・抜け漏れが起きやすい作業を自動化できます。

Vault 連携は、エージェント自動化を本番で使う際の大きな懸念だった「秘密情報の扱い」に答えるものです。API キーや認証情報をサンドボックスに残さず、ネットワーク境界でのみ注入する設計のため、漏洩面を抑えつつ自動化を広げられます。情シス・セキュリティ部門の承認を得やすくなる点でも実務的です。パブリックベータ段階のため、対象業務を絞って検証してから広げるのが安全です。

## 副業・個人活用視点

個人・小規模チームでは、スケジュール実行が「定期作業の自動化」に直接効きます。毎朝のデータ収集、週次の進捗レポート、定期的な情報チェックなどをエージェントに任せ、自分は確認だけにできます。Vault による秘密情報管理は、クライアント案件で API キーを扱う際の安心材料になります。Rakuten・Notion・Browserbase のような事例を参考に、まずは自分の定型業務を1つ自動化してみると、効果と運用上の注意が掴めます。
