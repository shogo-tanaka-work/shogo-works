## 要約

**Anthropic が Claude Opus 4.1（`claude-opus-4-1-20250805`）の非推奨を発表**しました。Claude API 上での退役は **2026年8月5日**に予定されています。移行先としては **Claude Opus 4.8（`claude-opus-4-8`）** への切り替えが推奨されており、移行ガイドも提供されています。

影響を受けるのは、**API 経由でモデル ID を固定指定している開発者・企業**です。Claude アプリ（UI）を使うエンドユーザーへの直接的な影響はありません。

## 何が変わったか

- `claude-opus-4-1-20250805` が**非推奨**に。
- **Claude API での退役日: 2026年8月5日**。
- 推奨移行先: **Claude Opus 4.8（`claude-opus-4-8`）**。移行ガイドあり。
- 対象: API でモデル ID を固定指定している実装。Claude アプリ（UI）のエンドユーザーへの直接影響なし。

## 業務インパクト（一般企業向け）

自社プロダクトや社内ツールで Claude API を呼んでいる場合、`claude-opus-4-1-20250805` をモデル ID としてハードコードしている箇所を洗い出す必要があります。退役日（2026-08-05）を過ぎると当該 ID へのリクエストが通らなくなるため、それまでに Opus 4.8 へ切り替えておく計画が必要です。

切り替えはモデル ID を差し替えるだけでなく、コスト・レイテンシ・出力傾向が変わる可能性があるため、本番投入前に主要ユースケースで出力差分を再評価しておくのが安全です。モデルの非推奨・退役は定期的に発生するため、「使用中モデル ID の一覧」と「退役スケジュール」を管理対象に含めておくと、こうした移行を慌てずに回せます。

## 副業・個人活用視点

個人開発や受託案件で Claude API を組み込んでいる場合、Opus 4.1 を指定したままの実装が残っていないか確認しておきましょう。納品済みのクライアント案件で Opus 4.1 を固定指定していると、退役後に動かなくなり保守対応が発生します。退役前に Opus 4.8 へ更新し、出力品質・コストの差分を確認しておくと、移行後のトラブルとクレームを避けられます。モデル ID をハードコードせず設定で切り替えられるようにしておくと、今後の非推奨にも強い構成になります。
