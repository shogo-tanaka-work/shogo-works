## この章の要点

直近の Launch Week（2023 年末〜2025 年）を振り返ると、Supabase の投資先は **AI 基盤・開発者体験・プラットフォーム化** の 3 軸に集中している。pgvector まわりの拡張（Automatic Embeddings、Vector Buckets）と MCP Server による AI エージェント連携は 2024〜2025 年を通じて継続的に強化された。開発者体験では Branching・Declarative Schemas・Postgres Language Server など「ローカル開発〜本番反映」のワークフロー全体が整備されつつある。プラットフォーム化の観点では Read Replicas・Dedicated Poolers・High Performance Disks・Compute 拡張など本番スケールへの対応が着実に追加されている。Auth では Third-party Auth（Clerk / Auth0 / Cognito / Firebase）の統合が進み、既存 Auth 基盤との共存パスが広がった。

## Launch Week とは

Supabase は年に複数回（概ね年 2〜3 回）**Launch Week** と呼ぶ発表イベントを開催する。1 週間（月〜金）の毎日午前 8 時 PT（太平洋時間）に新機能・新サービスを 1〜複数件ずつ公開し、公式ブログに詳細記事を掲載する形式をとる。期間中は並行して **Hackathon** も開催され、コミュニティのプロジェクトが集まる。全発表は週末に「Top 10 Launches」としてまとめ記事が公開される。

イベント番号は通し番号（LWX、LW12〜）で管理されているが、2024 年 4 月には **GA Week**（General Availability 宣言）として特別版が開催された。2025 年以降は Launch Week 14（2025 年 3〜4 月）・Launch Week 15（2025 年 7 月）・December Launch（2025 年 12 月）が確認されている。

## 採用判断への影響

**コンプライアンス対応の進展**

- **SOC 2 Type 2** は 2023 年 5 月 22 日付で取得済み（発表：2023-08-11）。年次監査を継続している。
- **HIPAA** も同時に対応完了。有料 Add-on を有効化し BAA を締結することで PHI（保護医療情報）を扱える。以前は医療系 SaaS での採用が難しかったが、2023 年以降は選択肢に入れやすくなった。
- ISO/IEC 27001:2022 は 2026 年に取得（May 2026 Developer Update で発表）。

**Branching が本番ワークフローで使えるレベルに**

- LWX（2023-12）でプレビュー → GA Week（2024-04）でパブリックベータ → LW15（2025-07）で **Branching 2.0**（GitHub 不要）と段階的に進化。CI/CD 連携が必須だったフェーズを経て、現在はダッシュボード単体で「PR 的なレビュー→マージ」が完結する。

**AI ワークロードのバックエンドとして Supabase が選ばれやすくなった**

- pgvector・Automatic Embeddings・Vector Buckets・MCP Server の組み合わせにより、「Supabase だけでベクトル検索 RAG を完結させる」構成が現実的になった。LW14 で Supabase が公式に「AI Builder のデフォルトバックエンドになる」と宣言しており、今後も強化が続くと見られる。

**サードパーティ Auth の統合パスが充実**

- Auth0・Cognito・Firebase・Clerk を既存のまま使いながら Supabase の DB・Storage・Realtime だけを採用する構成が公式にサポートされた（LW12〜LW14）。「Auth は移行できないが Supabase の他機能は使いたい」ケースへの回答が明確になった。

**スケール対応の強化**

- Read Replicas（LWX）・High Performance Disks（LW13）・Dedicated Poolers（LW14）・Analytics Buckets（LW15）と、エンタープライズ規模のワークロードに対応する機能が毎回追加されている。2025 年 10 月に Series E で $5B 評価額（Accel・Peak XV 主導）を達成しており、資金面での継続的な開発基盤も確認できる。
