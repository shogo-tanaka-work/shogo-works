## TL;DR

- 「複数エージェントを同時に動かす」前提でUIを完全再設計
- **並列セッション管理サイドバー**・**Side Chat（⌘+;）**・**拡張diffビューワー**が主要追加機能
- ドラッグ&ドロップでターミナル・チャット・プレビューを自由配置

---

## 概要

2026年4月15日前後（v2.1.111〜112系）、Claude Code のデスクトップアプリが全面刷新されました。従来は「1つのターミナルで1つのエージェントと作業する」構造でしたが、新設計は **複数エージェントの並列稼働** を前提としています。

---

## 主要な新機能

### 並列セッション管理サイドバー

複数リポジトリ・複数タスクのセッションを1画面で管理できます。

- **ステータス・プロジェクト・環境でフィルタリング**
- PR マージ時に対応セッションを**自動アーカイブ**
- セッションの優先度付け・並べ替えに対応

### ドラッグ&ドロップレイアウト

ターミナル・チャット・diffビューワー・プレビューペインをグリッド上で自由配置できます。作業スタイル（コーディング中心 / コードレビュー中心 / デバッグ中心）に合わせたレイアウトを保存して切り替え可能です。

### Side Chat（⌘+;）

実行中タスクのコンテキストを引用しながら **別スレッドで質問** できる機能です。メインスレッドには影響しません。

**活用シーン：**
- エージェントが実行中に別の実装方針を素早く確認する
- 完了を待たずに次のタスクを事前設計する
- エラーの原因を別視点で調査する

### 表示モード（3段階）

| モード | 内容 |
|---|---|
| Verbose | ツール呼び出し・推論ステップを全表示 |
| Normal | 主要な操作と結果を表示 |
| Summary | 完了サマリーのみ表示 |

### 統合ターミナル

アプリを離れずにテスト実行・ビルド・コマンド操作ができます。エージェントが提案したコマンドをワンクリックで実行することも可能です。

### 再設計 diff ビューワー

- 大規模な差分でもレンダリングが重くならない設計
- **インラインコメント** でエージェントとの議論が可能
- 変更のステージング・アンステージをビューワー上で直接操作

### 拡張プレビューペイン

| コンテンツ | サポート |
|---|---|
| HTML ファイル | ✅ |
| PDF | ✅ |
| ローカル開発サーバー | ✅（localhost ポートを指定） |
| Markdown | ✅ |

## 効果的な使い方

### 並列ワークフローの例

```
セッション1: フロントエンドのバグ修正（main ブランチ）
セッション2: 新機能の実装（feature/new-auth ワークツリー）
セッション3: テストの補完（test/coverage ワークツリー）
```

サイドバーで3セッションの進捗を確認しながら、必要なセッションに切り替えて介入します。

### Side Chatの活用

```
# メインエージェントが認証処理を実装中…

⌘+; → Side Chat を開く
「この実装でJWTの有効期限切れはどう扱われますか？」
→ 回答を確認してから /effort xhigh で詳細対応を指示
```
