## このレッスンでわかること

このカリキュラムの最初のレッスンへようこそ。第1章では「Claude Code とはそもそも何者なのか」を、専門用語をかみ砕きながら整理していきます。

このレッスンを読み終えると、次の3つが説明できるようになります。

- Claude Code が ChatGPT のようなチャットAIと**根本的に何が違うのか**
- Claude Code で**具体的に何ができるのか**（プログラミング以外も含めて）
- なぜ**非エンジニアにこそ**学ぶ価値があるのか

「自分はプログラマーじゃないから関係ない」と思っている方ほど、このレッスンは役に立つはずです。順番に見ていきましょう。

## Claude Code とは何か

Claude Code は、Anthropic（アンソロピック）社が開発した**AIエージェント**です。Anthropic は、AIアシスタント「Claude（クロード）」を作っている会社で、その Claude を「実際に作業をこなす道具」として使えるようにしたのが Claude Code です。

もともとは**ターミナル**で動くツールとして登場しました。ターミナルとは、マウスではなく文字（コマンド）でパソコンに指示を出すための画面のことです。黒い画面に文字がずらっと並ぶ、映画でハッカーが使っていそうなあれです。「うわ、難しそう」と感じるかもしれませんが、安心してください。Claude Code では難しいコマンドを覚える必要はなく、**日本語で話しかけるだけ**で使えます。

現在はターミナル版（CLI）のほかに、デスクトップアプリ、VS Code や JetBrains といったエディタの拡張機能、ブラウザで使える Web 版も提供されています。導入方法や対応環境は変化が早いので、細かい部分は公式サイトで最新情報を確認してください。

名前に「Code」と付いていますが、実際にはコードを書く以外の仕事もこなします。このカリキュラムでは「**ターミナルで動く、何でも屋のAIアシスタント**」というイメージで捉えてもらえれば十分です。

## チャットAIとの根本的な違い：「会話するAI」と「手を動かすAI」

ChatGPT や Claude（チャット版）、Gemini を使ったことがある方は多いと思います。これらと Claude Code の違いを一言でいうと、こうなります。

> チャットAIは「**答えを教えてくれる**」。Claude Code は「**作業を代わりにやってくれる**」。

たとえば「売上データを集計したい」というとき、それぞれのAIはこう動きます。

| | チャットAI（ChatGPT 等） | Claude Code |
|---|---|---|
| データの読み込み | 自分でコピペして貼り付ける | パソコン内のファイルを**自分で開いて読む** |
| 集計作業 | 手順ややり方を**文章で説明**してくれる | 集計プログラムを書いて**その場で実行**する |
| 結果の保存 | 出てきた文章を自分でファイルに貼る | 結果を**ファイルとして自動で保存**する |
| 間違いがあったら | もう一度質問して、また自分で直す | エラーを自分で見つけて**自分で修正**する |

チャットAIは優秀な「相談相手」ですが、最後に手を動かすのは人間です。AIの回答をコピーして、貼り付けて、ファイルを作って、間違いがあればまた質問して……という往復作業が必ず発生します。

Claude Code はこの往復をまるごと引き受けます。具体的には、次のことをAI自身が行います。

- **ファイルの作成・編集**：文書、表データ、プログラムなどを直接作る・書き換える
- **コマンドの実行**：プログラムを動かす、ファイルを変換する、ツールを呼び出す
- **結果の確認と修正**：うまくいったか自分でチェックし、失敗していたらやり直す

この「指示を受けて、計画して、実行して、確認して、修正する」という一連の流れを自律的に回す仕組みを**エージェント**と呼びます。Claude Code は「会話するAI」ではなく「手を動かすAI」、つまりエージェントなのです。

### Bad / Good で見る使い方の違い

イメージをつかむために、指示の出し方を比べてみましょう。

```text
【チャットAIへの相談（Bad というわけではないが、往復が多い）】
「Excelで月別売上を集計する関数を教えて」
→ 回答を読む → 自分でExcelを開く → 関数を入力 → エラー → また質問…

【Claude Code への指示（Good）】
「デスクトップの sales.csv を読んで、月別の売上合計を集計して、
 結果を summary.csv として保存して」
→ Claude Code がファイルを読み、集計し、保存まで完了させる
```

人間がやるのは「何をしてほしいかを伝える」ことと「結果を確認する」ことだけ。この役割分担の変化が、Claude Code を学ぶ最大の理由です。

## できることの全体像

「コードを書くツール」という枠を外して、Claude Code でできることを業務目線で整理すると、大きく5つの領域に分けられます。

| 領域 | できることの例 |
|---|---|
| 業務自動化 | 大量ファイルのリネーム・整理、定型レポートの自動生成、フォルダの一括処理 |
| 資料・文書作成 | 議事録の要約・整形、提案書のドラフト作成、マニュアルの下書き |
| データ処理 | CSV/Excel の集計・変換・突合、データのクレンジング、グラフ用データの準備 |
| Web 制作 | 自分のホームページやランディングページの作成、既存サイトの文言修正 |
| コード開発 | プログラムの新規実装、バグ修正、テスト作成、コードレビュー |

ポイントは、**5つのうち4つはプログラミングの知識がなくても恩恵を受けられる**ということです。たとえば「この100個のファイル、日付ごとにフォルダ分けして」と日本語で頼めば、Claude Code が裏側で必要なプログラムを書いて実行してくれます。あなたがプログラムの中身を理解する必要はありません。

もちろん、Claude Code の利用には Claude のサブスクリプション（Pro / Max プラン）または API の従量課金契約が必要です。料金まわりはレッスン 1-3 で詳しく扱います。

## 非エンジニアにこそ価値がある理由

「結局エンジニア向けのツールでは？」という疑問に答えておきます。私は、むしろ非エンジニアのほうが得るものが大きいと考えています。理由は3つあります。

**1. 「自動化の壁」が消えるから**

これまで業務の自動化には「プログラミングを学ぶ」か「エンジニアに依頼する」しか道がありませんでした。Claude Code は、日本語の指示をプログラムに翻訳して実行までしてくれるので、この壁が事実上なくなります。毎週30分かかっていた集計作業が、一言の指示で終わるようになる──これはエンジニアにとっては「便利」ですが、非エンジニアにとっては「不可能が可能になる」変化です。

**2. 反復作業の比率が高い職種ほど効果が大きいから**

営業、マーケティング、企画、事務といった職種には、レポート作成・リスト整理・データ転記のような定型作業が多く含まれます。定型作業はまさにAIエージェントの得意分野です。一方エンジニアの仕事には、すでに多くの効率化ツールが存在します。伸びしろが大きいのは非エンジニア側なのです。

**3. 「指示する力」は誰でも鍛えられるから**

Claude Code を使いこなす鍵は、プログラミングスキルではなく「やってほしいことを明確に伝える力」です。これは部下や外注先への依頼と同じ、ビジネスパーソンが日常的に使っているスキルの延長線上にあります。

ただし、過度な期待は禁物です。Claude Code も間違えることはありますし、指示が曖昧なら見当違いの作業をすることもあります。「優秀だが新人のアシスタントに仕事を頼む」くらいの距離感で付き合うのが、上手な使い方の第一歩です。

## このカリキュラムの歩き方

本カリキュラムは全10章で構成されています。全体像は次のとおりです。

| 章 | テーマ | ねらい |
|---|---|---|
| 第1章 | Claude Code を知る | 何ができるか・料金を理解する（いまここ） |
| 第2章 | 導入とセットアップ | 自分のPCで動かせるようにする |
| 第3章 | 基本操作 | 日本語で指示して結果を受け取る流れに慣れる |
| 第4章 | 指示の出し方 | 伝わる指示・伝わらない指示の違いを学ぶ |
| 第5章 | ファイル・データ操作 | 文書やExcel/CSVを扱えるようになる |
| 第6章 | 業務自動化の実践 | 自分の定型業務を自動化してみる |
| 第7章 | CLAUDE.md と設定 | 毎回の指示を覚えさせて効率化する |
| 第8章 | 外部ツール連携（MCP） | 普段使うサービスとつなげる |
| 第9章 | チーム活用 | 周囲に展開し、仕組みとして定着させる |
| 第10章 | 総合演習 | 実務テーマで一連の流れを完走する |

※章立ては学習の便宜上の構成です。順番どおりに進めるのがおすすめですが、すでに環境構築が済んでいる方は第3章から始めても構いません。各レッスンの最後には演習とクイズが付いているので、「読んで終わり」にせず手を動かしながら進めてください。

## やってみよう

1. **自分の業務の棚卸し**：普段の仕事の中で「毎回同じ手順でやっている作業」を3つ書き出してみましょう。それが Claude Code に任せる候補リストの原型になります。
2. **領域へのあてはめ**：書き出した3つの作業を、本文の表にある5領域（業務自動化／資料・文書作成／データ処理／Web制作／コード開発）のどれかに分類してみましょう。
3. **指示文の下書き**：そのうち1つについて、「新人アシスタントに頼むつもりで」依頼文を書いてみましょう。何のファイルを使い、何をして、結果をどう出してほしいか──この3点が入っていれば合格です。

## まとめ

- Claude Code は Anthropic 製のAIエージェントで、ターミナルのほかデスクトップアプリ・エディタ拡張・Web 版でも使える
- チャットAIが「答えを教える」のに対し、Claude Code は**ファイル作成・編集・コマンド実行まで実際に行う**
- できることは業務自動化・資料/文書作成・データ処理・Web制作・コード開発の5領域に整理できる
- 5領域のうち4つはプログラミング知識がなくても恩恵を受けられる
- 使いこなす鍵はプログラミングスキルではなく「やってほしいことを明確に伝える力」
- 本カリキュラムは全10章。演習を実際に手を動かしながら進めるのが上達の近道

- A. Claude Code のほうが日本語の文章がうまい
- B. Claude Code はファイル操作やコマンド実行など「作業そのもの」を実行できる
- C. Claude Code は無料で使える
- D. Claude Code はインターネット検索が速い

<details>

**正解：B**

チャットAIは回答（文章）を返すところまでが仕事で、その先の作業は人間が行います。Claude Code はエージェントとして、ファイルの作成・編集、コマンドの実行、結果の確認と修正までを自律的に行える点が本質的な違いです。なお C は誤りで、利用には Claude Pro/Max のサブスクリプションまたは API 従量課金が必要です。

**問2. 非エンジニアが Claude Code を学ぶ価値の説明として、本文の内容に合うものはどれでしょう？**

- A. プログラミング言語を暗記しなくても、日本語の指示で業務を自動化できるから
- B. Claude Code を使えばAIが絶対に間違えなくなるから
- C. エンジニアの仕事を完全に代替できるから
- D. ターミナルのコマンドを全部覚えられるから

<details>
<summary>解答を見る</summary>

**正解：A**

Claude Code は日本語の指示を受けて、必要なプログラムの作成・実行まで代行します。そのため「自動化したいがプログラミングはできない」という非エンジニアの壁を取り除ける点に大きな価値があります。B は誤りで、AIも間違えるため結果の確認は人間の役割です。「優秀な新人アシスタント」くらいの距離感で付き合うのが適切です。

</details>
