## 教材化ポイント

### BYOK が「AI 導入のボトルネック」を変える

コンプライアンス要件でクラウド AI の採用を躊躇していた企業が多いが、BYOK はその障壁の一つを取り除く。「データが GitHub のサーバーに行くのが問題」という懸念に対して、「自社テナント経由でルーティングできる」という回答を出した。

授業では「企業が AI を導入しない理由は何か」という議論のフレームとして使える。コスト・セキュリティ・規制・スキル不足の中で、BYOK がセキュリティ懸念に対して何を提供するか、それで十分かを議論できる。

### フロンティアモデルとローカルモデルの混在設計

Ollama（ローカル LLM）と Claude Opus（クラウド）を同一ツールで使い分けられる設計は、「どのタスクにどのモデルを使うか」という AI 調達の最適化問題として教材化できる。コスト・速度・品質・セキュリティのトレードオフを実ツールで説明できる具体例。

### Anthropic が BYOK プロバイダーに含まれる意味

GitHub Copilot（Microsoft 傘下）が Anthropic を BYOK プロバイダーとして正式サポートしたことは、「AI ツールのプロバイダー横断対応」として重要。Microsoft は OpenAI と深い資本関係があるが、Anthropic も公式に Copilot から繋げる。競合モデルを同じ UI から選べる設計は、ユーザーにとって「ベンダーロックインを防ぐ」という価値を提供する。
