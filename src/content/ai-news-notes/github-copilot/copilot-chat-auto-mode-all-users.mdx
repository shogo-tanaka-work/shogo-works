## 教材化のポイント

### Auto mode の仕組み

「システム状態」という表現が公式に含まれている点が重要。単にタスク難易度だけでなく、現在の API 負荷やモデル応答速度なども判断材料になっているとみられる。

教材では「完全に自分でコントロールしたい場合はモデルを手動指定、最適化を任せたい場合は auto」という選択軸を示せる。

### 10% 割引の意味

- 有料ユーザー限定（Free は割引なし）
- auto mode を使うことで割引になる → auto mode への誘導施策と読める
- チームでの総利用量が多い Business / Enterprise では実際のコスト差になる

### モデル選定教材との連携

既存の「どのモデルを使い分けるか」教材がある場合、「auto を使う選択肢」を追加の選択肢として提示する。完全に代替するわけではなく、特定タスクには手動選択が合う場面も残る。
