## 教材化ポイント

### 1. CLI→アプリへの展開は「機能の恒常化」のシグナル

2026年6月にCopilot CLIへ実験的プレビューとして追加された`/security-review`が、7月にはCopilotアプリでも全プラン向けパブリックプレビューとして展開された。単一インターフェースの実験機能から、複数インターフェースでの標準機能へ格上げされていく典型パターン。

**教材への落とし込み方:**
- 「実験的プレビュー→複数サーフェス展開→GA」というAI機能のロールアウト段階を見分ける実例として提示
- 受講者に「CLI限定の機能を見た時点で、アプリ版への展開を予測する」視点を持たせる

### 2. PRフローへのセキュリティレビュー組み込みタイミングの設計

Code Scanning・DependabotはPR後（CI実行時）に検出結果が返るのに対し、`/security-review`はコーディング中・PR作成前に任意で実行できる。検出タイミングを開発フローのどの段階に置くかという設計論点の実例になる。

**教材への落とし込み方:**
- 「Shift Left（早期発見）」の具体例として、AIレビューをコーディング段階に組み込む設計を紹介
- 既存のセキュリティ教材で「PR前チェック」の章に、Copilotの実例として追記可能

### 3. 全プラン提供という点の意味

Business/Enterprise限定ではなくFreeプランを含む全プランで提供される点は、セキュリティ機能をアクセス制限なしに広く使わせる方針の表れ。個人開発者・小規模チームでも導入コストゼロで試せる。

**教材への落とし込み方:**
- 「セキュリティ機能は誰でも使えるべき」という設計思想の実例
- 個人開発者向け教材で「無料でできるセキュリティ対策」の一項目として紹介
