## 教材化ポイント

### 「管理者が有効化する」という設計判断

個人プランでは即座に使えた MAI-Code-1-Flash が、Business/Enterprise では管理者のポリシー有効化を必要とする設計は、企業向け SaaS の標準的な権限管理モデル。

「新機能をデフォルトで全チームに有効にする」のではなく「管理者が明示的に許可する」設計は、課金モデル（使用量ベース）との組み合わせで理解できる。コスト管理責任が個人ではなく組織にあるため、組織単位でのコントロールが必要になる。

「機能追加 ≠ 即時全社展開」という企業システムの設計原則を、AI ツールの具体例で説明できる。

### Microsoft AI 独自モデルの展開戦略

MAI-Code-1-Flash は Microsoft AI が開発した独自モデルを GitHub Copilot に組み込んだもの。OpenAI との提携で GPT 系モデルを使いつつ、独自モデルも並行して展開するという戦略は、「特定プロバイダーへの依存を減らすリスクヘッジ」として読める。

「基盤モデルの調達戦略」という教材テーマで、Google（Gemini）・Amazon（Titan）・Microsoft（MAI）が独自モデルを抱える理由と、外部モデル（Claude・GPT）を使う理由のトレードオフを議論できる。

### Free/Pro → Business/Enterprise の展開順序

2026-06-18 に個人プランで先行リリース、2026-06-26 に企業プランで GA という順序は、「個人ユーザーで動作確認してから企業環境に展開する」PLG（Product-Led Growth）の定石パターン。

個人ユーザーの利用フィードバックを受けつつエンタープライズ向けに安定化する開発サイクルは、B2B SaaS 製品の開発ロードマップ設計の具体例として使える。
