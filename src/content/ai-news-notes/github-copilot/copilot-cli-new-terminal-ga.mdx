## 教材化ポイント

### MCP 設定の UI 統合が示す「設定体験の進化」

`/mcp add` コマンド一つで MCP サーバーを追加できるようになった点は、「AI ツールの設定体験」がどう進化してきたかを示す教材になる。初期の AI ツールは設定ファイルを手書きするのが前提だったが、UI とコマンドで完結するようになってきた。これは「技術者だけでなく非技術者も設定できる」方向への変化として読める。

### アクセシビリティ対応の実装例

`/theme high-contrast` や colorblind モード、スクリーンリーダー自動検出は、「開発ツールのアクセシビリティ」という切り口で教材化できる。多様なユーザーが使えるツール設計の具体例として、UX 設計の授業でも使えるネタ。

### CLI ツールの「リポジトリ文脈認識」

リポジトリ内で起動すると Issues・PR・Gists のタブが自動追加される機能は、「ツールがコンテキストを認識して UI を変える」設計の実装例。これはエージェント型ツールの「コンテキスト適応」という概念の具体例として使える。
