GitHub Copilot 向けの Knowledge 教材は現状未整備のため、教材本体への追記は行わない。ニュース記事 + 本メモに留める。

恒久的に効く論点として記録しておく価値があるのは次の2点。1点目は「IDE 拡張のうち Eclipse 版だけが MIT で OSS 化され、VS Code / JetBrains 版はクローズドのまま」という非対称。AI ツールガバナンスを扱う教材を将来作る場合、「同じ Copilot でも IDE によって透明性のレベルが違う」という具体例として使える。2点目は `microsoft/copilot-for-eclipse` が、IDE 拡張による LLM 補完・チャット・エージェント・MCP 連携の実装リファレンスになること。MCP 連携や IDE 拡張開発を扱う教材ができたときに、実プロダクトのコード例として参照先に挙げられる。

速報的な内容（リポジトリ URL、発表日）は教材へ持ち込まず、必要になった時点でニュース記事を参照する形にする。
