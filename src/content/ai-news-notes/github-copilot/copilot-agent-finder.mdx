## 教材化のポイント

### ARD（Agentic Resource Discovery）という新仕様

ARD は MCP エコシステムの「発見」問題を解決するために登場した仕様。教材では:

```
MCP エコシステムの課題:
- サーバーが増えた → どれを使えばいいかわからない
- 設定が複雑 → 入門者が試しにくい

ARD の役割:
- 「何をしたいか」を伝えれば対応サーバーを発見
- 設定手順を自動化する入口になる
```

「MCP サーバーの discover 問題」という表現で位置づけられる。

### Enterprise のプライベートレジストリ

企業導入の観点では重要な機能:
- IT 部門が承認済みツールをレジストリに登録
- 開発者全員が同じ手順でアクセス
- シャドー IT リスクの軽減

教材では「組織での MCP 管理方針」の章で取り上げられる。

### 教材での位置づけ

「MCP 入門」の章に「ツールの発見方法」として追加できる。
- 手動で探す方法（GitHub、MCP サーバー一覧）
- Agent Finder で探す方法（github.com 上で自然言語）

の対比として説明できる。
