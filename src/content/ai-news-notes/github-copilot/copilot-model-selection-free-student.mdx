## 教材化ポイント

### 「選択肢を減らす = UX 改善か差別化か」という問い

GitHub は「Auto の方が最適なので手動選択は不要」と説明しているが、これは同時に Free と有料プランの機能差別化でもある。「ユーザーの選択を減らすことが UX 向上か、それとも有料化誘導か」という二面性を議論する教材として使える。

Apple の「iPhone はデフォルト設定が最善、カスタマイズは不要」設計思想との比較としても使える。

### 有料プランの差別化要素として「制御権」を使う設計

Copilot Business・Enterprise が「モデル選択権」を維持しているのは、「高度なユーザーは自分でコントロールしたい」というニーズに応える設計。SaaS のプランティアリングで「制御権のグラデーション」を機能差別化に使う事例として教材化できる。

### 「Auto」という名前とブラックボックス化

Copilot Auto は内部でどのモデルを使っているか表示されない。「AI がどのモデルを使ったか分からない」という状況は、AI ツールの透明性・説明可能性の議論（Explainability in AI）に繋げられる。
