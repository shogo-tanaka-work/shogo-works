Workspace のガバナンス／管理自動化章に、Policy API の DLP 書き込みエンドポイント（Create/Update/Delete）を追記する。「管理コンソール手動運用」と「API によるコード運用（IaC）」を対比し、DLP ルールをバージョン管理・差分レビュー・複数環境へ一括反映する運用例を示す。

対象範囲（Drive / Gmail / Chat / Chrome）、スーパー管理者権限前提、API 実行アカウントの権限保護を明記する。情シス・セキュリティ部門向けに、監査証跡・環境間の再現性・一括更新という3つの実務メリットを整理し、Workspace 運用代行/情シス支援の受託メニューとしての展開可能性にも触れる。
