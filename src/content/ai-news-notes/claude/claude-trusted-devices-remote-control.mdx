## 教材化ポイント

### ゼロトラストモデルの具体例

「誰でも繋がれる状態がデフォルト」から「デバイスを認証してから繋げる」への移行は、ゼロトラストセキュリティモデルの典型的な実装。「社内ネットワークにいれば安全」という境界型セキュリティの対比として教材化できる。

Claude Code の Remote Control 機能に Trusted Devices を重ねることで、「AI ツールのリモート操作にもゼロトラストが必要になってきた」という時代の変化を示せる。

### 管理者が「操作可能な範囲」を定義する設計

Trusted Devices のポリシーは管理者が有効化する設定であって、デフォルトは（おそらく）オフ。「ツールが機能を追加しても、デフォルトは変えない → 管理者が明示的にポリシーを選択する」というアプローチは、エンタープライズ向け SaaS ツールの標準設計。

「機能を追加することと、その機能をデフォルト有効にすることは別の決断」という設計の考え方を教材として伝えられる。

### AI エージェントのリモート操作リスク

Claude Code は「ローカルで動くコーディングエージェント」という性質上、リモートから操作されると機密コードやファイルシステムへのアクセスに繋がりうる。通常のリモートデスクトップや SSH 接続のリスク管理と同じ枠組みで考えられる点は、「AI ツールのセキュリティも既存のセキュリティフレームワークで管理できる」という教材テーマに使える。
