「マルチエージェント」章で、サブエージェントが最大5階層まで自分のサブエージェントを生成できる点を追記する。`daily-ai-news-pipeline` スキルのようにオーケストレーターが複数の調査エージェントを生成し、各調査エージェントがさらに深化エージェントを生成する構成例を示す。ネスト上限（5階層）の意識も添える。

「Amazon Bedrock 設定」章で `AWS_REGION` 未設定時のフォールバック挙動（`~/.aws/config` → AWS SDK 優先順序に準拠）を追記し、`/status` でリージョン取得元を確認する方法も示す。

「オブザーバビリティ」章で、`claude_code.lines_of_code.count` OTEL メトリクスに `model` 属性が付与されたことを追記し、モデル別の利用状況計測の実装例を更新する。

「エンタープライズ制御」章で `availableModels` の制限がサブエージェント・エージェントディスパッチ・アドバイザーモデルに適用されるようになった（バグ修正）ことを追記し、ポリシー整合性を前提として設計を記述するよう修正する。
