## 教材化ポイント

### sandbox.credentials — AI ツールのサンドボックス設計

`sandbox.credentials` でサンドボックス内コマンドが認証情報を読めなくなる設定は、「AI ツールをセキュアに組織展開するための設計」の具体例。Claude Code のようなコード実行 AI において「実行環境に何を持ち込ませないか」というポリシー設計の重要性を教材化できる。

類似概念: Docker の secrets、GitHub Actions の secrets ブロックと同じ「最小権限の原則」を AI ツールに適用した例として説明できる。

### 組織モデル制限の徹底

管理者が組織設定でモデルを制限しても、これまで Claude Code 側では別モデルが選べてしまう不整合があった。今回の修正で「組織ポリシー = ツールの実際の動作」が一致するようになった。これは「管理者が設定したポリシーがエンドポイントまで一貫して適用されること」という AI ガバナンスの基本要件の話として教材化できる。

### CJK ペースト修正の技術背景

「バイト単位で拡張キーイベントを送るターミナル」でマルチバイト文字がバグる、という問題はターミナル入力の文字コード処理の教材として使える。日本語ユーザーが直面しやすいバグの種類として覚えておく価値がある。

### MCP ツールタイムアウトの設計

リモート MCP ツールが 5 分間応答なしでハングしていたバグ修正は、「MCP サーバーが応答しない場合のフォールバック設計」の教材として使える。`CLAUDE_CODE_MCP_TOOL_IDLE_TIMEOUT` で上書き可能な設計は「デフォルト値と設定可能値の両立」というエンジニアリング判断の例としても使える。
