## 教材化ポイント

### 1. 既存教材内の「default」表記を「Manual」へ更新

権限モードを解説している既存教材・スクリーンショットで「default」という表記を使っている箇所があれば、「Manual」への更新が必要。挙動自体は変わらないため、説明ロジックはそのまま流用できる。

**教材への落とし込み方:**

- 権限モード一覧表を持つ教材があれば、モード名だけ差し替える
- `--permission-mode manual` という明示指定の書き方も併記しておくと親切

### 2. AskUserQuestion自動継続廃止は「安全側のデフォルト変更」の好例

「便利さより安全を優先したデフォルト変更」という設計判断の実例として、AI運用ガバナンス系の教材で引用できる。

**教材への落とし込み方:**

- 無人バッチ運用をしているチーム向けに「`/config` でのオプトインが必要になった」という注意喚起を追記する
