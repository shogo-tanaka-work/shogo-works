## 教材化ポイント

### 1. エージェントの自律度の進化

バックグラウンドエージェントが「コードを書く → 自動コミット → PR作成」まで完結できるようになった。これは「AIが人間の確認なしに完結できるスコープがどこまで広がったか」を示す教材として使える。

**教材への落とし込み方:**
- エージェントの自律化ステップ（提案 → 実行 → 完了報告）の図解例として利用
- 「どこで人間が確認・承認すべきか」のガバナンス議論の起点に

### 2. Claude in Chrome の意味

Claude Code は元々CLIツールだったが、ブラウザ操作（Claude in Chrome）が統合された。「CLIからブラウザまでを単一インターフェースで制御できる」という拡張の方向性の典型例。

**教材への落とし込み方:**
- "Code → Browser → Action" のエージェントパイプラインの例として使える
- RPA（ロボティック・プロセス・オートメーション）との比較で AIエージェントを説明する材料

### 3. Hook設計（イベント駆動型エージェント）

`agent_needs_input` / `agent_completed` という2種類のHookが追加された。これは「エージェントを非同期で動かし、状態変化を外部に通知する」というイベント駆動型設計の実例。

**教材への落とし込み方:**
- Slackへの通知連携、JIRA ticket更新、メール送信などの実装例を添えると具体性が増す
- Webhook / EventBus 設計パターンとの比較

### 4. Explore エージェントのモデル継承問題（修正例）

従来 Explore エージェントが Haiku デフォルトだった → セッションモデル（Opus上限）を継承するよう修正された。

**教材への落とし込み方:**
- 「マルチエージェント構成でのモデル選定のデフォルト設計」が無意識に貧弱になりやすい、という注意点の実例
- 親セッションのコンテキストを子エージェントに継承させるパターンの説明に
