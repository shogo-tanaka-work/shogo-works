## 教材化のポイント

### /config コマンドの位置づけ

「設定をどこで変えるか」という問いへの答えが二層になった:

1. **永続設定**: `~/.claude/settings.json` または `.claude/settings.json`（プロジェクト）
2. **セッション内即時変更**: `/config key=value`

教材では両方を並べて説明すると理解しやすい。「なぜ二つあるか」という問いに答える形で、「再現性のある設定」vs「試行錯誤中の設定」という使い分けを語れる。

### ハンズオン例

```
# セッション内でモデルを切り替えて試す
/config model=claude-haiku-4-5-20251001
# 軽いタスクを実行してコストを確認

/config model=claude-opus-4-8
# 複雑な推論を実行して比較
```

### Bun 1.4 アップグレードの意味

Claude Code 自体が Bun で動いているため、Bun のバージョンアップはランタイム全体の安定性に影響する。受講者がバンドラ・ランタイムに詳しくない場合は「動作が安定した」程度の説明でよい。

### Apple Events オプション

macOS 固有の機能で、一般的なクラウド開発環境では使わない。デモやハンズオンで macOS を使う場合のみ言及すれば十分。
