## 教材化ポイント

### /rewind — 非破壊的な AI との試行錯誤

`/clear` でリセットした後に巻き戻せる `/rewind` は、「AI との対話を非破壊的に実験できる」設計の具体例。プロンプトエンジニアリングの授業で「どうやって試行錯誤するか」を説明する際に使える。

一度消したものに戻れるというのは、Git のコミット・stash の考え方に近い。「AI との会話履歴をバージョン管理する」という発想として教材化できる。

### CPU 37% 削減のアーキテクチャ

100ms コアレッシング（テキスト更新を合算）でストリーミング CPU を 37% 削減した。これは「高頻度の UI 更新を debounce/throttle する」という標準的なフロントエンド最適化パターンを CLI に適用した例。React の useState 最適化や仮想 DOM の考え方と同じ原理として説明できる。

### バックグラウンドエージェントの「停止の永続化」問題

AI エージェントが「止めても復活する」という問題はエージェント管理の典型的な課題。これは人間のプロセス管理（`kill -9` と普通の `kill` の違い）とも類比できる。「エージェントを確実に停止させるためにどんな設計が必要か」という授業テーマになりうる。

### MCP の信頼性設計パターン

MCP サーバーが一時的にダウンしたときにリトライ・バックオフで対応する設計は、分散システムの「retry with exponential backoff」の実例。Claude Code がどのように外部サービスとの接続を堅牢化しているかを示す実装教材として使える。
