## 教材化ポイント

### MCP 認証の自動化可能性

`claude mcp login <name>` の追加は「MCP サーバーを使った自動化パイプライン」の文脈で重要。これまで初回認証が手動ステップになっていたが、CI/CD やスクリプトからのセットアップが可能になった。SSH 環境（`--no-browser`）での対応も含めて、サーバー上での Claude Code 活用事例として教材化できる。

### `!` Bash 自動応答によるエージェントループの強化

「コマンド実行 → 出力確認 → Claude が次のアクションを決める」フローが一体化した。これはシステム設計として「Plan → Execute → Observe → Decide」サイクルの Observe フェーズが自動化された例。エージェント設計の教材として、「人間のフィードバックが不要になる条件」を議論する際に使える。

### Workflow スキーマ無限ループ修正

`agent({schema})` でスキーマ検証失敗が続くと無限ループしていた問題が5回制限で修正された。これは「AI エージェントに必要な安全弁」の具体例。「LLM が間違え続けた場合にどう止めるか」というロバスト設計の実例として使える。
