# shogoworks SNS運用プレイブック

作成日: 2026-05-05

## 目的

このドキュメントは、shogoworks のSNS運用方針をまとめたものです。

目的は、SNSを「なんとなく発信する場所」ではなく、以下の導線として使うことです。

- B2B向けの信頼形成
- AI業務自動化診断へのリード獲得
- note / YouTube / MENTA / 自サイトへの回遊
- AI業務システム構築の専門性証明
- 個人相談・教材・ワークショップの需要検証

対象SNS:

- LinkedIn
- X
- Threads
- note
- YouTube
- MENTA
- Instagram
- Facebook

結論として、主戦場は以下です。

1. LinkedIn
2. note
3. Threads
4. X
5. MENTA
6. YouTube

InstagramとFacebookは捨てるというより、主戦場にはしない。Threads連携・プロフィール補助・将来のPersonal / Lifestyle Lab用として最低限に留める。

## 前提

### 事業上の中心メッセージ

SNS全体で一貫させるメッセージは以下。

> AI業務システムを、診断・構築・運用まで支援します。

もう少し柔らかく言うなら以下。

> AIと自動化で、現場の仕事を減らす。必要なら動くシステムとして構築し、運用まで支える。

### 売りたいもの

SNSから最終的につなげたい商品は以下。

- 90分AI業務自動化診断
- AI業務自動化システム構築
- AIネイティブ プロダクト / 業務アプリ開発
- AI運用・エージェント活用伴走
- AI実践ワークショップ
- MENTA個人相談
- note有料記事 / テンプレート
- 将来的なYouTube / ライブ講座 / Udemy

### 運用方針

AIは使う。

- 投稿アイデア出し
- 下書き
- 長文から短文への展開
- note記事からThreads/X/LinkedIn投稿への分解
- YouTube台本案
- 分析メモの整理

ただし、初期は自動投稿・自動DM・自動いいね・自動コメントは使わない。

理由:

- LinkedInはスクレイピングやLinkedIn上の活動自動化を行う第三者ツールを禁止している
- Xも非APIのWeb操作スクリプトやスパム的自動投稿に厳しい
- 初期はアルゴリズム最適化より、どの言葉が自分の顧客に刺さるかを見る方が重要
- 手動投稿の方が、返信や反応から顧客理解が進む

参考:

- LinkedIn Automated activity: https://www.linkedin.com/help/linkedin/answer/a1340567
- X Automation rules: https://help.x.com/en/rules-and-policies/x-automation

## 全体戦略

### SNS別の役割

| SNS | 役割 | 優先度 | 運用方針 |
|---|---|---:|---|
| LinkedIn | B2Bリード・専門性証明 | 1 | プロフィールを営業LP化し、週2〜3投稿 |
| note | 長文の信頼形成・検索資産・有料導線 | 2 | 週1記事、月1有料記事候補 |
| Threads | 軽い会話・note拡散・日次ログ | 3 | 週3〜5投稿、noteと相互送客 |

| Facebook | 現時点では優先度低 | 低 | 必要時のみページ整備 |

### 1つのネタの展開方法

最初からSNSごとに別ネタを作らない。1つの素材を分解する。

例: `n8n / GAS / Make の使い分け`

1. note
   - 2,000〜4,000字の記事にする
   - 比較表、ユースケース、判断基準を入れる

2. LinkedIn
   - 意思決定者向けに「どう選ぶべきか」を短く投稿
   - 業務目線、コスト、運用性を中心にする

3. Threads
   - 3〜5行の気づきに分解
   - 何度か小出しする

4. X
   - 技術者向けTipsにする
   - 実装時のハマりどころを書く

5. YouTube
   - 5〜10分の比較解説にする
   - 画面デモを入れる

6. MENTA
   - 「あなたの業務ならどれを使うべきか相談できます」とプランに反映

## LinkedIn

### 位置づけ

LinkedInは最重要。

B2B向けの主戦場として使う。目的はバズではなく、経営者・事業責任者・採用/教育担当・AI導入担当に「この人はAI業務自動化を任せられそう」と思ってもらうこと。

LinkedInでは、B2B購買に直接出てこない社内関係者、いわゆる hidden buyers にもThought Leadershipが影響する。Edelman-LinkedInの2025年調査では、B2Bの購買意思決定においてThought Leadershipが信頼形成や社内合意に影響するとされている。

参考:

### プロフィール設計

案1:

案2:

> Web開発 × AI業務自動化 | 小規模法人向けにAI業務システムを構築します

> AI Automation / Web App Developer | 業務自動化・AIエージェント・AIネイティブ開発

英語も少し入れるなら、検索性のために `AI Automation`, `AI Agent`, `Web App Development` を含める。

1. 何をしている人か
2. 誰を支援しているか
3. どんな課題を解くか
4. 具体的な提供サービス
5. 実績・強み
6. 相談導線

例:

```text
AI・自動化・Web開発を組み合わせて、現場の繰り返し業務を減らす仕組みを作っています。

対応領域:

AI講師・セミナー経験、100名以上の指導実績、AI開発ツールやハーネスエンジニアリングに関するKnowledge記事を継続発信しています。
```

### 投稿スタイル

頻度:

- 週2〜3本から開始
- 慣れたら週3〜5本

LinkedInでは品質を落とさず週2〜5投稿が現実的。Bufferの2025年調査では、週2〜5投稿で投稿あたりインプレッションが伸びる傾向がある。

参考:

- Buffer LinkedIn frequency: https://buffer.com/resources/how-often-to-post-on-linkedin/

投稿型:

1. 業務課題の分解
   - 「AI導入で最初に見るべき業務は、派手なものではなく毎週繰り返すものです」

2. Before / After
   - 「問い合わせ対応をこう変えると、担当者の作業が減る」

3. 導入判断チェックリスト
   - 「n8nに向く業務、GASに向く業務、Webアプリ化すべき業務」

4. 失敗パターン
   - 「ChatGPT研修だけでAI活用が止まる理由」

5. 技術選定の考え方
   - 「Difyで十分なケースと、独自Webアプリにすべきケース」

6. 図解 / ドキュメント
   - LinkedInではnative documentやmulti-imageが強い傾向。週1でスライド風画像を試す。

7. デモ動画
   - 30〜90秒程度で、n8n / GAS / Dify / 業務アプリの動きを見せる。

### LinkedIn投稿テンプレート

#### 課題提起型

```text
AI導入で最初にやるべきことは、ツール選定ではありません。

先に見るべきなのは、毎週・毎月くり返している業務です。

例:
- スプレッドシート転記
- メール通知
- 問い合わせ分類
- 議事録からタスク化
- レポート作成

このあたりは、GAS / n8n / Make / Dify で小さく自動化しやすいです。

いきなり大きなAI導入をするより、1業務を減らす方が現場には定着しやすい。
```

#### 事例分解型

```text
問い合わせ対応をAIで改善するなら、いきなりチャットボットを作るより先に分解します。

1. 問い合わせを分類する
2. よくある質問を抽出する
3. 返信テンプレートを作る
4. 人が見るべきものだけ通知する
5. 履歴をナレッジ化する

この流れなら、AIが苦手な判断を人間に残しつつ、定型部分だけ減らせます。

AI業務自動化は、全部任せるより「人とAIの分担」を設計する方がうまくいきます。
```

#### CTA型

```text
AIで自社業務のどこを減らせるか分からない場合は、90分で棚卸しできます。

見るポイント:
- 繰り返し頻度
- 使っているツール
- 人の判断が必要な箇所
- 自動化した場合の削減時間
- 運用できる担当者

診断では、構築できそうな自動化案まで整理します。
```

### DM方針

DMは自動化しない。

送る相手:

- 投稿にコメントしてくれた人
- 何度かリアクションしてくれた人
- 自分が相手の投稿にコメントした後
- 既に知人関係がある人
- 共通イベント・共通テーマがある人

送らないもの:

- 初手の長文営業
- 一斉送信
- テンプレそのまま
- 即商談URL

DM例:

```text
〇〇さん、先ほどのAI活用の投稿、特に「現場に定着しない」という点が印象に残りました。

自分も最近、AI業務自動化はツール導入よりも、業務の分解と運用設計が大事だと感じています。

もし今後、GAS / n8n / Dify あたりで自動化候補を整理したいタイミングがあれば、壁打ちできます。
```

## Threads

### 位置づけ

Threadsはnoteとの相性が良い、軽い会話・気づき・検証ログの場所。

noteは2026年2月にThreadsシェアボタンを追加し、2025年1月にはThreads投稿のリッチ埋め込みにも対応している。note記事の要点をThreadsに流し、Threadsで反応が良かった話をnoteに深掘りする導線が作りやすい。

参考:

- note Threads共有: https://note.com/info/n/ncd5299e350a8
- note Threads埋め込み: https://note.com/info/n/n409ae478e9fb
- Threads Insights: https://www.facebook.com/help/instagram/3675908612671136/

### Instagram / Facebookとの関係

ThreadsはInstagramアカウントとの連携が強い。だからInstagramを完全に捨てるというより、Threadsの土台として最低限整える。

Instagramの役割:

- プロフィールの本人感
- Threadsとの相互導線
- Personal / Lifestyle Labの軽い置き場
- 筋トレ記録アプリや日常の実践ログ

Facebookの役割:

- 現時点では優先度低
- 個人アカウントで営業しない
- 必要なら将来、法人向けページだけ整備

InstagramとFacebookには無料Insightsがあるが、現時点のshogoworksのB2B戦略ではLinkedIn / note / Threadsの方が優先度が高い。

参考:

- Instagram Insights: https://www.facebook.com/help/instagram/788388387972460

## X
