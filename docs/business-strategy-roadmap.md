## 目的

このドキュメントは、shogoworks の今後の案件獲得・発信・商品設計・ホームページ改善の方針をまとめたものです。

後から見返したときに、次の判断がしやすい状態にすることを目的とします。

- 法人向けに寄せるべきか、個人向けも残すべきか
- AI伴走支援だけでなく、Webアプリ / プロダクト開発もサービスとして置くべきか
- note / Udemy / MENTA / 自サイトをどう使い分けるか
- どこから順番に手をつけるべきか
- 90日単位で何を検証すべきか

## 結論

shogoworks は「AI講師」だけでも「Webエンジニア」だけでもなく、次のポジションで見せるのが最も強い。

> AI時代の小さな開発チーム代行

より具体的には、以下のような立ち位置。

> AIと自動化で、現場の仕事を減らす。必要ならMVP・業務アプリ・社内ツールまで作り、運用まで支える。

海外リサーチを踏まえると、さらに一段具体化して以下の見せ方がよい。

> AI業務システムを設計・構築・運用する人。

ここでいう「AI業務システム」は、単なるChatGPT活用相談ではない。問い合わせ対応、社内FAQ、転記、通知、レポート、リード管理、議事録、営業支援、開発ワークフローなどを、GAS / n8n / Make / Dify / Webアプリ / LLM API で動く仕組みにすることを指す。

このポジションなら、現在持っているスキル・実績・発信テーマが一本につながる。

- AI講師・セミナー経験
- ChatGPT / Claude / Claude Code / Codex の知見
- Dify / n8n / Make / GAS の業務自動化スキル
- Next.js / React / TypeScript / Astro / Cloudflare などのWeb開発スキル
- ハーネスエンジニアリング / コンテキストエンジニアリングの体系的な知見
- Knowledge記事の蓄積

今回の海外調査で分かった重要な修正点は、サービス名を「相談」「伴走」「支援」だけに寄せすぎないこと。英語圏のAI automation agencyは、`AI Audit -> Build -> Operate` のように、診断、構築、運用を一連の導線として売っている。

そのため、ホームページ上の主語は以下に寄せる。

- 弱い: AI活用を支援します
- 普通: AI業務改善を支援します
- 強い: AI業務自動化システムを構築・運用します

## 市場認識

### 1. ソフトウェア開発にAI活用が広がっている

Gartner Japan の2025年調査では、国内のソフトウェア開発においてAI活用が急速に進んでいる。

- コード生成・補完: 49.0%
- コードレビュー: 40.0%
- 要件定義: 39.8%
- AIツール利用者の過半数が生産性向上を実感
- 一方で、セキュリティ、ライセンス、保守性が課題

重要なのは、AIの利用が「コードを書く工程」だけではなく、要件定義・レビュー・プロジェクト管理側にも広がっていること。

つまり、これから価値が出るのは単なる実装者ではなく、AIを使いながら要件定義、設計、実装、品質管理、運用まで扱える人。

参考:

- https://www.gartner.co.jp/ja/newsroom/press-releases/pr-20251015-swdev-ai-survey

### 2. フリーランス市場は「AIを使った高付加価値提供」へ移っている

Findy の2026年調査では、IT/Webフリーランス市場は堅調で、AI活用度と報酬に相関が出ている。

- 平均月単価は約80万円
- 時間単価は5,319円
- AIでコードの50%以上を生成する層は、活用度が低い層より月単価が約10万円高い
- フルスタック対応できるエンジニア需要は依然高い
- 生産性向上を「高単価案件へのシフト」や「付加価値提供」につなげられるかが分岐点

ここから考えると、「Web開発できます」ではなく、「AIを使って速く・安く・保守可能にプロダクトを形にできます」と見せる方が市場に合う。

参考:

- https://findy.co.jp/3922/

### 3. 企業の生成AI導入は進むが、AIエージェント活用はまだ浅い

矢野経済研究所の2026年調査では、国内企業の生成AI利用は43.4%まで進んでいる。一方で、AIエージェントの実利用はまだ3.3%にとどまる。

これは、企業が「AIに興味がある」「ChatGPTなどを使い始めている」段階から、「業務や開発プロセスに組み込む」段階へ移る途中であることを示している。

つまり、今後の需要は以下に寄っていく。

- 社内業務にAIをどう入れるか
- どの業務から自動化すべきか
- 生成AIを安全に使うルールをどう作るか
- AIエージェント / AI駆動開発をどう実務に落とすか
- 作った仕組みを現場で使い続けるにはどうするか

ここに shogoworks の「教えられる」「作れる」「運用に落とせる」が刺さる。

参考:

- https://www.yanoresearch.com/en/press-release/show/press_id/3991

### 4. 中小企業・個人事業主は「AIそのもの」より「面倒な業務の削減」にお金を出しやすい

AI導入という言葉は広すぎる。特に小規模法人・個人事業主にとっては、AIの抽象的な可能性よりも、次のような具体課題の方が購買につながりやすい。

- スプレッドシート転記を減らしたい
- 請求書作成や入金管理を楽にしたい
- 問い合わせ対応を整理したい
- 日報・週報・議事録を自動化したい
- Google Workspace / Slack / Notion / LINE / Gmail をつなげたい
- 社内資料を検索しやすくしたい
- 小さな業務アプリを作りたい

したがって、入口の言葉は「生成AI導入」より「業務を減らす」「社内ツールを作る」「繰り返し作業を自動化する」に寄せる。

### 6. 米国の雇用市場では、若手・定型業務・汎用作業の価値が下がり始めている

米国のAI雇用影響を見ると、「AIで全体失業率が急騰している」というより、採用の選別化、若手入口の狭まり、定型ホワイトカラー業務の圧縮として表れている。

重要な観察:

- BLSはソフトウェア開発者を2024〜2034年で15.8%増と予測している
- 一方で、事務・管理支援職は2024〜2034年で約76万人減と予測されている
- Stanford Digital Economy Lab の分析では、AI曝露が高い職種の22〜25歳労働者で相対雇用が16%減少
- Indeed Hiring Lab は2026年を `low-hire, low-fire`、つまり採用も解雇も鈍いが入口が狭い市場と見ている
- UpworkではAI関連スキル需要が前年比109%増

この状況から見ると、価値が下がるのは「AIに置き換えられやすい作業者」。

価値が上がるのは以下。

- 業務課題を定義できる
- AI / 自動化 / 人間の判断を組み合わせられる
- LLM出力を検証できる
- システムとして実装できる
- 現場で運用される状態まで持っていける
- 企業のAI利用ルールやガードレールを設計できる

shogoworks の方向性は、この価値が上がる側に寄せられる。特に「AIを使った開発」「業務自動化」「研修」「運用伴走」を持っているため、単なる制作・単なる講師よりも、実装責任を持つポジションが合う。

## 避けるべき見せ方

### 1. 「AIできます」だけで売る

広すぎて比較されやすい。買い手から見ると、何を頼めばいいか分からない。

避けたい表現:

- AI活用を支援します
- ChatGPTの使い方を教えます
- 生成AI導入をサポートします

これだけだと抽象的で、購買理由が弱い。

### 2. 「Webアプリ開発できます」だけで売る

これも競合が多い。価格比較になりやすい。

避けたい表現:

- Next.js / React でWebアプリを作ります
- フロントエンドからバックエンドまで対応します
- モダンなWeb開発を行います

もちろん事実としては残すが、看板にはしない。

### 3. note / Udemy をいきなり主収益にする

noteやUdemyは有効だが、最初から主収益化を狙うと重い。

特にUdemyは作り込みコストが高い。先にライブ講座やMENTAで需要を検証し、反応が良かったテーマだけ動画化するのが安全。

### 4. 「伴走」「顧問」だけを前面に出す

伴走や顧問は継続収益として重要。ただし、初回接点では買い手が成果を想像しづらい。

避けたい表現:

- AI伴走します
- AI顧問になります
- 月額で相談できます

これらは構築後の継続商品としてはよいが、最初の看板は「何が手に入るか」が分かる方がよい。

海外の売り方を見ると、月額商品も `managed AI automation` や `AI workflow operations` のように、運用対象が明確になっている。shogoworks でも、伴走は「作った仕組みを運用・改善する月額プラン」として置く。

### 1. AI業務自動化システム構築

#### 位置づけ

入口商品かつ最重要商品。最も売りやすい。小規模法人・個人事業主に刺さりやすい。

「AI業務改善・自動化」よりも、「AI業務自動化システム構築」と言い切る。理由は、買い手が欲しいのは相談ではなく、面倒な業務が減る動く仕組みだから。

#### 対象

- 従業員5〜30名程度の小規模法人
- 個人事業主
- バックオフィス担当者
- ひとり社長
- 現場の手作業が多いチーム

#### 課題

- スプレッドシート転記が多い
- メール・Slack・LINE通知が手動
- 請求・見積・レポート作成が面倒
- 日報・議事録・問い合わせ管理が属人化している
- AIや自動化に興味はあるが、何から始めればいいか分からない

#### 提供内容

- 業務フローの棚卸し
- 自動化候補の洗い出し
- GAS / n8n / Make / Dify などの選定
- 1業務の自動化実装
- 運用マニュアル作成
- 導入後の軽微な改善

成果物例:

- 問い合わせ対応AIシステム
- 社内FAQ / ナレッジ検索ボット
- スプレッドシート転記自動化
- レポート自動生成フロー
- Slack / Gmail / Notion 通知フロー
- 営業・リード管理自動化
- AI議事録・タスク化フロー
- n8n / Make / GAS ワークフロー

#### 価格案

- AI業務自動化診断: 1.5万〜3万円 / 90分
- 1業務自動化パック: 10万〜20万円
- 業務改善スプリント: 30万〜60万円
- AI Agent / n8n 構築: 25万〜80万円
- 月額運用・改善: 10万〜30万円 / 月

### 2. AIネイティブ プロダクト / 業務アプリ開発

#### 位置づけ

中〜高単価商品。Web開発スキルを活かす中核サービス。

「Web開発」ではなく「AIを活用したプロダクト / 業務アプリ開発」として見せる。

海外リサーチでも、AI consulting / product development は自動化パックより高単価になりやすい。AI MVP / production-ready system は $5K〜$50K+ の領域として売られている。Webアプリ開発スキルを持っていることは、単なるAI相談者との差別化になる。

#### 対象

- MVPを作りたい個人・スタートアップ
- 社内ツールを作りたい小規模法人
- スプレッドシート管理に限界が来たチーム
- 業務アプリを低コストで作りたい企業
- AI機能付きの小規模プロダクトを検証したい人

#### 課題

- 作りたいアイデアはあるが、要件定義ができない
- エンジニアを雇うほどではない
- ノーコードでは限界がある
- 外注すると高額になりすぎる
- AIを使って速く検証したい

#### 提供内容

- 要件整理
- 画面設計
- 技術選定
- MVP開発
- 社内業務アプリ開発
- AI機能の組み込み
- 認証 / DB / API / デプロイ
- 運用保守

#### 技術例

- Next.js
- React
- TypeScript
- Astro
- Supabase
- Cloudflare Workers
- Vercel
- Dify
- OpenAI / Claude / Gemini API

#### 価格案

- MVP検証: 50万〜100万円
- 小規模業務アプリ: 100万〜200万円
- AI機能付きプロダクト開発: 150万〜300万円

## 営業方針

営業は「売り込み」ではなく「診断オファー」で入る。

### 基本オファー

> 御社のスプレッドシート・メール・Slack・Notion周りで、AI / GAS / n8n を使って月10時間以上削れそうな業務を一緒に洗い出します。初回90分で改善候補と優先順位表まで出します。

### 声をかける相手

- 知人の経営者
- 個人事業主
- 小規模法人
- 士業
- 教育事業者
- コミュニティ運営者
- 採用・広報・バックオフィス担当者
- AIに興味があるが、手が動いていない人

### 初回メッセージ例

> 最近、GAS / n8n / AIを使った業務改善の支援を整理していて、もし日々の転記・通知・レポート作成・問い合わせ対応などで面倒な作業があれば、90分で自動化できそうな箇所を一緒に洗い出せます。
>
> いきなり開発前提ではなく、まずは「どこを減らすと効果が大きいか」を整理する診断メニューです。

### 提案時に見るポイント

- 毎週 / 毎月繰り返している作業か
- 人の判断が必要な部分と不要な部分が分けられるか
- 既存ツールがGoogle Workspace / Slack / Notion / Excelなどか
- 自動化すると月何時間減るか
- ミス削減の価値があるか
- その後の運用者がいるか

## 90日ロードマップ

海外リサーチ反映後の90日方針は、`相談商品を作る` ではなく、`Audit -> Build -> Operate` の導線を作ること。

つまり、最初の90日で整えるべきものは以下。

- Audit: 90分AI業務自動化診断
- Build: 1業務自動化パック / AI Agent / n8n構築
- Operate: 月額AI運用・改善
- Workshop: 法人向けAI実践ワークショップ

## 判断基準

迷ったときは以下で判断する。

### 法人向けか個人向けか

両方やってよい。ただし売上の本命は法人・個人事業主。

### AI伴走か開発か

両方置いてよい。ただし「AI伴走」と「開発」を分けすぎず、以下の流れでつなげる。

> 診断 → 自動化 → 業務アプリ / MVP開発 → 継続伴走

海外リサーチ後は、より明確に以下で考える。

> Audit -> Build -> Operate

- Audit: AI業務自動化診断
- Build: AI業務自動化システム / AI Agent / 業務アプリ構築
- Operate: AI運用・改善・ワークショップ

### noteかUdemyか

先にnoteとライブ講座。Udemyは後。

海外視点でも、Udemyよりライブ講座・コホート型・法人ワークショップの方が高単価化しやすい。Udemyは需要が見えた後の再利用先。

### 低単価か高単価か

低単価は入口として使う。最終的には小案件・開発案件・月額AI運用・改善へつなげる。

### 発信か営業か

両方必要。ただし初期は営業寄り。

発信だけではリーチ獲得に時間がかかるため、診断オファーで直接接触を作る。

海外リサーチ反映後は、LinkedInをB2B接点として必ず使う。noteは信頼形成、Xは技術者向け認知、LinkedInは法人向けリードと役割を分ける。

### 相談か成果物か

初回接点では、相談より成果物を前面に出す。

弱い:

- AI相談できます
- AI伴走します
- ChatGPT活用を教えます

強い:

- 問い合わせ対応AIを構築します
- スプレッドシート転記を自動化します
- 社内FAQボットを作ります
- AI業務自動化診断で、削減できる業務と構築案を出します
- Claude Code / Codex の開発ワークフローを整備します

## 最後に

shogoworks の強みは、AIを語れることではなく、AIを使って現場の業務や開発を前に進められること。

今後の見せ方は以下に寄せる。

> AIを教える人ではなく、AI業務システムを設計・構築・運用する人。

この軸を保ったまま、入口は分散させる。

- 法人 / 個人事業主には、AI業務自動化診断
- 小さな案件には、GAS / n8n / Make / Dify 自動化
- 中〜高単価には、AIネイティブ プロダクト / 業務アプリ開発
- 継続には、AI運用・エージェント活用伴走
- 研修には、AI実践ワークショップ
- 個人向けには、MENTA / note / ライブ講座
- 教材化には、有料note / 自前講座 / Udemy

最初にやるべきことは、ホームページの見せ方を整え、AI業務自動化診断を入口商品として明確にすること。

そこから、note・MENTA・LinkedIn・X・営業・講座を同じ導線に接続していく。

海外市場からの学びは明確。

> AIを使えることではなく、AIで動く仕組みを納品し、運用できることに価値が寄っている。

したがって、shogoworks のホームページは「AI支援の一覧」ではなく、「AI業務システムの診断・構築・運用の入口」として設計する。
