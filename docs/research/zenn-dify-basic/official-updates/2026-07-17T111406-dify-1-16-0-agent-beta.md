# 2026-07-17 Dify 1.16.0

## 公式内容の日本語要約

Dify Agent（Beta）が正式に「Open Beta」入りし、全ユーザーに既定で有効化された。Linuxサンドボックス上でコード実行・シェルコマンドを行うagentで、ツール・APIを呼び出すskillシステム、ワークスペース横断でagentを管理・再利用できるagentロスター、ファイル参照対応を持つ。UIからプロンプト設定・Skillアップロード・ツールやナレッジ連携を行うビルダーに加え、対話しながらLinuxサンドボックス環境を構成しSkill/ファイルを作ってくれる「agentを作るagent」も提供される。Dify Workflowへの統合（既存Dify AgentをWorkflowノードとして使う、あるいはインラインで一時的なagentを作る）、Webアプリとしての公開にも対応する。公式ドキュメントで「信頼できる非悪意ユーザーにのみ提供すること」という警告が明記されている。

あわせて重要な変更告知が入っている。OpenAIプラグインの既定APIタイプが、非推奨化が進み新モデル系列でのサポートが限定的なChat Completions APIから、Responses APIに変更された。新しく発表されたGPT-5.6系列はChat Completions利用時に特有のパラメータ制約が多く、既存のカスタムAPIキー設定でChat Completionsのままの場合は予期しないエラーが発生し得るため、GPT-5.6以降のモデルを使う場合はAPIタイプをResponsesに切り替えるようユーザーに注意喚起している。

## できるようになったこと

- Dify Agent（Beta）が全ユーザーに既定で有効化（Linuxサンドボックス・Skillシステム・agentロスター）
- UIビルダーと「agentを作るagent」でDify Agentを構築可能
- Dify WorkflowからDify Agentを呼び出し・インライン生成できる
- OpenAIプラグインの既定API種別がChat CompletionsからResponsesに変更（GPT-5.6系列対応）

## 影響範囲

- 対象ユーザー: Dify上でagentを構築・運用する開発者全般、既存のOpenAIカスタムAPIキー設定を持つユーザー（GPT-5.6利用者は要設定確認）
- 対象プラン: 制限なし（セルフホスト・Cloud共通、Agent Appは全ユーザー既定有効）
- API / UI / 管理者機能: Agent Appビルダー、Workflow統合、OpenAIプラグイン設定（API種別）

## 教材化メモ

- Dify Agent（Beta）はサンドボックス実行系エージェントの実例としてClaude Code/Codexとの比較教材に使える。
- OpenAI Responses API既定化の告知は、既存のDify×OpenAI連携ガイドがあれば「GPT-5.6利用時はAPI種別をResponsesに」という注意点を追記する価値が高い。
