# 2026-07-09 Dify 1.16.0-rc1（pre-release）— 実験的な新エージェント「Dify Agent」導入

## 公式内容の日本語要約

Dify 1.16.0-rc1（pre-release）がリリースされた。目玉は実験的な新機能「Dify Agent」。他のトップクラスのエージェントと同様にLinuxサンドボックス内で動作するshellベースのLLMエージェントで、Skillsのアップロード・ツールやナレッジベースの接続をUIから設定できるビルダー、Dify Workflowへの組み込み（既存Dify Agentの利用または一時的なインラインAgent生成）、Web appとしての公開に対応する。公式は「現時点では全Dify Agentが同一サンドボックスで動作するため、単純な指示で他のAgentの環境やユーザーデータに干渉されうる」という重大な制約を明記し、「信頼できる悪意のないユーザーにのみ提供すること」を強く警告している。

このリリースにはDBマイグレーション・環境変数変更・docker-compose設定変更が含まれるため、アップグレード時は個別の手順に従う必要がある。ほかにMCPプロトコル2025-06-18対応（workflow-as-MCPサーバーのバージョンネゴシエーション・構造化出力）、ワークフロープロバイダー設定のキャッシュ化などの改善も含まれる。

## できるようになったこと

- UIからのDify Agentビルダー（ベースプロンプト・Skills・ファイル・ツール・ナレッジの接続設定）
- Dify Agent構築を支援する対話型エージェント（サンドボックス環境構築・パッケージインストール・Skills/ファイル作成）
- Dify WorkflowでのDify Agent利用（既存Agent呼び出し／インラインAgent生成）
- Dify AgentのWeb app公開
- MCPプロトコル2025-06-18対応（workflow-as-MCPサーバー）

## 影響範囲

- 対象ユーザー: Dify Agentの実験的機能を試すセルフホストDifyの開発者・運用者
- 対象プラン: pre-release（rc1）、DBマイグレーション・環境変数・docker-compose変更を伴う
- API / UI / 管理者機能: 新規`dify-agent`・`shellctl`コンポーネント（shellctlは組み込み認証なし、コンテナ内実行を強く推奨）

## 教材化メモ

- サンドボックス分離が未実装という明記された制約（マルチテナントでの相互干渉リスク）を必ず含めて紹介する。実験的機能であることと本番運用非推奨の位置付けを明確にする。
