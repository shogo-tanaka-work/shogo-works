# 2026-05-25 n8n@1.123.47（1.x backport stable）

## 公式内容の日本語要約

n8n の 1.x 系 stable backport である `1.123.47` が 2026-05-25 10:04 UTC に公開された。中身は依存ライブラリ起因のセキュリティ修正のみで、`vm2`、`ws`、`protobufjs` ほか合計 16 件の脆弱性を解消する PR (#30852) と、`urllib3` を 2.6.3 → 2.7.0 にマイナーアップして既知脆弱性を解消する PR (#31058) の 2 本が取り込まれている。新機能や挙動変更は含まれない。

## できるようになったこと

- 1.x 系を本番運用しているテナントが、`vm2` / `ws` / `protobufjs` などのトランジティブ依存に存在した 16 件の脆弱性を、メジャーアップなしで解消できる。
- Python ベースの内部パッケージで利用される `urllib3` の脆弱性が `2.7.0` で解消される。

## 影響範囲

- 対象ユーザー: n8n を self-host で 1.x 系 stable に固定しているユーザー、特に企業内で 2.x への昇格をまだ評価中のテナント。
- 対象プラン: OSS / self-hosted。Cloud は通常 2.x 系に追従するため直接の対象ではない。
- API / UI / 管理者機能: 依存ライブラリ更新のみで、API・ノード仕様・UI に変更なし。

## 教材化メモ

- 自社の n8n を 1.x 系 stable で運用しているチーム向けに「依存パッチ単独の backport が出たら速やかに適用する運用」の好例として使える。
- vm2 / ws / protobufjs は Node.js エコシステムで頻繁に脆弱性報告が出る依存。n8n を CVE 緩和の観点でどう運用するかの教材化に向く。
