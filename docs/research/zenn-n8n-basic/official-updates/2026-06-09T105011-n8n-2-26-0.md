# 2026-06-09 n8n 2.26.0

## 公式内容の日本語要約

n8n の stable マイナー 2.26.0 がリリースされた。本文は Bug Fixes セクションのみで新機能の追加はなく、多数の core / editor 修正とセキュリティ修正が中心。CVE 対応の依存バンプ（hono / postcss、qs / tmp / uuid 等）、MCP OAuth トークンのハードコード `aud` クレーム修正、OAuth CSRF・PKCE state のフロー別キャッシュ保存、サブワークフローのタイムアウト尊重、OIDC ログイン復旧など。並行して beta（2.26.1）も公開。

## できるようになったこと（変わったこと）

- 新機能追加なし（Bug Fixes のみのマイナー）。
- **セキュリティ修正**: Trivy nightly スキャン由来の CVE 対応で hono / postcss をパッチ、qs / tmp / uuid / @tootallnate/once をバンプ。MCP OAuth トークンのハードコード `aud` クレーム修正、OAuth CSRF・PKCE state をフロー別にキャッシュ保存。
- **core 修正（一部）**: サブワークフローが自身のタイムアウト設定とグローバル設定を尊重、親ワークフローがサブワークフロー完了待ちでスタックする問題の修正、OIDC（`expires_in: 0` を返す IdP）ログイン復旧、PDF Data Loader の DOMMatrix polyfill。
- **editor 修正（一部）**: SSO ロールマッピング規則の全件ロード、cURL インポート時の非文字列 JSON 値保持、データリダクションポリシー周りの整備。

## 影響範囲

- 対象ユーザー: n8n セルフホスト / クラウド利用者（特に OAuth / MCP / SSO / サブワークフローを使う構成）
- 対象プラン: 全般。channel: stable（2.26.0）、beta は 2.26.1
- API / UI / 管理者機能: 認証（OAuth / OIDC / MCP）、ワークフロー実行、editor、依存セキュリティ

## 教材化メモ

新機能はなく教材化の中心ではないが、セキュリティ修正（CVE バンプ・MCP OAuth `aud`・OAuth CSRF/PKCE）とサブワークフロー タイムアウト尊重は運用メモに残す価値がある。AIニュース速報化は見送り、日次サマリーと本詳細に記録。OAuth / SSO / MCP を使う本番環境はアップデートを推奨、と注記。
