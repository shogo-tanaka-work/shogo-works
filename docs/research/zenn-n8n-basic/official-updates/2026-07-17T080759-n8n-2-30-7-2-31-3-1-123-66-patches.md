# 2026-07-17 n8n 2.30.7 / 2.31.3 / 1.123.66 パッチリリース

## 公式内容の日本語要約

n8nのstableライン3系統（2.30系・2.31系・1.x系）で同日にパッチリリースが出た。共通の修正はAWS SESリクエストの署名でサービス名を正しく`ses`にする不具合修正（2.30.7・2.31.3双方に含まれる）。2.30.7ではコミュニティパッケージをprovenance-attestedなソースに対して検証（lint）する機能修正も入った。2.31.3ではエンドユーザーの認証情報検証・再評価・identity-hook認識の修正も追加されている。1.123.66（1.x系バックポート）では、Code Nodeで許可されているはずの外部モジュールが動作しない不具合を修正した。

## できるようになったこと

- AWS SES連携のリクエスト署名不具合を修正（誤ったサービス名によるSES呼び出し失敗を解消）
- コミュニティパッケージのprovenance検証を強化
- エンドユーザー認証情報の検証・再評価・identity-hook認識を修正
- （1.x系）Code Nodeで許可済み外部モジュールが動作しない不具合を修正

## 影響範囲

- 対象ユーザー: n8nでAmazon SESノードを使うユーザー、コミュニティパッケージ利用者、Code Nodeで外部モジュールを許可設定しているユーザー（特に1.x系）
- 対象プラン: 全プラン（OSS/Cloud共通のコア修正）
- API / UI / 管理者機能: SES連携、コミュニティパッケージ検証、認証情報管理、Code Node

## 教材化メモ

パッチ規模の修正のため単独記事化の優先度は低いが、SES連携やCode Nodeの外部モジュール許可を使っている既存ワークフローがあれば注意点として一言添える価値がある。
