# 2026-07-15 n8n 2.29.11 / 2.30.5 / 2.31.1（stable系列3チャンネル同時パッチ）

## 公式内容の日本語要約

3つのstableチャンネル（2.29.x、2.30.x、2.31.x）に同時展開されたバグ修正パッチ。最もセキュリティ上重要なのは2.31.1の「動的認証情報トークンをユーザーIDに紐付け（bind dynamic credential tokens to a user ID）」（#34134）で、動的に発行される認証情報トークンが特定ユーザーに正しく紐付けられていなかった不具合を修正した。あわせて「AI assistantのワークフローテスト実行を手動実行としてカウント」（統計の正確性向上）、「AIツールバリアント生成時のノードエイリアス保持」、「AI assistantのクォータ超過エラー状態を明示化」などのAI Assistant関連の細かな修正が2.30.5・2.31.1双方に含まれる。

2.29.11では「実行可視性とシェアリングライセンス機能の分離」「Webhook実行のPromise早期エラー時のsettle処理」「Notionノードの旧バージョンでのASCII簡略化プロパティキー復元」が修正された。いずれもバグ修正中心で新機能追加はない。

## できるようになったこと

- 動的認証情報トークンがユーザーIDに正しく紐付けられるように（セキュリティ修正）
- AI assistantのワークフローテスト実行が統計上「手動実行」として正しくカウントされる
- AI assistantのクォータ超過時にエラー状態が明示表示される

## 影響範囲

- 対象ユーザー: 動的認証情報（MCP OAuth等）を利用するn8nインスタンス管理者、AI Assistant利用者
- 対象プラン: stable系列（2.29.x / 2.30.x / 2.31.x）全チャンネル
- API / UI / 管理者機能: 動的認証情報トークン発行、AI Assistant統計・クォータ表示

## 教材化メモ

- 動的認証情報トークンのユーザー紐付け修正はセキュリティ関連のため、MCP OAuth等の動的認証情報を使う構成では該当バージョンへの更新を案内する価値がある。教材上は「セキュリティパッチとして早めに追従すべき例」として軽く触れる程度が適切（詳細な脆弱性内容は公式非開示）。
