# 2026-07-14 n8n 2.31.0（stable）

## 公式内容の日本語要約

管理者・セキュリティ関連の新機能が中心のマイナーリリース。プライベート認証情報（private credentials）がデフォルトで有効化され、エンドユーザーの認証情報作成権限がowner/admin/project adminロールに制限されるようになった（#34064）。「Security & policies」設定にworkflow reviewsトグルが追加され、公開APIに セキュリティポリシー設定用のエンドポイントが追加された（#33922）。トークン交換プロビジョニングでカスタムグローバルロールをサポート。

AI関連では、AWS Bedrock Chat Modelノードの推論パラメータが拡張され、AWS IAM/CognitoノードにAssume Role認証が追加された。エディタにはAI活動ステップコンポーネント、eval collectionsの一覧・セットアップウィザード・比較ビュー（スコアチャート付き）、Group/Ungroupのコンテキストメニュー操作、n8n Connectセクションがノードクリエーターパネルに追加されるなど、AI Assistant / eval機能まわりの土台が拡充された。パッケージインポートで名前+タイプ・タイプのみの認証情報マッチングモードも追加。

バグ修正も多数含まれ、Postgresプール解放処理、実行可視性とシェアリングライセンスの分離、ワークフロー検証時のノードエラー報告、MCP Server Triggerのキューモード永続化などが修正された。

## できるようになったこと

- プライベート認証情報がデフォルト有効化、認証情報作成をロールで制限可能に
- Security & policies設定にworkflow reviewsトグル、セキュリティポリシー公開APIエンドポイント追加
- AWS Bedrock Chat Modelノードの推論パラメータ拡張、AWS IAM/CognitoにAssume Role認証追加
- eval collectionsの一覧・セットアップウィザード・比較ビューを追加（評価機能の土台整備）
- Group/Ungroupコンテキストメニュー操作、n8n Connectセクション追加

## 影響範囲

- 対象ユーザー: n8nインスタンス管理者、AWS Bedrock/IAM連携を使うワークフロー開発者、AI Assistant/eval機能を使う開発者
- 対象プラン: 制限なし（Security & policies・公開APIはエンタープライズ/管理者向け機能が中心）
- API / UI / 管理者機能: Security & policies設定、公開APIセキュリティポリシーエンドポイント、認証情報作成権限、AWS Bedrock/IAMノード、eval collections

## 教材化メモ

- プライベート認証情報のデフォルト有効化は、既存ワークフローの認証情報共有設定に影響し得るため、アップグレード時の注意点として案内価値がある。
- eval collections機能群（一覧・セットアップウィザード・比較ビュー）は今後のAI Assistant評価機能の布石であり、継続ウォッチ候補。
