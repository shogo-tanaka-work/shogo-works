# 2026-06-23 n8n 2.28.0

## 公式内容の日本語要約

n8n 2.28.0 がリリースされた。新機能はなく、バグ修正・セキュリティパッチ（protobufjs・js-yaml・form-data 等の脆弱性パッチ）・コア安定性改善が中心。

## できるようになったこと

- **主な修正:**
  - API: 公開済みワークフロー削除時に 409 を返すよう修正
  - Bitbucket Trigger: username ではなく account_id で認証情報検証
  - Compression Node: アーカイブメンバーのみ解凍するよう修正
  - core: CSV アップロードのデータテーブルサイズバジェット計算を修正
  - core: COOP ヘッダーのデフォルト値変更
  - core: ワークフローフォルダ配置を validatedParentFolderId で制御
  - core: AI ツール説明の式評価を修正
  - core: エラーワークフロー実行を課金対象実行カウントから除外
  - core: MCP ツールのエラーパスのスキーマミスマッチを修正
- **セキュリティ:** form-data / tmp / protobufjs / js-yaml / @opentelemetry/core を脆弱性パッチ済みバージョンへ更新

## 影響範囲

- 対象ユーザー: n8n セルフホスト・クラウド利用者
- 対象プラン: 全プラン
- API / UI: コア・各ノード・MCP ツール

## 教材化メモ

新機能なし。セキュリティパッチの観点から「依存ライブラリ定期更新」の重要性として言及できる程度。
