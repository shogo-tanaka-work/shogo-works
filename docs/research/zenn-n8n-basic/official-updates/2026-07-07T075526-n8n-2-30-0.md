# 2026-07-07 n8n 2.30.0（pre-release）— Instance AI課金分離・Kafka mTLS・Microsoft Service Principal認証など大型機能集約

## 公式内容の日本語要約

n8n 2.30.0（pre-release、2.29.0以降の変更を集約したマイナーバージョン）がリリースされた。Instance AIの課金を専用クレジットプールへ分離、コミュニティノード型定義のAI Assistant対応、KafkaノードへのmTLS（クライアント証明書）認証追加、Microsoft Excel/Teams/To DoノードへのService Principal（アプリ専用）認証追加、プライベート資格情報の共有再有効化、データテーブルノードの機能改善、AI Assistantの推論ブロックを実行タイムラインで別表示、`N8N_WEBHOOK_URL`統一環境変数の追加など、機能追加・バグ修正・パフォーマンス改善を広範に含む。

## できるようになったこと

- Instance AIの課金を専用クレジットプールへ分離（`route Instance AI billing to a dedicated credit pool`）
- コミュニティノードの型定義をAI Assistantが認識
- KafkaノードでmTLS（クライアント証明書）認証
- Microsoft Excel/Teams/To DoノードでService Principal（アプリ専用）認証
- プライベート資格情報の共有を再有効化
- データテーブルノードの機能改善、CSVアップロードの拡張子検証
- AI Assistantの推論（reasoning）を実行タイムラインで独立ブロック表示
- 本番・テストWebhook URLを統一する`N8N_WEBHOOK_URL`の追加
- ワークフロー統計集計の負荷軽減（ホットロー競合の回避）

## 影響範囲

- 対象ユーザー: Instance AI利用者、Microsoft 365/Kafka連携ユーザー、セルフホスト管理者
- 対象プラン: 2.30.x系（pre-release、今後stableへ昇格予定）
- API / UI / 管理者機能: Instance AI課金、認証情報管理、Kafka/Microsoft各種ノード、Public API（プロジェクトエクスポート、評価テスト実行の読み取り）

## 教材化メモ

- Instance AI課金の専用クレジットプール化とMicrosoft各種ノードのService Principal認証は、企業導入シナリオ（コスト管理・アプリ専用認証）の教材にそのまま使える。マイナーバージョンへの機能集約リリースのため、個別PRは6月中〜7月上旬の複数日にまたがって積み上げられたもの。
