# 2026-07-16 n8n 2.30.6 / 2.31.2 パッチリリース

## 公式内容の日本語要約

n8nのstableライン2系統（2.30系・2.31系）で同日にパッチリリースが出た。共通の修正は「MCPワークフロー実行にexecution modeを必須化する」バグ修正（コア）。2.31.2ではこれに加えてUI上の全ドロップダウンの幅が崩れていた不具合も修正された。

## できるようになったこと

- MCP経由のワークフロー実行時にexecution modeが必須化され、意図しない実行モードでの動作を防止（セキュリティ/堅牢性寄りの修正）
- （2.31.2のみ）UIドロップダウンの幅崩れを修正

## 影響範囲

- 対象ユーザー: n8nをMCPサーバー/クライアントとして利用しているユーザー全般
- 対象プラン: 全プラン（OSS/Cloud共通のコア修正）
- API / UI / 管理者機能: コア実行エンジン、UI（ドロップダウン）

## 教材化メモ

パッチ規模の修正だが、「MCPワークフロー実行のexecution mode必須化」は運用中のMCP連携ワークフローに影響し得るため、既存のn8n運用ガイドがあれば注意点として一言添える価値がある。単独記事化の優先度は低い。
