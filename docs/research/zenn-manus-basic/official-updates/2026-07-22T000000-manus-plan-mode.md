# 2026-07-22 Manus Plan Mode

## 公式内容の日本語要約

Manusが「Plan Mode」を発表した。「AIのミスの多くは、アプローチが事前にレビューされていないことが原因」という課題意識のもと、実行前にManusがフィージビリティチェックを行い、目的・手順・制約をまとめたMarkdown形式の計画書を提示する。ユーザーは計画を確認・編集・修正依頼でき、承認するまでManusは実行を開始しない。Web版は「/」、モバイル版は「+」から起動でき、進行中のプロジェクトの途中フェーズに対しても利用可能。デフォルトでは無効になっており、通常タスクの速度を落とさないよう任意発動の機能として全ユーザーに提供開始。

## できるようになったこと

- タスク実行前にManusが計画書（目的・手順・制約）を生成し、承認まで実行を保留
- 計画書はユーザーが直接編集、またはManusに修正を依頼可能
- 進行中タスクの途中フェーズでも計画レビューを追加可能
- Web/モバイル双方で本日から全ユーザーに提供（デフォルト無効、任意発動）

## 影響範囲

- 対象ユーザー: Manus全ユーザー（Web/モバイル）
- 対象プラン: プラン制限の言及なし（全ユーザー対象と明記）
- API / UI / 管理者機能: チャットUI内のコマンド起動（「/」またはモバイルの「+」）

## 教材化メモ

「実行前に計画を確認・承認するステップを挟む」という設計は、AIエージェント活用の教材で汎用的に使える論点（暴走防止・レビューポイントの明示）。Claude CodeのPlan Modeとの比較で紹介すると理解が深まりやすい。
