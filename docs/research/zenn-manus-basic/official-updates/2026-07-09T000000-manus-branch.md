# 2026-07-09 Manus Branch

## 公式内容の日本語要約

Manusが新機能「Branch」を発表した。実際の作業は一本道ではなく非線形に進むという課題に対応するため、既存の会話の任意のメッセージから並列セッションを分岐できるようにした機能。分岐先の新セッションは、分岐時点までの元の指示・ファイル・会話履歴全体を引き継ぐため、単一スレッドが複数の新しい方向性の作業で希薄化する問題を避けられる。操作はメッセージにホバーしてBranchアイコンをクリックするだけで、元の会話はそのまま維持される。同じ会議メモから複数の成果物を生成したり、プロジェクトの生きたログを維持しながら特定の出力だけ別ブランチで探索する、といった使い方を想定している。

## できるようになったこと

- 会話中の任意のメッセージを起点に、独立した並列セッション（ブランチ）を作成できる
- ブランチは分岐時点までの指示・ファイル・会話履歴を完全に継承する
- 元の会話スレッドを維持したまま、複数方向の作業を並行して進められる

## 影響範囲

- 対象ユーザー: Manus利用者全般（プラン別の制限は本文に記載なし）
- 対象プラン: 記載なし
- API / UI / 管理者機能: チャットUI上の操作（メッセージホバー→Branchアイコン）

## 教材化メモ

「1つの会話に複数タスクを詰め込むとコンテキストが薄まる」という、AIエージェント活用でよくある失敗パターンへの製品側の解決策。Claude CodeやChatGPTの「会話の分岐・フォーク」的な機能と比較する教材ネタになりうる。
