# 2026-07-13 Manus Auto-Publish

## 公式内容の日本語要約

Manusが新機能「Auto-Publish」を発表した。従来はサイト公開のたびにメニューを開いて設定を確認し、公開ボタンを押すという手動操作が必要だったが、Auto-Publishをオンにすると、ビルドが成功するたびに確認操作なしで自動的に公開URLへデプロイされるようになる。複数の変更をキューに入れて順次処理させることも可能。トグルはデフォルトでオフになっており、ユーザーが明示的に有効化する必要がある。失敗中/進行中のビルドは公開サイトに反映されず、クライアント向けURLを保護する設計。Web/iOS/Androidの全プラットフォームで即時展開。

## できるようになったこと

- ビルド成功のたびに手動確認なしで自動的に公開URLへデプロイできるようになった
- 複数の変更指示をチャットに投げてキュー処理させ、Auto-Publishと組み合わせて「まとめて依頼→離席→完成した公開済みサイトに戻る」というワークフローが可能になった
- 失敗/進行中のビルドは公開サイトに影響しない安全設計

## 影響範囲

- 対象ユーザー: Manus利用者全般（方向性が固まった開発を素早く反復したいユーザー向け）
- 対象プラン: 記載なし（Web/iOS/Androidで即時利用可能）
- API / UI / 管理者機能: サイト公開設定のトグルUI

## 教材化メモ

「方向性が決まったら確認プロセスを省いて自動デプロイ」という、AIエージェント活用の後半フェーズ（検証済みタスクの高速反復）向け機能。CI/CDの自動デプロイ概念と絡めた教材ネタになりうる。前回（2026-07-09）の「Branch」（並列セッション分岐）と対になる、Manusのワークフロー高速化路線の一環。
