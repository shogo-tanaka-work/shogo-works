# 2026-05-18 Manus Scheduled Tasks 2.0

## 公式内容の日本語要約

Manus がスケジュール実行機能の刷新版「Scheduled Tasks 2.0」を発表した。従来は固定スケジュール（毎日/毎週/指定時刻）で都度新規タスクを生成していたが、2.0 では「どこで実行され、どのコンテキストを引き継ぎ、どの成果物を更新し続けるか」を理解する文脈認識型になった。

主な強化点:

- **タスク継続**: 繰り返し実行が新規タスクを生成せず同一タスク内で継続。既存の指示・ファイル・会話履歴を保持。
- **Web アプリ組み込み**: Manus で構築した Web アプリにデータ更新やレポート生成などのスケジュール動作を組み込める（手動トリガー不要）。
- **可視化3ビュー**: スケジュール作業と連動実行を示すサイドパネル、タイミングを把握するスケジュールビュー、繰り返し作業を俯瞰するカレンダービュー。
- **実行制御**: 同一タスク継続か別タスク生成かの選択、信頼ワークフローの承認スキップ、接続アプリをデータソース化、実行環境の選択。

## できるようになったこと

- 定期実行を同一タスク内で継続し、指示・ファイル・会話履歴を保持できる。
- Manus 製 Web アプリにスケジュール動作（データ更新・レポート生成）を組み込める。
- サイドパネル/スケジュール/カレンダーの3ビューで定期作業を可視化できる。
- 同一タスク継続/別タスク生成の選択、承認スキップ、接続アプリのデータソース化、実行環境選択ができる。

## 影響範囲

- 対象ユーザー: Manus で定期業務（日次スタンドアップ、週次レポート、ステータス確認、ダッシュボード更新）を運用する利用者・チーム
- 対象プラン: Manus 利用環境全般（公式記載の範囲）
- API / UI / 管理者機能: スケジュール機能 / Web アプリ組み込み / スケジュール・カレンダービュー

## 教材化メモ

Manus 活用教材では「一過性の生成」から「文脈を引き継ぐ定期実行」への発展として扱える。教材化するなら、定期レポート/ダッシュボード更新を題材に「タスク継続 vs 新規生成の使い分け」「承認スキップの安全設計」を実務章に追加候補。2.0 固有の UI 詳細は持ち込まず、設計論として反映する。
