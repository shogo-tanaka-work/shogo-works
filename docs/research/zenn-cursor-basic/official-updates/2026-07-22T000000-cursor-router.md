# 2026-07-22 Cursor Router

## 公式内容の日本語要約

Cursorが「Cursor Router」を発表した。60万件超の実リクエストで訓練した分類器が、タスクの複雑さ・コンテキスト・ドメインごとに最適なモデルへ自動でルーティングする仕組みで、Auto modeの内部に組み込まれる。Intelligence（最高品質重視）、Balance（品質とコストのバランス、日常利用向け）、Cost（コスト最適化）の3モードを切り替え可能。公式発表では、Auto Intelligenceモードがユーザー満足度でOpus 4.8を上回りつつ約60%低コスト、Auto Balanceも同程度の満足度でコストを抑えられると主張している。Team/Enterpriseプランでデスクトップ・Web・iOS・CLI・SDK横断で提供開始。

## できるようになったこと

- Auto modeが「Cursor Router」という専用の分類器ベースルーティングに刷新
- Intelligence / Balance / Cost の3モードで品質とコストのトレードオフをチーム単位で調整可能
- Team/Enterprise向けに管理者コントロール付きで利用開始

## 影響範囲

- 対象ユーザー: Cursor Team / Enterprise プランユーザー
- 対象プラン: Team、Enterprise（デスクトップ/Web/iOS/CLI/SDK）
- API / UI / 管理者機能: Auto modeの内部ルーティング、管理者向けモード制御

## 教材化メモ

「モデル選択の自動化」はコスト最適化文脈で教材価値が高い。Auto modeが単純なフォールバックではなく分類器ベースのルーティングに刷新された点、Intelligence/Balance/Costの使い分け指針を短く紹介する形が良い。
