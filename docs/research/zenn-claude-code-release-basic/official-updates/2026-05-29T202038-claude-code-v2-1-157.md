# 2026-05-29 Claude Code v2.1.157

## 公式内容の日本語要約

Anthropic は 2026-05-29 20:20 UTC（JST 2026-05-30 05:20）に Claude Code **v2.1.157** を公開した。前日の v2.1.154 が「Opus 4.8 既定化＋dynamic workflows」という当窓最大級の機能リリースだったのに対し、v2.1.157 は **プラグイン運用の摩擦削減**・**worktree 運用の自動化**・**telemetry 観測性の強化**・**多数のバグ修正** をまとめた整理リリース。

### プラグイン関連

- **`.claude/skills` 配下のプラグインがマーケットプレイス不要で自動ロード** されるようになった。これまではローカルプラグインも一旦 `/plugin` で明示する必要があった。
- **`claude plugin init <name>`** で `.claude/skills` 配下に新規プラグインのスキャフォールド生成。
- **`/plugin` 引数のオートコンプリート** 追加：サブコマンド名、インストール済みプラグイン名、既知マーケットプレイス由来のプラグイン名を補完。

### `claude agents` / Worktree

- `settings.json` の **`agent` フィールド** が dispatch されたセッションで尊重される。`--agent <name>` で上書き可。
- **`EnterWorktree` がセッション中断なしに切り替え可能** に（Claude 管理 worktree 間の遷移）。
- Claude 管理 worktree が **エージェント終了時に unlock 状態で残る**。`git worktree remove`/`prune` でクリーンアップ可能。
- `.claude/worktrees/` 配下の背景エージェント worktree が **30 日 retention sweep 後に orphan 化する問題** を修正。

### Telemetry / 観測性

- `tool_decision` telemetry イベントに **`tool_parameters`**（bash コマンド、MCP/skill 名）が含まれるようになった。`OTEL_LOG_TOOL_DETAILS=1` 設定時のみ。

### バグ修正（抜粋）

- **ゼロバイト / 破損画像** をペースト・MCP・ダイアログから添付するとリクエストがクラッシュしていた問題 → text placeholder に変換するよう修正。
- デスクトップアプリ・IDE 拡張・SDK 使用時に **auto / bypass-permissions モードでもサンドボックスのネットワーク許可プロンプト** が出ていた問題を修正。
- `claude agents` 完了セッションが idle subagent の parked / leaked backgrounded shell によって retire されない問題を修正。
- `claude agents` の "opening..." 中の Esc キャンセル不能問題を修正。
- 背景セッションが sleep/wake 後に再 attach した際、モデルに正しい日付を伝えていなかった問題を修正。
- tmux + `set-clipboard on` の `claude agents` で copy-on-select がシステムクリップボードに届かない問題（2.1.153 で混入したリグレッション）を修正。
- `--resume` が前回プロセス終了時に走っていた背景 subagent を報告していなかった問題を修正。
- `--resume` セッションピッカーが fullscreen 終了後に端末上に残骸を残す問題を修正。
- `--worktree` / `--worktree --tmux` が linked worktree ではなくリポジトリ root に戻る問題を修正。
- `/model` ピッカーで最新モデル選択中に "Newer version available" の誤表示が出る問題を修正。pinned-model 行はモデル ID ではなく description を表示。
- fullscreen モードの in-progress メッセージにマークダウン記号が literal 表示される問題を修正。
- 起動時の managed-settings security ダイアログ承認後にターミナルがフリーズする問題を修正。
- スクロールバック内に稀に重複行が現れる問題を修正。
- VS Code / Cursor / Windsurf 統合ターミナルで右クリックペーストがクリップボード重複する問題を修正。
- WSL: image paste（`alt+v`）、Windows 11 のスクリーンショットペースト、Windows Explorer からのドラッグ追加対応。

### パフォーマンス / UX

- 長時間・再開会話のメッセージレンダリング再計算の冗長を排し **パフォーマンス改善**。
- `/terminal-setup` が VS Code/Cursor/Windsurf 統合ターミナルで **GPU acceleration を無効化** し描画破損を防止。
- Feature of the Week のクレジット claim ステータスを **status area の notification** として表示（プロンプト上の行ではなくなった）。
- `claude agents` の dispatch 入力のスラッシュコマンド補完が **部分一致** に対応。
- "bash commands will be sandboxed" 起動バナー削除（`/status` とブロック時に表示維持）。
- "/ide for ..." 起動ヒント toast 削除。
- [IDE] 背景 subagent 実行中の Stop クリックが実際に停止しなかった問題を修正。
- [VSCode] Opus 4.8 で fast mode インジケータが出ない問題を修正。
- ワークフロー trigger キーワード直後の backspace で「ワークフロー要求のキャンセル」が（文字削除ではなく `alt+w` 同等として）扱われるよう調整。
- `/config` に **"Workflow keyword trigger"** 設定追加。プロンプト中の "workflow" が dynamic workflow を起動する挙動を停止可能。

## できるようになったこと

- ローカルプラグインを `.claude/skills` に置くだけで自動ロード・スキャフォールドコマンドで雛形生成・`/plugin` 補完で運用負荷が減る。
- `EnterWorktree` でセッションを切らずに worktree 間移動。終了時は unlock 状態のため自動クリーンアップしやすい。
- `tool_decision` イベントに bash コマンドや MCP/skill 名を含められるため、Enterprise の監査・利用分析の解像度が上がる。

## 影響範囲

- 対象プラン: 全プラン（Plugins 自動ロード、Worktree 改善、画像クラッシュ修正は全員）
