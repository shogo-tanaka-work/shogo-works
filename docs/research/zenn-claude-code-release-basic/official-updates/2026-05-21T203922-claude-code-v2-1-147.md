# 2026-05-21 Claude Code v2.1.147 / v2.1.148

## 公式内容の日本語要約

Anthropic は 2026-05-21 に Claude Code **v2.1.147** を、続けて 2026-05-22 に **v2.1.148** を公開した。v2.1.147 は前日 v2.1.146 の「`/simplify` → `/code-review` 改名」を引き継ぎ、`/code-review` を実体のあるコードレビューコマンドへ仕上げた更新。v2.1.148 は v2.1.147 で混入した Bash ツールのリグレッションを潰すホットフィックス。

## できるようになったこと

- `/code-review` が、選択した effort level（例: `/code-review high`）で **correctness バグ（正しさの不具合）を報告**するコマンドになった。`--comment` を付けると、検出結果を **GitHub PR のインラインコメント**として投稿できる。旧来の「整理・修正（cleanup-and-fix）」挙動は削除された。
- `claude agents` でピン留め（`Ctrl+T`）したバックグラウンドセッションが、アイドル時も生存し続け、Claude Code 更新時はその場で再起動して適用され、メモリ逼迫時は非ピン留めセッションより後に解放されるようになった。
- プロンプト履歴が連続した重複エントリを記録しなくなった（↑キーで呼び出して再送しても複製が増えない）。
- 自動アップデータが一時的なネットワーク障害をリトライし、失敗時に具体的なエラー分類と OS エラーコードを表示するようになった。

## 主なバグ修正（v2.1.147）

- `!` コマンド出力中の `&` が `&amp;` と表示され、`gcloud auth login` 等の URL のコピペが壊れる問題を修正。
- headless / SDK モードで未知のスラッシュコマンドが無反応だったのを、エラー表示するよう修正。
- プラグインエージェントが `tools:` frontmatter で複数の `Agent(...)` 型を宣言すると最後の1件以外が落ちる問題を修正。
- フック `if` 条件（`PowerShell(git push*)` のような形）が一致しなかった問題を修正。
- 小さい端末で `/help` のタブヘッダーが壊れる、`/effort` のスライダー初期位置がずれる、貼り付けテキストが `[Pasted text #N]` プレースホルダで届く、`/plugin` のコンポーネント数が二重計上される、などを修正。
- `CLAUDE_CODE_SUBAGENT_MODEL` がエージェントチームのteammateプロセスへ伝播しない問題を修正。

## v2.1.148

- v2.1.147 で混入した、一部ユーザーで Bash ツールが毎回 exit code 127 を返すリグレッションを修正。

## 影響範囲

- 対象ユーザー: Claude Code 利用者全般。特にコードレビュー運用チーム、`claude agents` でバックグラウンドエージェントを常用するユーザー、Windows / プラグイン / フック利用者。
- 対象プラン: プラン制限の記載なし（CLI 更新）。
- API / UI / 管理者機能: CLI 機能。`--comment` 利用は GitHub 連携が前提。

## 教材化メモ

`/code-review` は v2.1.146 の「改名」から v2.1.147 で「correctness バグ報告 + effort level + `--comment`」へ実体化した。new-commands-2026 教材の `/code-review` 記述を「改名」から「コードレビューコマンド」へ更新し、effort level と `--comment` の使い分け例を1段追加する候補。ピン留めセッションのライフサイクル（常駐・その場再起動・メモリ逼迫時の解放順）は agent-teams / background 教材の恒久情報。
