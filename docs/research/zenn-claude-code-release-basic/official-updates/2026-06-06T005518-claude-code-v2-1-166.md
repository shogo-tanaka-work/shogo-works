# 2026-06-06 Claude Code v2.1.166

## 公式内容の日本語要約

Claude Code v2.1.166 がリリースされた。プライマリモデルが過負荷・利用不可のときに最大3つの代替モデルを順番に試す `fallbackModel` 設定の追加が目玉。あわせて deny ルールでの glob パターン対応、他セッションからリレーされるメッセージのセキュリティ強化、thinking の明示無効化挙動の整理、多数の不具合修正が入った。直後の v2.1.165 / v2.1.167 / v2.1.168 はいずれも「バグ修正・信頼性改善」のみ。

## できるようになったこと

- **`fallbackModel` 設定**: プライマリモデルが overloaded / 利用不可のときに順番に試す代替モデルを最大3つまで設定可能。`--fallback-model` も対話セッションに適用されるようになった。
- **deny ルールの glob 対応**: ツール名位置で glob パターンを使用可能（`"*"` で全ツール拒否）。allow ルールは非 MCP の glob を拒否し、deny ルール中の未知ツール名は起動時に警告。
- **クロスセッションメッセージの堅牢化**: 他の Claude セッションから `SendMessage` 経由でリレーされたメッセージはユーザー権限を引き継がなくなった。受信側はリレーされた権限要求を拒否し、auto モードはこれをブロック。
- **thinking 無効化の整理**: `MAX_THINKING_TOKENS=0`、`--thinking disabled`、モデル別 thinking トグルで、Claude API 経由でデフォルト思考するモデルの thinking を無効化できる（サードパーティプロバイダーは挙動据え置き）。
- **fallback リトライ**: API が予期しない non-retryable エラーを返したとき、fallback モデルで1回リトライ。auth / rate-limit / request-size / transport エラーは即時に表面化。
- `claude update` がダウンロード前に対象バージョンを表示するようになった。
- `claude agents`: 一覧に URL を入力すると、最初のプロンプトにその URL を含むセッションへ絞り込み。

## 影響範囲

- 対象ユーザー: Claude Code 利用者全般
- 対象プラン: 全プラン（managed settings 関連は組織管理者向け）
- API / UI / 管理者機能: CLI 設定（`fallbackModel`）、権限ルール（deny glob）、セキュリティ（クロスセッション権限）、IDE 連携の不具合修正（JetBrains ターミナルのちらつき等）

## 主な修正

- JetBrains IDE（IntelliJ / PyCharm / WebStorm 等 2026.1+）のターミナルちらつきを synchronized output で解消。
- Kitty キーボードプロトコル（WezTerm / Ghostty / kitty）で Shift+非ASCII（例 Shift+ä → Ä）が落ちる問題。
- Windows で killed プロセスの子が出力パイプを掴んだ際に PowerShell コマンド検証が時間超過してハングする問題。
- macOS で daemon 死亡後に `claude --bg-pty-host` が 100% CPU で残る問題。
- 処理不能画像送信時の「image could not be processed」エラーと余分なトークン消費。
- managed settings の `allowedMcpServers`/`deniedMcpServers` 述語が `${VAR}` 参照時にマッチしない問題。

## 教材化メモ

「モデル設定」章に `fallbackModel`（最大3モデル、対話セッション対応）を追加し、過負荷時の自動フェイルオーバー運用例を示す。「権限・管理設定」章に deny ルールの glob 対応（`"*"` 全拒否、未知ツール名の起動時警告）を追記。v2.1.163 で扱ったクロスセッション・managed settings の続きとして、リレーメッセージが権限を引き継がない仕様を「マルチセッション運用時のセキュリティ前提」として明示する。
