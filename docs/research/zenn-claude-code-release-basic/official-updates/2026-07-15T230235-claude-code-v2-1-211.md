# 2026-07-15 Claude Code v2.1.211

## 公式内容の日本語要約

セキュリティ強化が中心のリリース。チャットチャンネルに中継される権限プレビューが、双方向オーバーライド文字・ゼロ幅文字・見た目が引用符に似た文字を無害化していなかったため、ツール入力が承認メッセージを視覚的に改ざんできてしまう不具合を修正した。またauto modeがサンドボックス外Bashに対するPreToolUseフックの`ask`判定を上書きしてしまう不具合、権限プレビューの入力検証で複数ハードリンクを持つファイルを拒否しないなど関連するセキュリティ修正も含む。

新機能として`--forward-subagent-text`フラグと環境変数`CLAUDE_CODE_FORWARD_SUBAGENT_TEXT`が追加され、stream-json出力にsubagentのテキストとthinkingを含められるようになった。Bedrock・Vertex・Mantle・Foundryで末尾のシステムコンテキストブロックが毎リクエスト新規入力トークンとして課金されてしまうプロンプトキャッシュの回帰バグを修正し、コスト面でも重要な修正が入っている。

その他、スリープ復帰後に複数セッションが同時ログアウトする不具合、プラグインMCPサーバーがアイドルWebセッション復帰後に再接続しない不具合、`/clear`後にセッションコストカウンターがリセットされない不具合など多数のバグ修正を含む。permission ruleが「always allow」保存先がgitリポジトリルートに変更され、worktree間で承認が引き継がれるようになった。

## できるようになったこと

- `--forward-subagent-text`でsubagentのテキスト・thinkingをstream-json出力に含められる
- 権限プレビューの視覚偽装対策（双方向オーバーライド・ゼロ幅・偽装引用符文字の無害化）
- Bedrock/Vertex/Mantle/Foundryのプロンプトキャッシュ課金バグを修正（コスト影響あり）
- 「always allow」権限ルールがリポジトリルート保存になり、worktree間で共有

## 影響範囲

- 対象ユーザー: チャットチャンネル連携でClaude Codeを使う運用者、Bedrock/Vertex/Mantle/Foundry利用者、subagentのstream-json出力を使う開発者
- 対象プラン: 制限なし（クラウド連携利用者はキャッシュ課金バグ修正でコストが変わる可能性）
- API / UI / 管理者機能: 権限プレビュー、stream-json出力、プロンプトキャッシュ課金、権限ルール保存先

## 教材化メモ

- 権限プレビューの視覚偽装対策（プロンプトインジェクション/ソーシャルエンジニアリング系）はセキュリティ教材のネタとして紹介価値が高い。
- プロンプトキャッシュ課金バグ修正は、Bedrock/Vertex経由で運用しコストを注視している企業ユーザー向けに一言案内する価値がある。
