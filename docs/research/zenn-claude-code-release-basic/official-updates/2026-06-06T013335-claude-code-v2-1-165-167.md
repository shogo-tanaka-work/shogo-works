# 2026-06-06 Claude Code v2.1.165〜v2.1.167

## 公式内容の日本語要約

直近24時間で Claude Code が v2.1.165 / v2.1.166 / v2.1.167 と立て続けにリリースされた。v2.1.165 と v2.1.167 はバグ修正・信頼性改善が中心。実質的な機能追加は v2.1.166 に集約されており、`fallbackModel` 設定、deny ルールでの glob パターン、クロスセッションメッセージのセキュリティ強化、thinking の無効化制御などが入った。

## できるようになったこと

- **`fallbackModel` 設定**: プライマリモデルが過負荷／利用不可のときに順番に試すフォールバックモデルを最大3つ設定可能。`--fallback-model` が対話セッションにも適用されるように。
- **deny ルールの glob**: ツール名位置で glob パターンを使え、`"*"` で全ツールを deny できる。allow ルールは非 MCP の glob を拒否、deny ルールの未知ツール名は起動時に警告。
- **クロスセッションメッセージの堅牢化**: 他の Claude セッションから `SendMessage` で中継されたメッセージはユーザー権限を持たなくなり、受信側は中継された権限要求を拒否、auto モードはブロックする。
- **thinking 無効化制御**: `MAX_THINKING_TOKENS=0`、`--thinking disabled`、モデル別の thinking トグルが、Claude API 経由で既定で thinking するモデルでも thinking を無効化できる（サードパーティプロバイダーは変更なし）。
- **フォールバックリトライ**: API が想定外の非リトライ可能エラーを返したとき、フォールバックモデルで1回だけターンを再試行（認証・レート制限・リクエストサイズ・トランスポートエラーは即時表面化）。
- `claude update` がダウンロード前に対象バージョンを表示。`claude agents` で URL を入力するとそのセッションへ絞り込み。
- **主な修正**: 画像処理不可エラーと余分なトークン消費、起動時 worker 登録中の一時的バックエンド障害でリモートセッションが固着、JetBrains IDE ターミナル（2026.1+）のちらつき、Kitty キーボードプロトコル端末での Shift+非ASCII脱落、Windows での PowerShell コマンド検証ハング、macOS の `claude --bg-pty-host` 孤児プロセス 100% CPU、無効エントリを含む managed settings が残りの有効ポリシーを黙って無効化する問題、`allowedMcpServers`/`deniedMcpServers` の `${VAR}` 参照不一致、git worktree に入ったバックグラウンドエージェントの「No conversation found」クラッシュループ、など多数。

## 影響範囲

- 対象ユーザー: Claude Code 利用者全般。managed settings・クロスセッション権限の修正は管理者・情シス向け。
- 対象プラン: 制限なし。
- API / UI / 管理者機能: モデル選択（fallbackModel）、権限ルール（deny glob・managed settings）、マルチセッション連携（SendMessage 権限）、IDE/端末互換。

## 教材化メモ

「モデル設定」章に `fallbackModel`（最大3つ・順次試行）を追記し、過負荷時の継続性を担保する運用例を示す。「権限ルール」章には deny の glob（`"*"` で全 deny）と、クロスセッションメッセージが権限を持たない仕様（`SendMessage` 中継の安全性）を加える。後者はマルチエージェント運用のセキュリティ前提として重要。thinking 無効化制御は「コスト/レイテンシ調整」の項に入れる。v2.1.163 からの連続パッチである点を明記。
