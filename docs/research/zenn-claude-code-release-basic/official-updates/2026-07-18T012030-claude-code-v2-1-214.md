# 2026-07-18 Claude Code v2.1.214

## 公式内容の日本語要約

セキュリティ修正が中心のリリース。`Edit(src/**)`のような単一セグメントの`dir/**`許可ルールが、意図した`<cwd>/dir`だけでなくツリー内のどこにあっても同名の`dir/`ディレクトリへの書き込みを自動承認してしまう不具合を修正した。Windows PowerShell 5.1セッションで権限チェックがバイパスされる不具合、bashのリダイレクト構文の解釈差異により権限チェックが安全側に倒れない（fail closedにならない）不具合、1万文字を超える長大コマンドの誤判定、zshの変数展開・修飾子を`[[ ]]`比較内で無害なテキストとして扱ってしまい承認プロンプトが出ない不具合、一部の`help`・`man`コマンドが危険なオプション・コマンド置換・バックスラッシュパスを含んでいても自動承認されてしまう不具合、リモートセッションでローカル確認ダイアログより先に権限プロンプトが進んでしまう不具合など、権限チェック関連の修正が集中している。

新機能としてEndConversationツールが追加され、Claudeが著しく悪意あるユーザーやジェイルブレイク試行に対してセッションを終了できるようになった（claude.aiでは2025年から提供済みの機能、詳細は`https://www.anthropic.com/research/end-subset-conversations`）。長時間ツール呼び出しの定期進捗ハートビート追加、メモリファイルのfrontmatterへのISO `modified`タイムスタンプ追加、OpenTelemetryログイベントへの`message.uuid`・`client_request_id`・`tool_source`属性追加、`CLAUDE_CODE_OTEL_CONTENT_MAX_LENGTH`によるOTelコンテンツ切り詰め上限の設定可能化、`subagentStatusLine`ペイロードへのreasoning effort追加、daemon-redirectフラグ（`--url`・`--connection`・`--identity`、PodmanのDockerシムのリモートモード含む）を持つ`docker`コマンドへの権限プロンプト追加も含む。

## できるようになったこと

- 複数の権限チェックバイパス（`dir/**`ルールの誤承認、PowerShell 5.1、bashリダイレクト、長大コマンド、zsh比較構文、help/manコマンド、リモートセッション競合）を修正
- EndConversationツールで悪意あるユーザー・ジェイルブレイク試行時にセッション終了が可能に
- OpenTelemetryのメッセージレベル相関属性追加、docker daemon-redirectフラグへの権限プロンプト追加

## 影響範囲

- 対象ユーザー: 権限ルール（allow rules）を運用しているチーム、Windows PowerShell 5.1環境の利用者、OpenTelemetryでログ収集している運用者
- 対象プラン: 制限なし
- API / UI / 管理者機能: 権限プレビュー・permission rule評価、EndConversationツール、OpenTelemetryログ、docker権限プロンプト

## 教材化メモ

- `dir/**`許可ルールの誤承認は、既存の権限ルール解説教材があれば「意図しないディレクトリへの書き込み自動承認」の注意点として追記価値が高い。
- EndConversationツールはAI安全性・ジェイルブレイク対策の話題として紹介できる。
