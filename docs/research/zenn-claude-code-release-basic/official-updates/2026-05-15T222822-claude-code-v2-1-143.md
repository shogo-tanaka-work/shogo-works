# 2026-05-15 Claude Code v2.1.143

## 公式内容の日本語要約

Claude Code v2.1.143は、plugin依存関係の扱い、background sessionの継続性、Windows / PowerShell対応を中心に、複数agent運用の実務摩擦をまとめて下げるリリース。`claude plugin disable`時の依存関係保護、`/plugin` marketplaceでの projected context cost 表示、`worktree.bgIsolation: "none"`、background sessionでのmodel / effort保持などが追加された。

同時に、PowerShell toolの既定有効化、Windows Terminal / WSLまわりの修正、`/bg`での各種設定保持、worktree cleanupの安全性改善など、長時間運用やWindows環境で効く修正が多い。

## できるようになったこと

- plugin disable時に依存関係を保護し、enable時に推移的依存も有効化できる。
- `/plugin` marketplaceで、turn単位・呼び出し単位の projected context cost を確認できる。
- `worktree.bgIsolation: "none"`で、worktreeが不向きなrepoでもbackground sessionを直接 working copy に向けられる。
- background sessionがidle復帰後もmodel / effort設定を保持する。
- `claude agents`から起動するsessionへ、`--model`、`--effort`、`--settings`、`--mcp-config`などの既定値を渡しやすくなった。
- Windows / PowerShell環境でのpaste、session表示、権限、filesystemアクセスまわりが改善された。

## 影響範囲

- 対象ユーザー: Claude Code利用者、plugins / background sessions / agentsを使う開発チーム、Windows利用者
- 対象プラン: Claude Code利用環境全般
- API / UI / 管理者機能: CLI / plugins / background sessions / agents / PowerShell

## 教材化メモ

plugins教材では、依存関係 enforcement と projected context cost を、plugin選定・導入時の実務ポイントとして追記候補にできる。background agents教材では、`worktree.bgIsolation: "none"`、model / effort保持、`/bg`での設定保持を、長時間運用の設計論として扱える。Windows向け教材がある場合は、PowerShell toolの既定有効化とWindows関連修正をまとめて補足できる。
