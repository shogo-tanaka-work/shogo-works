# 2026-06-12 Claude Code v2.1.176

## 公式内容の日本語要約

**新機能・改善:**
- **セッションタイトルの多言語自動生成**: セッションタイトルが会話の言語で自動生成されるようになった。`language` 設定で特定の言語にピン可能。
- **`footerLinksRegexes` 設定追加**: フッター行のリンクバッジを正規表現でマッチさせる設定。ユーザーまたは managed settings で設定可。
- **Bedrock 認証情報キャッシュ改善**: `awsCredentialExport` のクレデンシャルが固定1時間ではなく Expiration まで有効になるように。

**`availableModels` enforcement の追加修正（v2.1.175 に続く）:**
- `ANTHROPIC_DEFAULT_*_MODEL` 環境変数経由でブロックされたモデルへのリダイレクトが不可になるように修正
- `/fast` が allowlist 外のモデルに切り替わる場合に拒否するように修正
- Fable 5 停止の緊急対応として、auto mode が Fable 5 で失敗する問題（Opus 4.8 が無効な組織向け）を修正 — classifier が最良の利用可能 Opus モデルにフォールバック

**hook `if` 条件のパスマッチ修正:**
- `Edit(src/**)`, `Read(~/.ssh/**)`, `Read(.env)` などのドキュメント通りのパターンが正しくマッチするように

**Linux sandbox 修正:**
- `.claude/settings.json` が絶対パスを持つシンボリックリンクの場合に sandbox が起動できない問題を修正

**tmux / SSH クリップボード修正:**
- `/copy` とマウス選択コピーが tmux over SSH 環境でシステムクリップボードに届かない問題を修正
- tmux 3.2 より古いバージョンで tmux ペーストバッファが読み込まれない問題を修正

**Remote Control の大型バグ修正（9件）:**
- Web/モバイルからの接続がセッションのモデルをサイレントに変更してしまう問題を修正
- 切断通知が人間が読めるメッセージでなく数値コードを表示していた問題を修正
- 接続失敗時に会話トランスクリプトに重複行が追加されていた問題を修正
- 別アカウントにサインインしてもセッションが切断されない問題を修正

**background sessions・agents view の多数の修正:**
- `/cd` と worktree 移動後に前のディレクトリの git ブランチが報告される問題を修正
- `claude agents` で1ウィンドウの Back 操作が他ウィンドウのセッションをデタッチする問題を修正
- `/bg` mid-turn 後に backgrounded session が永続的に "Working" を表示する問題を修正
- スケジュール起動やジョブブロック中に開かれた PR が `claude agents` の検索に表示されない問題を修正
- Windows での agents view 入力にテキストカーソルが表示されない問題を修正
- `claude --bg -cn <name>` でセッション名がシードされない問題を修正
- Windows のネットワークパスが永続状態で正規化されずに respawn 失敗する問題を修正
- 破損した状態ファイルからの不正な resume ID で respawn が拒否される問題を修正
- `~/.claude/daemon` が ReadOnly 属性の場合に Windows background-service が起動しない問題を修正
- Cloud sessions がアイドル後 "Could not resolve authentication method" で失敗する問題を修正

## できるようになったこと

- セッションタイトルが会話言語で自動生成される（日本語会話なら日本語タイトル）
- `ANTHROPIC_DEFAULT_*_MODEL` 経由での `availableModels` allowlist 回避が不可に
- Bedrock での長時間セッションでクレデンシャル期限切れが起きにくくなった
- Remote Control を使ったときにモデルが予期せず変わる問題が解消
- hook の `if` 条件パスパターンがドキュメント通りに動作する

## 影響範囲

- 対象ユーザー: Claude Code 全ユーザー（特に Remote Control 利用者・Bedrock 利用者・Windows ユーザー・tmux 利用者に重要）
- 対象プラン: `availableModels` enforcement 修正は Enterprise/Team managed 環境に重要
- API / UI / 管理者機能: managed settings の `footerLinksRegexes` 追加

## 教材化メモ

- v2.1.175 で追加された `enforceAvailableModels` に続き、v2.1.176 では `ANTHROPIC_DEFAULT_*_MODEL` 経由の抜け穴も塞いだ。モデル利用統制の完成度が段階的に高まるパターン。
- Remote Control バグ修正9件は、モバイル/Web からの Claude Code 操作を組み込みたい企業向けに重要。
