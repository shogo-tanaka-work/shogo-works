# 2026-06-22 Claude Code v2.1.186 — MCP CLI 認証・Bash コマンド自動応答・多数のバグ修正

## 公式内容の日本語要約

多数の新機能と 20件以上のバグ修正を含む大規模リリース。特に MCP サーバー認証の CLI 対応、`!` Bash コマンドの出力への自動応答機能、AWS 認証リフレッシュ対応が注目点。

## できるようになったこと

- `claude mcp login <name>` / `claude mcp logout <name>` で MCP サーバーをインタラクティブメニューを開かず CLI から直接認証可能（SSH 越し `--no-browser` stdin リダイレクト対応）
- `!` Bash コマンドの出力を Claude が自動で受け取って応答するようになった（`"respondToBashCommands": false` で従来動作に戻せる）
- `/workflows` エージェント詳細ビューにステータスフィルター（`f` キー）追加
- `/plugin` の Installed タブに「Skills」セクション追加
- `teammateMode: "iterm2"` 設定追加（`it2` CLI が見つからない場合の警告付き）
- `/login` に「Claude Platform on AWS - refresh credentials」オプション追加（`awsAuthRefresh` 設定時）

## バグ修正（主要）

- マシンスリープ復帰後にストリーミングリクエストが "Content block not found" / JSON パースエラーで失敗する問題を修正
- サブエージェントトランスクリプトのスクロール位置がメイントランスクリプトに引き継がれる問題を修正
- `Agent(type)` 拒否ルールと `Agent(x,y)` 許可タイプ制限が名前付きサブエージェント起動に適用されない問題を修正
- Workflow `agent({schema})` サブエージェントがスキーマ検証失敗時に無限ループする問題（5回失敗後に中断するよう修正）
- バックグラウンドエージェント実行後に Esc と Ctrl+C が反応しない問題を修正
- `~~取り消し線~~` がリテラルのチルダとして表示される問題を修正
- セッションコストが Enterprise / Team 使用量ベースサブスクライバーに表示されない問題を修正
- エージェントチーム: tmux/pane バックエンドで起動したチームメートがリーダーの `--effort` レベルを継承するよう修正

## 影響範囲

- 対象ユーザー: Claude Code 全ユーザー
- 対象プラン: 全プラン（一部機能は Business/Enterprise 向け）
- API / UI / 管理者機能: CLI コマンド追加・設定追加・TUI 改善

## 教材化メモ

MCP 認証の CLI 対応は、自動化スクリプトや CI/CD パイプラインで Claude Code を活用する開発者向けの重要な改善。`!` コマンド自動応答は「Claude Code が Bash 実行結果を自分で解釈して次のアクションを取る」というエージェント的な動作のリアルな事例として教材化できる。
