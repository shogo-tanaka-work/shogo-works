# 2026-06-09 Claude Code v2.1.170

## 公式内容の日本語要約

Claude Code v2.1.170 がリリースされ、同日 GA となった新モデル **Claude Fable 5** へのアクセスが追加された（v2.1.170 へ更新すると利用可能）。あわせて、VS Code 統合ターミナル（または Claude Code の環境変数を継承したシェル）から起動した際にセッションのトランスクリプトが保存されず `--resume` に出てこない不具合を修正した。

## できるようになったこと

- **Claude Fable 5 へのアクセス追加**（v2.1.170 へ更新で利用可能）。Fable 5 自体の詳細は `../zenn-claude-release-basic/official-updates/2026-06-09T000000-claude-fable-5-mythos-5.md` を参照。
- VS Code 統合ターミナル等から起動したセッションのトランスクリプトが保存されず `--resume` に現れない問題を修正。

## 影響範囲

- 対象ユーザー: Claude Code 利用者全般（Fable 5 を使いたいユーザー、VS Code 統合ターミナル利用者）
- 対象プラン: 全プラン
- API / UI / 管理者機能: モデル選択（Fable 5）、セッション永続化の修正

## 教材化メモ

Claude Code のモデル選択章に、v2.1.170 以降で Fable 5 を選択できる旨を追記（Fable 5 の特性は Claude モデル章へ）。VS Code 統合ターミナル起動時のトランスクリプト未保存修正は「よくあるハマり（--resume にセッションが出ない）」の解消として記録。リリース単体ではモデルアクセス追加＋1修正のため、AIニュースは Fable 5 ローンチ記事側で扱う。
