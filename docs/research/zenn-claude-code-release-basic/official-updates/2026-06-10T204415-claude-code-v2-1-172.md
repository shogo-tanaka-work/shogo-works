# 2026-06-10 Claude Code v2.1.172

## 公式内容の日本語要約

Claude Code v2.1.172 がリリースされた（UTC 2026-06-10T20:44:15Z）。今回の目玉は**サブエージェントが自分のサブエージェントを生成できるようになったこと（最大5階層）**。これにより、マルチエージェント構成で並列作業を深くネストさせる使い方が可能になる。合わせて、Amazon Bedrock 利用時の AWS リージョン自動検出（`~/.aws` 設定ファイルを参照）、`/plugin` マーケットプレイスの検索バー追加など、開発者向けの利便性改善が複数含まれる。バグ修正は20件超に及ぶ。

## できるようになったこと

- **サブエージェントのネスト実行（最大5階層）**: `Agent` ツールを持つサブエージェントが、自分自身のサブエージェントを生成できるようになった（最大5階層まで）。`daily-ai-news-pipeline` のようなオーケストレーション型ワークフローで、さらに深い並列化が可能。
- **Bedrock AWS リージョン自動検出**: `AWS_REGION` 環境変数が未設定の場合、`~/.aws/config` を参照してリージョンを自動解決。AWS SDK の挙動と一致させた。`/status` でリージョン取得元も表示。
- **`/plugin` マーケットプレイスに検索バー追加**: プラグイン一覧をブラウズする際に検索で絞り込めるようになった。
- **OTEL メトリクスに `model` 属性追加**: `claude_code.lines_of_code.count` メトリクスに `model` 属性が付くようになった（オブザーバビリティ改善）。
- **1M コンテキスト超過セッションの自動コンパクト**: usage credits がないまま 1M コンテキストを使うセッションが永久に詰まる問題を修正。コンテキスト制限内に自動コンパクトされるようになった。

## 影響範囲

- 対象ユーザー: Claude Code を利用する全ユーザー
- 対象プラン: Claude Code 対応プラン全般
- Amazon Bedrock 利用者は `~/.aws/config` のリージョン設定が自動で有効になる
- マルチエージェント・オーケストレーション構成を使っている開発者に特に影響あり

## 主なバグ修正（抜粋）

- 複数画像を含む会話で「画像が処理できなかった」エラーが繰り返し表示される問題を修正
- agents ビューでワーカーが返答後30秒まで "Working" スピナーが残る問題を修正
- バックグラウンドエージェントが別ディレクトリのプロジェクト設定（`.mcp.json` 承認、trust）を読み込む問題を修正
- `availableModels` の制限がサブエージェントモデルオーバーライド・エージェントディスパッチモデルピッカー・アドバイザーモデルに適用されなかった問題を修正
- `opusplan` モデル設定がプランモードで1Mコンテキストを送らない問題を修正
- `WebFetch(domain:*.example.com)` のワイルドカードドメインルールがサブドメインにマッチしない問題を修正
- Bedrock でプロバイダーが提供しないモデルを `/model` ピッカーが提示していた問題を修正
- アップ矢印プロンプト履歴がサブエージェントのタブ開き中にメインエージェントの履歴を表示していた問題を修正
- モデルID に 1M コンテキストサフィックスが二重に付く問題を修正（例: `[1M][1m]`）
- Windows コンソールでのマウストラッキング無効化
- VSCode の PowerShell ツール呼び出し表示を修正
- パフォーマンス改善（長い会話の不要なメッセージ正規化除去、/goal ステータスチップのアイドル時 CPU 削減、Chromeツールのバッチロード）

- Bedrock 章で「`AWS_REGION` が未設定でも `~/.aws/config` から自動解決」を追記。
