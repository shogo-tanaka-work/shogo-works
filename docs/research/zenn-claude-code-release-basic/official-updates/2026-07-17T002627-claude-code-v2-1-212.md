# 2026-07-17 Claude Code v2.1.212

## 公式内容の日本語要約

`/fork`の挙動が変更され、会話をバックグラウンドセッション（`claude agents`に自分の行として現れる）にコピーしてユーザーはそのまま作業を継続できるようになった。従来`/fork`が起動していたセッション内サブエージェントは`/subtask`という別コマンドに分離された。

暴走防止の上限が2つ追加された。セッション全体でのWebSearchツール呼び出し数に既定200回の上限（`CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION`で調整可）、セッションあたりのsubagent起動数にも既定200件の上限（`CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION`で上書き可、`/clear`でリセット）が付き、検索ループや委譲ループの暴走を止める。また2分以上かかるMCPツール呼び出しは自動でバックグラウンドに移行するようになり（`CLAUDE_CODE_MCP_AUTO_BACKGROUND_MS`で閾値調整・無効化可）、セッションが固まらなくなった。

その他、`claude auto-mode reset`コマンド追加（確認プロンプト付き、`--yes`でスキップ可）、agent viewでの`/resume`が削除済みセッションも含むピッカーを開きバックグラウンドセッションとして再開できるようになった。バグ修正としては、plan modeが`touch`・`rm`等のファイル変更を伴うBashコマンドを権限確認なしに実行してしまう不具合、`.claude/worktrees`のシンボリックリンクを辿ってリポジトリ外にファイルを作成し得るworktree作成の不具合、`continue:false`フックの停止がツール失敗時に握りつぶされる不具合、print/SDKモードでのBash実行中SIGTERMがプロセスツリーを孤立させる不具合などを修正した。

## できるようになったこと

- `/fork`がバックグラウンドセッション化され、`claude agents`から追跡可能に（従来の`/fork`挙動は`/subtask`に分離）
- WebSearch呼び出し数・subagent起動数にセッション単位の上限を追加し、暴走ループを防止
- 長時間MCPツール呼び出しの自動バックグラウンド化
- `claude auto-mode reset`コマンド、`/resume`の削除済みセッション対応ピッカー

## 影響範囲

- 対象ユーザー: Claude Codeでagent view / auto-mode / MCPを使う全ユーザー、特に長時間セッションや大量委譲を行う運用者
- 対象プラン: 制限なし
- API / UI / 管理者機能: `/fork`・`/subtask`・agent view・auto-mode・MCPツール実行・環境変数設定

## 教材化メモ

- WebSearch/subagentの暴走防止上限は、agentic運用を教える際の「安全弁」の実例として紹介価値が高い。
- `/fork`のバックグラウンドセッション化はagent view活用ガイドの更新ポイントになる。
