# 2026-06-16 Claude Code v2.1.179

## 公式内容の日本語要約

バグ修正中心のパッチリリース。ストリーム途中での接続切断時に部分レスポンスが失われる問題を修正し、WSL2 環境でのマウスホイールスクロール、Linux 大規模ディレクトリでの `denyRead`/`allowRead` グロブパターン適用不全、フィードバックサーベイの誤記録、ウェルカムスクリーンの複数バナー重複、Ctrl+O でのサブエージェントトランスクリプト表示、プロンプト入力フォーカス回帰を修正した。リモートセッションのプラグイン読み込み速度も向上している。

## できるようになったこと

- ストリーム途中切断時に部分レスポンスを保持（スピナー「running tool」固まりも解消）
- Windows Terminal / VS Code 上の WSL2 でマウスホイールスクロールが機能（v2.1.172 の回帰修正）
- `denyRead`/`allowRead` グロブが Linux 大規模ディレクトリでも正しく動作
- Bash ツール説明が肥大化して Linux セッションが使用不可になる問題を解消
- フィードバックサーベイで 1 桁回答がセッションレーティングに誤記録される問題を修正
- ウェルカムスクリーンのプロモーションバナーをセッションあたり最大 1 件に制限
- Ctrl+O がサブエージェント表示中にトランスクリプトを開けない問題を修正
- サブエージェント/フッターパネルからプロンプト入力へのフォーカス遷移を修正
- リモートセッションでのプラグイン読み込み速度向上

## 影響範囲

- 対象ユーザー: 全 Claude Code ユーザー（特に WSL2 / Linux ユーザー、リモートセッション利用者）
- 対象プラン: 全プラン
- API / UI / 管理者機能: CLI UI 改善
