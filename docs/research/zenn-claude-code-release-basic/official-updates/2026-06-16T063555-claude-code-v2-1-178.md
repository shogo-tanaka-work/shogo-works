# 2026-06-16 Claude Code v2.1.178

## 公式内容の日本語要約

**新機能・改善:**

- **`Tool(param:value)` 権限構文の追加**: 権限ルールでツールの入力パラメータを条件にできるようになった。`*` ワイルドカード対応。例: `Agent(model:opus)` で Opus サブエージェントをブロック。
- **ネスト `.claude/skills` ディレクトリのサポート**: そのディレクトリ内のファイルを扱う際に、ネストされた `.claude/skills` ディレクトリのスキルが自動ロードされる。名前衝突時は `<dir>:<name>` 形式で両方が利用可能。
- **ネスト `.claude/` ディレクトリの設定優先**: agent / workflow / output-style は作業ディレクトリに最も近い `.claude/` 設定が優先される。プロジェクトスコープの workflow 保存も最も近い `.claude/workflows/` をターゲットにする。
- **auto mode 改善（サブエージェントの事前分類）**: サブエージェントの起動がクラシファイアで評価されてから行われるようになり、サブエージェントがブロックされたアクションをレビューなしに要求できていたギャップを解消。
- **`/doctor` 改善**: 全セクションで一貫したフラットツリーレイアウト、より明確なセクションステータスアイコン、コマンド名のハイライト表示。
- **skill 一覧の切り捨て警告改善**: 影響を受ける skill description の数が表示されるようになった。
- **workflow キーワードのトリガー変更**: 紫色のシマーハイライトを使用し、「run a workflow」「workflow:」などの明示的なフレーズでのみ発火するように。単なる「workflow」という単語では発火しない。
- **Remote Control エラーメッセージ改善**: 接続失敗時にフッターに永続的な赤色の "/rc failed" インジケータを表示。"not yet enabled" エラーが gate / check failure / stale entitlement / org policy のいずれかを明示するように。
- **`/bug` の改善**: 送信前に説明が必須になり、モデル拒否テキストを GitHub issue タイトルに使わなくなった。

**バグ修正:**

- websocket/OAuth ファイルディスクリプタ環境変数が stale な場合の OOM クラッシュを修正
- 別アカウントの OAuth トークンで Chrome の Claude 接続がサイレント失敗していた問題を修正
- 非インタラクティブ実行でのネスト `.claude/skills` の権限プロンプトブロックを修正
- サブエージェントのトランスクリプト閲覧時にツール結果とライブ進捗が表示されるように修正
- バックグラウンドサブエージェントを ctrl+b で backgrounding しても再起動しないように修正
- `ANTHROPIC_BASE_URL` / `ANTHROPIC_AUTH_TOKEN` 経由の API ゲートウェイ使用時の `401 Invalid bearer token` エラーを修正
- コンパクションが `--fallback-model` を尊重するように修正
- セッション外で認証情報が更新された後も古いキャッシュ設定でリクエストが失敗する問題を修正
- `/bg` や `←←` 後に background sessions が "Working" のまま固まる問題を修正
- `CLAUDE_CODE_PLUGIN_KEEP_MARKETPLACE_ON_FAILURE=1` が新規 marketplace インストールのクローンを妨げていた問題を修正
- サブエージェントの `disallowedTools` における MCP サーバーレベルスペックの処理修正

## できるようになったこと

- 権限ルールでモデル名・パラメータ値ベースのブロックが可能に（例: Opus サブエージェントのみブロック）
- 作業ディレクトリに応じてネストされた skills / workflows / agent 設定が自動適用される
- auto mode でサブエージェントが起動前にブロック判定されるため、意図しないアクションの防止が強化
- Remote Control 接続問題の原因がフッターのエラーメッセージで特定しやすくなった

## 影響範囲

- 対象ユーザー: Claude Code 全ユーザー
- 対象プラン: `Tool(param:value)` 権限は Enterprise / managed settings 利用者に特に有用
- API / UI / 管理者機能: ネスト `.claude/` 対応はモノレポ・マルチプロジェクト構成で重要

## 教材化メモ

- `Tool(param:value)` 構文は細粒度の権限管理の新パターン。`Agent(model:opus)` でコスト制御も可能になる。
- ネスト `.claude/skills` 対応はマルチプロジェクトのリポジトリで skills を局所化できる重要な改善。
- auto mode のサブエージェント事前分類は、エージェント実行の安全性向上のアーキテクチャ変更として教材的に重要。
