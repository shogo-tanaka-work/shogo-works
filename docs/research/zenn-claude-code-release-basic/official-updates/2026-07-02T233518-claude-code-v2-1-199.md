# 2026-07-02 Claude Code v2.1.199

## 公式内容の日本語要約

Claude Code v2.1.199 が 2026-07-02T23:35Z にリリース。目玉は `/skill-a /skill-b do XYZ` のようなスタック型スキル呼び出しで先頭最大5個のスキルを全て読み込めるようになった点。残りはバックグラウンドエージェント・サブエージェントまわりの信頼性改善が中心で、SSL証明書エラーの即時ガイド表示、ストリーミング応答が途中エラーで破棄されずpartialを保持、サブエージェントがレート制限/サーバーエラーで打ち切られても部分結果を親へ返す、Linux上のバックグラウンドデーモンが不正終了後に自滅ループする不具合の修正などを含む。

## できるようになったこと

- **スタック型スキル呼び出し**: `/skill-a /skill-b do XYZ` で先頭から最大5個のスキルを全てロード（従来は最初の1つのみ）
- SSL証明書エラー（TLSインスペクトプロキシ、`NODE_EXTRA_CA_CERTS`未設定、期限切れ証明書）を即座に検知し、リトライを浪費せず対処ヒントを表示
- ストリーミング中にAPIがオーバーロード/サーバーエラーを返しても、部分出力を「不完全な応答」通知付きで保持
- サブエージェントがレート制限やサーバーエラーで打ち切られた際、無言で失敗させず部分作業を親エージェントへ返す
- サブエージェントのAPIエラー（利用上限到達等）を「成功」と誤報告していた不具合を修正し、親エージェントへ正しくエラー報告
- Linux上のバックグラウンドデーモンが不正終了後の破損ワーカー記録により約50秒ごとに自滅する不具合を修正
- macOS上でSSH経由のバックグラウンドエージェントがコールドスタートに失敗する不具合（2.1.196の退行）を修正
- `claude stop` がバックグラウンドエージェントの再起動と競合して無効化される不具合を修正
- `SessionStart`/`Setup`/`SubagentStart` フックがexit code 2で終了した際にstderrを隠していた不具合を修正し、トランスクリプトにエラー表示
- 一時的なレート制限（429、利用上限とは無関係）をSubscriber向けに自動バックオフ付きリトライへ変更
- `CLAUDE_CODE_RETRY_WATCHDOG` のデフォルトリトライ回数を300に引き上げ、`CLAUDE_CODE_MAX_RETRIES`の上限15を撤廃

## 影響範囲

- 対象ユーザー: Claude Code 全ユーザー、特にバックグラウンドエージェント（`claude agents`）・サブエージェントの利用者
- 対象プラン: 全プラン
- API / UI / 管理者機能: CLI/エージェントビューの信頼性向上が中心。管理者向け設定変更なし

## 教材化メモ

- スタック型スキル呼び出し（最大5個）は複数スキルの組み合わせワークフロー教材の題材になる
- サブエージェントの「部分結果を親に返す」設計は、エージェントオーケストレーションの障害時フォールバック設計の実例として使える
