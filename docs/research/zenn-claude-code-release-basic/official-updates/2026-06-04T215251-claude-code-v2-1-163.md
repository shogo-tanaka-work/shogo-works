# 2026-06-04 Claude Code v2.1.163

## 公式内容の日本語要約

Claude Code v2.1.163 がリリースされた。管理者向けのバージョン強制（`requiredMinimumVersion` / `requiredMaximumVersion`）、`/plugin list` コマンドの追加、Stop / SubagentStop hooks がエラー扱いにならずにフィードバックを返せる仕組みなど、機能追加と多数の不具合修正を含む。

## できるようになったこと

- **バージョン強制（managed settings）**: `requiredMinimumVersion` / `requiredMaximumVersion` を組織の管理設定で指定でき、許可範囲外のバージョンでは Claude Code が起動を拒否し、承認済みバージョンへ誘導する。
- **`/plugin list`**: インストール済みプラグインを一覧表示。`--enabled` / `--disabled` フィルタ付き。
- **`/btw` に「c to copy」**: 回答の生 Markdown をクリップボードへコピーし、他所への貼り付けで書式を保持。
- **Hooks**: Stop / SubagentStop hooks が `hookSpecificOutput.additionalContext` を返せるようになり、hook エラーとして扱われずに Claude へフィードバックを与えてターンを継続できる。
- **Skills**: コマンド本文で数字の前にリテラルの `$` を含めるための `\$` エスケープ構文を追加。
- **MCP**: stdio MCP サーバーが `--resume` 時に hooks / Bash と同じ `CLAUDE_CODE_SESSION_ID` を受け取る。
- **主な修正**:
  - `claude -p` が最終結果後に永久ハングする問題（バックグラウンドのシェルを stdin クローズ後 ~5 秒で停止）。
  - `CI=true` かつ Anthropic API キー未設定の Bedrock / Vertex / Foundry で `claude -p` が「ANTHROPIC_API_KEY required」で失敗する問題。
  - bazel・EDR 保護下の Go ワークフローで Bash が失敗する `$TMPDIR` 上書きの回帰（2.1.154 由来）。
  - Windows で session-env ディレクトリが読み取り専用 / OneDrive 配下のとき「EEXIST」で Bash が失敗する問題。
  - 起動中に管理設定の取得が完了すると、org 管理の権限ルールがセッション全体に適用されない問題。
  - `claude agents` のバックグラウンドセッションが更新後の再アタッチで実行中タスクを失う問題。
  - agent view を Esc で抜ける際の表示崩れと数秒のハング。
  - hook の `if: "Bash(...)"` 条件がサブシェル・バッククォート内のコマンドにもマッチするよう修正。
  - ホームディレクトリパスの deny ルール（例 `Read(~/Desktop/**)`）が `$HOME` 経由参照の Bash をブロックしない問題。

## 影響範囲

- 対象ユーザー: Claude Code 利用者全般。バージョン強制・org 管理権限の修正は管理者・情シス向け。
- 対象プラン: 制限なし（managed settings は組織管理が前提）。
- API / UI / 管理者機能: 管理設定（バージョン範囲・org 権限）、CLI（`/plugin list`・`/btw`）、Hooks、Skills、MCP に影響。

## 教材化メモ

「管理設定」章に `requiredMinimumVersion` / `requiredMaximumVersion` を追記し、社内で許可バージョンを固定する運用例を示す。「Hooks」章には Stop / SubagentStop の `additionalContext` を、ターンを止めずにフィードバックを差し込む手段として加える。v2.1.157 以降の累積改善（権限ルール・MCP・Windows）の延長として位置づける。
