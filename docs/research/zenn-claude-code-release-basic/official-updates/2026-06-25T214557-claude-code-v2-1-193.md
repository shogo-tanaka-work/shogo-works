# Claude Code v2.1.193

## 主要変更点

### autoMode.classifyAllShell
- 全ての Bash/PowerShell コマンドを auto-mode classifier 経由でルーティングする設定を追加
- 従来は任意コード実行パターンのみが対象だった

### OTel assistant_response ログイベント
- `claude_code.assistant_response` OpenTelemetry ログイベントを追加
- モデルの応答テキストを含む
- デフォルトはリダクション。`OTEL_LOG_ASSISTANT_RESPONSES=1` で有効化
- 未設定の場合は `OTEL_LOG_USER_PROMPTS` に従う（prompts ログを有効にしている環境では自動的にレスポンスも記録される — `OTEL_LOG_ASSISTANT_RESPONSES=0` で停止可能）

### bash モードのライブファイルパス補完
- `!` bash モードでファイルパスのライブ補完機能を追加

### MCP スタートアップ通知
- MCP サーバーが認証を必要とする場合、起動時に `/mcp` を指す通知を表示

### メモリ圧迫の自動回収
- アイドル状態のバックグラウンドシェルコマンドを自動的にメモリ圧迫回収
- `CLAUDE_CODE_DISABLE_BG_SHELL_PRESSURE_REAP=1` で無効化可能

### MCP headersHelper 自動再接続
- ツール呼び出しが 401/403 を返した場合、headersHelper が自動的に再実行・再接続

### プラグイン自動リネーム
- マーケットプレイスの `renames` マップを自動的に追従し、設定を新名称に更新

## バグ修正

- `/model` 等のクライアントデータ依存 UI が `/login` 直後に古い/空の状態を表示する問題を修正
- バックグラウンド化時（←←）に「N 個のバックグラウンドタスクが破棄される」と誤ってキャンセルされる問題を修正
- ピン留めバックグラウンドエージェントが自動更新のたびに「続きから再開しますか？」と再度尋ねる問題を修正
- メインターンをバックグラウンド化したときに「general-purpose (resumed)」というファントムサブエージェントが生成されメイン会話を再実行する問題を修正
- サブエージェント表示中にエージェントパネルが兄弟エージェントを非表示にする問題を修正
- バックグラウンドエージェント改善: 起動結果が「応答を終了してください」と指示しなくなった — エージェント実行中に他のタスクを継続できる
- `/add-dir` のメッセージ改善（指定ディレクトリが既に作業ディレクトリの場合）
