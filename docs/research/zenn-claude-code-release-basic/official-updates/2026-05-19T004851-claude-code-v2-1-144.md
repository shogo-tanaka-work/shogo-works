# 2026-05-19 Claude Code v2.1.144

## 公式内容の日本語要約

Claude Code v2.1.144 は、background session の運用性向上と、起動・ターミナル表示・接続性まわりの安定化を中心とした大型リリース。`/resume` が background session（`claude --bg` / agent view 起動分）も対象になり、interactive session と並んで `bg` 表示で一覧・再開できるようになった。background subagent の完了通知に経過時間（例: "Agent completed · 3h 2m 5s"）が表示される。

UI 面では `/model` がデフォルト挙動を変更し、当該 session のみモデルを変更、`d` キーで新規 session の既定を設定する方式になった。CLI 文言の "extra usage" は "usage credits" に改称され、`/extra-usage` は `/usage-credits` へ（旧名も継続動作）。`/plugin` の browse / discover ペインに plugin の最終更新日が表示される。

安定性では、`api.anthropic.com` 到達不可時に最大75秒起動がハングしていた問題（side-channel API 呼び出しを15秒でタイムアウト化）、長時間 session でのターミナル表示崩れ、VS Code ペイン分割でのリサイズ崩れ、2.1.143 リグレッションだった Full Disk Access 保護フォルダ配下での macOS background session クラッシュなど、多数の修正が入った。MCP の paginated `tools/list` が1ページ目しか返さずツールを取りこぼす問題や、Bedrock / Vertex で「Opus (1M context)」が `/model` から選べない（v2.1.129 リグレッション）問題も修正。

## できるようになったこと

- `/resume` が background session も対象になり、`bg` 表示で interactive と並んで再開できる。
- background subagent 完了通知に経過時間が表示される。
- `/model` は当該 session のみ変更、`d` で新規 session 既定を設定する方式になった。
- CLI 文言が "usage credits" に統一され、`/usage-credits`（旧 `/extra-usage`）で利用枠を確認できる。
- `/plugin` browse / discover ペインで plugin の最終更新日を確認できる。
- `api.anthropic.com` 到達不可時の起動ハング（最大75秒）が解消され、15秒でタイムアウトする。
- MCP の paginated `tools/list` で全ページのツールを取得できるようになった。
- Bedrock / Vertex で `/model` から「Opus (1M context)」を選べるようになった（リグレッション修正）。
- Windows での attached background session スクロール（PgUp/PgDn・ホイール・Ctrl+O）が動作するようになった。

## 影響範囲

- 対象ユーザー: Claude Code 利用者、background session / agents / plugins を使う開発チーム、Bedrock / Vertex 利用組織、Windows 利用者
- 対象プラン: Claude Code 利用環境全般
- API / UI / 管理者機能: CLI / background sessions / `/resume` / `/model` / `/plugin` / MCP / Bedrock・Vertex モデル選択

## 教材化メモ

background agents 教材では、`/resume` の background session 対応、完了通知の経過時間表示、`/bg` での設定維持を「長時間・複数 agent 運用の管理」観点で扱える。モデル選定教材では `/model` の session 限定変更と `d` での既定設定という新しいメンタルモデルを補足候補にできる。Bedrock / Vertex 利用前提の教材がある場合は「Opus (1M context) 選択不可の解消」を1行追記できる。v2.1.144 固有の修正一覧は持ち込まず、運用設計の結論だけ反映する。
