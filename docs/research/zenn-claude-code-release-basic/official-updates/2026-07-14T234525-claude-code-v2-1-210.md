# 2026-07-14 Claude Code v2.1.210

## 公式内容の日本語要約

セキュリティ関連の修正として、`isolation: 'worktree'`で起動したsubagentが、本来隔離されるべき自分のworktreeではなくメインリポジトリのcheckoutに対してgit変更コマンドを実行できてしまう不具合を修正した。また`Write(path)`・`NotebookEdit(path)`・`Glob(path)`という権限ルールは実質的に無効（`Edit(path)`や`Read(path)`を使うべき）であるため、起動時に警告を表示するようになった。`ultracode`キーワードのopt-inがWebhookペイロードや中継されたPRコメントなど人間以外の入力でも誤発火する不具合も修正されている。

UI面では、折りたたまれたツールサマリー行に経過時間のライブカウンターが追加され、長時間実行中のツール呼び出しが視覚的に分かるようになった。その他、`claude attach`が「job not found」等のエラーでセッション遷移中に失敗する問題、hookコールバックのタイムアウトがユーザー拒否と誤解釈されて無人セッションが停止する問題、`claude agents --effort ultracode`の値が破棄されて反映されない問題など、多数のバグを修正した。

Auto modeの権限判定モデルがSonnet 5をデフォルトとし、セッションの最初のリクエストで検証・固定されるよう改善された。またBedrock・Vertex・Foundry利用時、`/doctor`のauto mode既定提案がスキップされる不具合を修正した。

## できるようになったこと

- 危険な権限ルール（`Write(path)`等）への起動時警告
- ツールサマリー行のライブ経過時間表示
- worktree分離subagentのgit操作隔離を修正（セキュリティ上重要）
- `ultracode`キーワードの誤発火防止（Webhook/中継コメント対策）

## 影響範囲

- 対象ユーザー: subagentでworktree隔離を使う開発者、`claude agents`ユーザー、Bedrock/Vertex/Foundry利用者
- 対象プラン: 制限なし
- API / UI / 管理者機能: subagent権限モデル、権限ルール設定、`claude agents`、auto mode

## 教材化メモ

- worktree分離のgit漏れ修正はセキュリティ上重要な修正。subagentのisolation設定を使う教材では「隔離が期待どおり効いているか」の確認手順に触れる価値がある。
- `ultracode`のWebhook誤発火防止は、外部トリガー経由でClaude Codeを操作する自動化構成を組む際の注意点として紹介候補。
