# 2026-06-09 Claude Code v2.1.169

## 公式内容の日本語要約

Claude Code v2.1.169 がリリースされた。全カスタマイズ（CLAUDE.md・プラグイン・スキル・フック・MCP サーバー）を無効化して起動する `--safe-mode` フラグ、プロンプトキャッシュを壊さずセッション中に作業ディレクトリを移すための `/cd` コマンド、バンドル済みスキル/ワークフロー/組み込みスラッシュコマンドをモデルから隠す `disableBundledSkills` 設定が追加された。あわせて、エンタープライズ管理 MCP ポリシー（`allowedMcpServers`/`deniedMcpServers`）の適用漏れ修正など、多数の不具合修正が入った。

## できるようになったこと

- **`--safe-mode`（`CLAUDE_CODE_SAFE_MODE`）**: CLAUDE.md・プラグイン・スキル・フック・MCP サーバーを全無効化して起動。問題の切り分けに使える。
- **`/cd` コマンド**: セッション中にプロンプトキャッシュを壊さず作業ディレクトリを変更。
- **`disableBundledSkills`（`CLAUDE_CODE_DISABLE_BUNDLED_SKILLS`）**: バンドル済みスキル・ワークフロー・組み込みスラッシュコマンドをモデルから隠す。
- `claude agents --json` が blocked / 直後 dispatch のバックグラウンドセッションを含むよう修正し、`--all`（完了セッションも含む）と `id` / `state` フィールドを追加。
- `/workflows` がターン進行中でも即座に開くように。`TaskCreate` の信頼性向上（不正入力の自動修復、未ロードツールのバリデーションエラーにスキーマを同梱）。
- 「CLAUDE.md が長すぎる」警告の閾値がモデルのコンテキストウィンドウに応じてスケール。

## 影響範囲

- 対象ユーザー: Claude Code 利用者全般
- 対象プラン: 全プラン（managed MCP ポリシー関連は組織管理者向け）
- API / UI / 管理者機能: 起動オプション（safe-mode）、セッション操作（/cd）、設定（disableBundledSkills）、エンタープライズ MCP ポリシー適用、各種安定性修正

## 主な修正

- **エンタープライズ管理 MCP ポリシー**（`allowedMcpServers`/`deniedMcpServers`）が、再接続時・IDE 入力設定・`--mcp-config` サーバー・インストール直後の初回セッション・リモート設定読込前に適用されない問題を修正。リモート設定を持たない組織のコールドスタート遅延も改善。
- macOS で claude.ai 認証ログイン時、各ターン開始時の ~30-50ms UI スタールを修正。
- Windows で `claude -p` がスラッシュコマンド/スキルスキャン待ちで遅延・ハングする問題（2.1.161 由来の回帰）を修正。
- 信頼されていないプロジェクト設定が trust 確認なしに OTEL クライアント証明書パスを設定できた問題を修正。
- Vertex / Foundry にデフォルト 5 分のアイドルタイムアウトを復活（`API_FORCE_IDLE_TIMEOUT=0` でオプトアウト）。

## 教材化メモ

「トラブルシューティング」章に `--safe-mode` を追加し、カスタマイズ全無効化での切り分け手順を示す。「セッション操作」章に `/cd`（プロンプトキャッシュを保ったままのディレクトリ移動）を、「設定」章に `disableBundledSkills` を追記。v2.1.166 で扱ったエンタープライズ MCP ポリシーの続きとして、再接続・初回セッション・リモート設定読込前の適用漏れ修正を「管理 MCP ポリシー運用時の前提」として明記する。
