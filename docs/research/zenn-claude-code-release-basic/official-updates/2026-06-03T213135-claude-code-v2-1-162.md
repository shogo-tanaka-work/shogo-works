# 2026-06-03 Claude Code v2.1.162

## 公式内容の日本語要約

Claude Code v2.1.162 がリリースされた。新機能というより、`claude agents` まわりの操作性改善・権限ルールの不具合修正・起動まわりの堅牢化が中心。日常利用の安定性と細かな UX を底上げするパッチ。

## できるようになったこと

- `claude agents --json` の出力に `waitingFor` が追加され、待機中セッションが何でブロックされているか（例: 権限プロンプト）が分かる。
- `--tools` で Grep/Glob を明示すると、埋め込み検索を持つネイティブビルドで専用検索ツールが提供されるように（従来は無視されていた）。
- `/effort` が、選んだレベルを新規セッションの既定として保持する旨を確認表示するように。
- オートコンプリートのスラッシュコマンドをクリックすると、即実行ではなくプロンプトに挿入されるように（Enter で実行）。
- Remote Control が起動メッセージではなく、永続フッターのピル表示（セッションへのリンク付き）に。
- `/ide` メニュー等で Windsurf を「Devin Desktop」へ改名（エディタのリブランドに追随）。

## 主な修正

- 設定ディレクトリが読み取り専用/書き込み不可のときの無言ハングを修正（メモリ内設定で起動しエラーを表示）。
- WebFetch の権限ルールがプリ承認ドメインに適用されない問題を修正（明示的な allow/deny/ask が優先）。
- Windows の権限ルールがバックスラッシュ表記や大文字小文字差でマッチしない問題を修正。
- stream-json/SDK セッションでターン開始直後の Esc（中断）が無言で失われる問題を修正。
- MCP サーバー単位の `timeout` が 1000ms 未満で全ツール呼び出しを中断していた問題を修正。
- `claude agents` の表示幅・名前列の切り詰め、アタッチ不具合、画像ペースト不可などを多数修正。

## 影響範囲

- 対象ユーザー: Claude Code 利用者全般。`claude agents` / Remote Control / MCP / Windows 環境の利用者に効果が大きい。
- 対象プラン: 全プラン。
- API / UI / 管理者機能: CLI/TUI（UI）、権限ルール・MCP timeout（設定）。

## 教材化メモ

Claude Code 教材の「権限ルール」章に、WebFetch のプリ承認ドメイン優先順位と Windows パス表記マッチの修正を反映。`--tools` の Grep/Glob 明示と `/effort` の既定化挙動は CLI オプション解説に追記。`claude agents --json` の `waitingFor` はマルチセッション運用の監視フックとして触れる。
