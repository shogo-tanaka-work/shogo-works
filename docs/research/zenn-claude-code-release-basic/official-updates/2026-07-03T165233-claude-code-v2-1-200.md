# 2026-07-03 Claude Code v2.1.200

## 公式内容の日本語要約

Claude Code v2.1.200 が 2026-07-03T16:52Z にリリース。ユーザー影響が大きい変更は2点。1つ目は `AskUserQuestion` ダイアログがデフォルトで自動継続しなくなり、アイドルタイムアウトを使う場合は `/config` から明示的にオプトインする必要がある変更。2つ目は、CLI・`--help`・VS Code・JetBrains全体で「default」権限モードの表記が「Manual」に統一され、`--permission-mode manual` / `"defaultMode": "manual"` も `default` と併用可能になった変更。その他、`.claude.json` の `disabledMcpServers`/`enabledMcpServers` に非配列値を設定した際の起動クラッシュ修正、スリープ/復帰後にバックグラウンドセッションが無言で停止する不具合の修正、破損した `daemon.lock` によりバックグラウンドエージェントが二度と起動しない不具合の修正、tmux 3.4+でのレンダリングちらつき修正（同期端末出力）、スクリーンリーダー向け出力改善などが含まれる。

## できるようになったこと

- **`AskUserQuestion` の自動継続を廃止**: デフォルトで自動継続しなくなり、アイドルタイムアウトは `/config` からのオプトイン制に変更
- **権限モード表記統一**: デフォルト権限モードの名称を「default」から「Manual」に変更。CLI・`--help`・VS Code・JetBrains全てで一貫表示。`--permission-mode manual`/`"defaultMode": "manual"` は `default` と同じ意味で併用可
- `disabledMcpServers`/`enabledMcpServers` に非配列値が設定されている場合の起動クラッシュを修正
- スリープ/復帰後や停滞セッション再開時にバックグラウンドセッションが無言で停止する不具合を修正
- 古いビルドが再インストールされてもデーモンを乗っ取れないよう、ビルド新旧判定をバージョン埋め込みのビルドタイムスタンプで実施するよう修正
- バックグラウンドエージェントのロースター管理不具合（孤児クリーンアップの恒久停止、古いバイナリでのフィールド欠落、デーモン再起動時のソケット認証トークン消失）を修正
- `claude agents --plugin-dir <dir>` でフラグを `agents` の後に置いた場合、プラグインのagents/skillsがagent viewに表示されない不具合を修正
- 同一リポジトリのgit worktreeからプロジェクトスコープのpluginが正しくロードされない不具合を修正
- tmux 3.4+でのレンダリングちらつきを同期端末出力の有効化で修正
- スクリーンリーダー出力を改善（装飾グリフを非表示化、トランスクリプト記号を短いラベル化、ネストしたテーブルを読み上げ可能な形式に）
- 音声入力で無音録音時に誤解を招く「Voice connection failed」表示を修正

## 影響範囲

- 対象ユーザー: Claude Code 全ユーザー。特に `AskUserQuestion` を使うワークフロー、CLI/VS Code/JetBrains拡張の権限設定を確認しているユーザー
- 対象プラン: 全プラン
- API / UI / 管理者機能: CLI・VS Code・JetBrains拡張のUI表記変更、`.claude.json`設定バリデーション、アクセシビリティ改善

## 教材化メモ

- 権限モード名称統一（Manual）は、既存の `defaultMode` 設定を使っている教材・スクリーンショットの表記更新が必要になる可能性がある
- `AskUserQuestion` の自動継続廃止は「Human-in-the-loopをデフォルトにする」設計思想の実例として、自動化教材の注意点に使える
