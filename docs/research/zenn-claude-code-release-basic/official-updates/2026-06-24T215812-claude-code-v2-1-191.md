# 2026-06-24 Claude Code v2.1.191

## 公式内容の日本語要約

Claude Code v2.1.191 がリリースされた。`/rewind` コマンドの追加、バックグラウンドエージェントの停止永続化、MCP サーバーの信頼性向上、ストリーミング CPU 使用率 37% 削減が主な改善点。v2.1.190 は同日の "Bug fixes" のみだったため v2.1.191 が実質的な主リリース。

## できるようになったこと

**新機能:**
- `/rewind`: `/clear` を実行する前の状態から会話を再開できる機能を追加

**バグ修正（主要）:**
- バックグラウンドエージェントの再起動問題を修正（タスクパネルから停止すると永続的に停止するように）
- hooks でカンマ区切りマッチャー（例: `"Bash,PowerShell"`）が起動しなかった問題を修正
- `/voice` が組織ポリシーで無効化されている場合、汎用エラーではなくポリシー制限の説明を表示するよう改善
- `/permissions` で「最近拒否されたタブ」の承認が閉じると破棄されていた問題を修正
- Windows Terminal でログイン URL が折り返し時に切り捨てられる問題を修正
- Ghostty over ssh/tmux でフルスクリーンモードの Cmd+click が機能しない問題を修正
- `claude agents` で `/usage` のような組み込みスラッシュコマンドがバックグラウンドセッションにテキストとして送られていた問題を修正
- `claude agents` でペースト画像がファイルシステムのフルパスで表示されていた問題を修正（→ `[Image #N]` に変更）
- エージェントパネルでロスタースクロール時に 1 行ジャンプする問題を修正
- ウェルカムスプラッシュが macOS Terminal のデフォルト 80×24 ウィンドウからはみ出す問題を修正
- managed settings の `forceRemoteSettingsRefresh` が MDM / ファイルポリシー経由で設定された場合に有効にならない問題を修正

**改善:**
- サンドボックスネットワーク許可ダイアログ: "Yes" で許可したホストがセッション中記憶されるように（毎回プロンプトが出る問題を解消）
- MCP サーバー信頼性向上: capability discovery が一時的なネットワークエラーをバックオフ付きでリトライ
- MCP OAuth 改善: discovery とトークンリクエストが一時エラーをリトライ。ヘッドレス環境ではブラウザポップアップをスキップして URL 貼り付けプロンプトに直行
- MCP エラーメッセージ改善: HTTP 404 エラーに URL と MCP 設定へのポインタを表示
- vim モードのプロンプト履歴検索（NORMAL `/`）にスラッシュコマンドへのヒントを追加
- **ストリーミングレスポンス中の CPU 使用率を約 37% 削減**（100ms でテキスト更新を合算）
- 長セッションでのターミナル出力キャッシュによるメモリ増加を軽減

## 影響範囲

- 対象ユーザー: Claude Code 全ユーザー。特に MCP ユーザー・agents view ユーザー・長時間セッションユーザー
- 対象プラン: 全プラン
- API / UI: CLI・フルスクリーン・agents view・MCP・MDM 管理環境

## 教材化メモ

- `/rewind` は「AI との会話を分岐・巻き戻す」UX 設計の実装例。非破壊的な実験が可能になる
- 「バックグラウンドエージェント停止の永続化」は AI エージェント管理の UX 問題（停止が無効になる）の具体例
- CPU 37% 削減（100ms コアレッシング）はレンダリング最適化のパターンとして教材化できる
- MCP OAuth のヘッドレス対応は「GUI 前提設計 → CLIファースト設計」の改善例として使える
