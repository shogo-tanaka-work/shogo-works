# 2026-05-21 Claude Code v2.1.146

## 公式内容の日本語要約

Claude Code v2.1.146 を公開。`/simplify` を `/code-review` に改名し、optional な「effort level」（処理深度）を受け付けるようになった。Auto Mode は、ユーザーまたはスキルが明示的に `AskUserQuestion` に依存する場合は確認質問を抑制しない動作に変更（自動化と対話のバランス調整）。Windows / MCP / OTEL / 権限プロンプト / 大規模ファイル差分など、運用上効く修正が15項目入っている。

## できるようになったこと

- `/simplify` → `/code-review` 改名、optional effort level の指定
- Auto Mode が `AskUserQuestion` 必須のスキル・ユーザー指定では確認を抑制しなくなる
- 大規模ファイル編集の差分表示性能向上
- ネイティブアップデータがネットワーク一時障害でリトライ
- バックグラウンド化セッションが既付与権限を再プロンプトしない

## 主なバグ修正

- Windows: winget / Microsoft Store 経由 pwsh でのコマンドラインエラー
- MCP: ページネーション対応サーバーの2ページ目以降がドロップ
- Windows Terminal: ストリーミング中の背景セッションちらつき
- Windows: バックグラウンドジョブのワークツリー削除が NTFS ジャンクションに非追従
- スキルのみ入力セッションで `/background` が拒否されるバグ
- `/theme` 色エディタで Esc が無反応
- Agent SDK 経由実行時のストリーミング終了後の例外
- `forceLoginOrgUUID` / `forceLoginMethod` ポリシーがサードパーティプロバイダーに非適用
- GNOME Terminal の右クリック・中クリック貼り付け
- `CLAUDE_CODE_SUBAGENT_MODEL` がマルチエージェントセッションで子プロセスに非伝播

## 影響範囲

- 対象: Claude Code 全ユーザー（特に Windows ユーザー、MCP 連携、Auto Mode 運用）
- 対象プラン: 無償 / 有償全プラン
- 影響面: CLI / Hooks / MCP / OTEL / 権限プロンプト

## 教材化メモ

`/simplify` → `/code-review` の改名は教材側のコマンド表記更新候補。Auto Mode の `AskUserQuestion` 非抑制は、スキル側で明示的にユーザー確認を挟みたいケースの教材化対象。MCP ページネーション修正は MCP 教材で「ページ送り対応の必要性」を補強する材料。
