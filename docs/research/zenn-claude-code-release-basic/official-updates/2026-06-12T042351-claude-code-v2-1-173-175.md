# 2026-06-12 Claude Code v2.1.173–175

## 公式内容の日本語要約

窓内（2026-06-11T10:18 JST〜2026-06-12T13:50 JST）に Claude Code の 3 リリースが出た。

**v2.1.173**（2026-06-11T05:41:54Z）:
- Fable 5 モデル名の `[1m]` サフィックスが正規化されるように（Fable 5 はデフォルトで 1M コンテキストを持つため、サフィックスを自動除去）
- Windows でサンドボックス有効時に「sandbox dependencies missing」警告が誤表示される問題を修正

**v2.1.174**（2026-06-12T01:16:36Z）:
- `wheelScrollAccelerationEnabled` 設定を追加 — フルスクリーンモードでのマウスホイールスクロール加速を無効化できるように
- `/model` ピッカーが Default が解決するモデルファミリーを隠す問題を修正（Max/Team Premium/Enterprise では Opus が独立行として表示、Pro/Team では Sonnet、PAYG API アカウントでは Opus）
- `/model` ピッカーで `ANTHROPIC_DEFAULT_SONNET_MODEL` がピンする Sonnet 以外のバージョンなのにハードコードされた Sonnet バージョンラベルを表示する問題を修正
- 「Fable 5 is now consuming usage credits」バナーが usage-based billing の enterprise アカウントに誤表示される問題を修正
- Bedrock GovCloud リージョン（`us-gov-*`）で inference profile prefix が `global` と誤導出され、モデル ID 400 エラーになる問題を修正
- バックグラウンドセッションが設定を誤継承する問題の修正（本文 truncated）

**v2.1.175**（2026-06-12T04:23:51Z）:
- `enforceAvailableModels` managed setting を追加 — 有効にすると `availableModels` allowlist が Default モデルの選択も制限（allowlist 外に解決する Default は最初の許可モデルにフォールバック）。ユーザーやプロジェクト設定から managed な `availableModels` リストを拡張不可になる。

## できるようになったこと

- **enterprise managed 環境で Default モデルも allowlist で統制可能**（v2.1.175 `enforceAvailableModels`）
- `/model` ピッカーで実際に解決するモデルファミリーが正確に表示されるようになった
- Fable 5 使用時の誤バナー・誤 suffix 表示が解消
- Bedrock GovCloud 環境でのモデル ID エラーが解消
- フルスクリーンモードでホイールスクロール加速を無効化できるようになった

## 影響範囲

- 対象ユーザー: Claude Code 全ユーザー（v2.1.173〜175 に更新で適用）
- 対象プラン: `enforceAvailableModels` は Enterprise / Team managed 環境向け
- API / UI / 管理者機能: `enforceAvailableModels` は managed setting（admin が設定）。Bedrock GovCloud ユーザーには breaking fix。

## 教材化メモ

- `enforceAvailableModels` は `availableModels` の「穴」を塞ぐ設定。v2.1.172 で「availableModels の制限がサブエージェントに適用されていなかった問題」が修正されたが、Default モデルが allowlist 外に解決するケースへの対処が本設定で追加された。組織でモデル利用ポリシーを定める場合に両方の設定を理解する必要がある。
- Bedrock GovCloud の修正は「us-gov-east-1 / us-gov-west-1 を使う政府・公共機関向け」に重要。
