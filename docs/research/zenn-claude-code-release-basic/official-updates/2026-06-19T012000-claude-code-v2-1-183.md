# 2026-06-19 Claude Code v2.1.183

## 公式内容の日本語要約

コンテキスト外で破壊的なgitコマンドや terraform/pulumi/cdk destroy を実行しようとした場合にブロックする安全機能が追加された。deprecated またはモデルが自動変更された場合に stderrへの警告も導入。また `/config --help` によるショートハンドキー一覧表示、`/config` トグル操作の改善（Enter/Spaceで変更・Escで保存終了）、WebSearchや思考ブロックに関する複数バグ修正が含まれる。

## できるようになったこと

- `git reset --hard` / `git checkout -- .` / `git clean -fd` / `git stash drop` を、作業破棄の指示がないコンテキストでブロック
- `git commit --amend` をそのセッションで行っていないコミットに対してブロック
- `terraform destroy` / `pulumi destroy` / `cdk destroy` を、スタック指定なしでブロック
- 非推奨・自動更新されたモデル使用時にstderrへ警告出力（エージェントfrontmatterも対象）
- `/config --help` で利用可能な全ショートハンドキーを一覧表示
- `/config` トグルをEnter/Spaceで変更、Escで保存終了する操作改善

## 影響範囲

- 対象ユーザー: Claude Code 全ユーザー
- 対象プラン: 全プラン
- API / UI / 管理者機能: CLI

## 教材化メモ

- 安全機能追加の背景: 誤ってローカル変更を吹き飛ばすリスクを減らすため
- エージェント frontmatter でモデル指定している場合、deprecated 警告が出るようになる点は運用上の注意点
- `/config` 操作改善は前バージョン(v2.1.181)の `/config key=value` と合わせて教えると一連の流れになる
