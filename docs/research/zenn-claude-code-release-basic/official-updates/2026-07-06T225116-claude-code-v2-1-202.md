# 2026-07-06 Claude Code v2.1.202

## 公式内容の日本語要約

Claude Code v2.1.202 が 2026-07-06T22:51Z にリリース。`/config` に「Dynamic workflow size」（small / medium / large）が追加され、ワークフロー生成時のエージェント数の目安を指定できるようになった。ただし強制的な上限ではなく、あくまで目安値として扱われる。また、ワークフロー生成エージェントのOTelテレメトリに `workflow.run_id` / `workflow.name` 属性が追加され、ワークフロー実行単位でのトレース再構築が可能になった。そのほか、Remote Control（モバイル/web）からのコマンド送信が失敗する不具合、キャプションなしで送った添付ファイルが消失する不具合の修正、`/review <pr>` が複数エージェントによる並列レビューから単発の高速レビューに戻る変更が含まれる。

## できるようになったこと

- `/config` に「Dynamic workflow size」（small/medium/large）を追加。ワークフロー生成時のエージェント数の目安を選択可能（強制上限ではない）
- ワークフロー生成エージェントのOTelテレメトリに `workflow.run_id` / `workflow.name` 属性を追加し、ワークフロー実行単位でのトレース再構築が可能に
- Remote Control（モバイル/web）からのコマンド送信が失敗する不具合を修正
- キャプションなしで添付したファイルが消失する不具合を修正
- `/review <pr>` の挙動を、複数エージェントによる並列レビューから単発の高速レビューに戻す変更

## 影響範囲

- 対象ユーザー: 全ユーザー。特にWorkflows機能利用者、OTel計装で運用監視をしている担当者、Remote Control利用者
- 対象プラン: 全プラン
- API / UI / 管理者機能: `/config`設定項目追加、OTelテレメトリ属性追加、Remote Control（モバイル/web）

## 教材化メモ

- 「Dynamic workflow size」は目安であり強制上限ではない点は、ワークフロー教材で誤解されやすいので明記すると良い
- OTelの`workflow.run_id`/`workflow.name`追加は、運用監視・可観測性の教材でトレース単位の説明に使える
