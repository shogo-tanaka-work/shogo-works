# 2026-07-14 Claude Code v2.1.208

## 公式内容の日本語要約

アクセシビリティ機能として、スクリーンリーダー利用者向けの「screen reader mode」（`claude --ax-screen-reader`、環境変数`CLAUDE_AX_SCREEN_READER=1`、または設定`axScreenReader`で有効化するプレーンテキスト表示モード）が追加された。あわせて`vimInsertModeRemaps`設定でvimモードのinsertモード中の2キーシーケンス（例: `jj`→Escape）をリマップできるようになり、`CLAUDE_CODE_PROCESS_WRAPPER`により企業のランチャー経由でのプロセス起動を強制できるようになった。フルスクリーンモードでの複数選択メニュー・「Other」入力行へのマウスクリック対応も追加。

パフォーマンス改善が大きく、多数のMCPツールを持つprint/SDKセッションでツールプール組み立てをキャッシュすることでツール呼び出しラウンドが最大7倍高速化、編集ヘビーなセッションでのセッショントランスクリプトサイズを最大79倍削減、ファイル編集読み取りキャッシュを1,000ファイル分から16MB上限に変更するなどメモリ使用量を削減した。また、MCP stdioサーバーのstderr蓄積（最大64MB/サーバー）、LSPドキュメントの無制限保持、非同期hook出力の保持、大きなツール結果ペイロードによるメモリ増大など複数のメモリリークを修正した。

その他、fast modeが対応モデルに戻した際に自動復帰しない不具合、バックグラウンドエージェントへの返信が配信失敗時に消える不具合、バイナリ更新後にバックグラウンドセッションのattachが恒久的に失敗する不具合、コンテキストウィンドウ表示が自動更新後に一時的に200kへリセットされる不具合など多数のバグを修正した。

## できるようになったこと

- スクリーンリーダーモード（`--ax-screen-reader`）でプレーンテキスト表示に対応
- `vimInsertModeRemaps`でvim insertモードの2キーリマップに対応
- `CLAUDE_CODE_PROCESS_WRAPPER`で企業ランチャー経由の自己起動を強制可能に
- 多数MCPツール利用時のツール呼び出しが最大7倍高速化
- 編集ヘビーセッションのトランスクリプトサイズを最大79倍削減

## 影響範囲

- 対象ユーザー: スクリーンリーダー利用者、vimモードユーザー、企業ランチャー配下でClaude Codeを実行する管理者、MCPを多用する開発者
- 対象プラン: 制限なし
- API / UI / 管理者機能: アクセシビリティ設定、vim設定、プロセス起動制御、MCPツール呼び出しパフォーマンス

## 教材化メモ

- ツール呼び出し7倍高速化・トランスクリプト79倍削減は、MCP多用・長時間セッション運用者への実務的インパクトが大きいため、パフォーマンス系Tipsとして紹介候補。
