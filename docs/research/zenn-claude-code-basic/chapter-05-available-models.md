# Claude Codeで使えるモデル一覧：Opus・Sonnet・Haikuの選び方

「Claude Codeを使い始めたけど、Opus・Sonnet・Haikuって何が違うの？」と迷っている方も多いのではないでしょうか。

今回伝えたいことの要点は、**タスクに応じてモデルを使い分けることで、効率とコストを両立できる**ということです。  

一言で言えば、「デフォルトのOpusで始めて、コストを抑えたいときはSonnet、調査はHaiku」という使い分けがおすすめです。

この記事では、Claude Codeで利用可能なモデルの特徴と、タスク別の選び方を解説します。読み終わる頃には、どのモデルをいつ使うべきか判断できるようになるでしょう。

## 忙しい人のために要約

時間がない方のために、重要なポイントだけまとめておきます。

- **Opus**: デフォルトモデル、最高性能で複雑なタスクに最適
- **Sonnet**: コストパフォーマンス重視、日常的な開発作業向け
- **Haiku**: 高速・低コスト、調査や簡単な確認作業向け
- `/model`**コマンド**: セッション中いつでもモデルを切り替え可能

## 利用可能なモデル一覧

Claude Codeでは、Anthropicが提供する3種類のClaudeモデルを利用できます。それぞれ性能とコストが異なるため、用途に応じた使い分けが重要になります。

!

モデル名・バージョンは随時更新される可能性があります。そのため、最新情報は[Anthropic公式ドキュメント](https://docs.anthropic.com/en/docs/about-claude/models)で確認してください。

### Opus（デフォルトモデル・最高性能）

Opusは、Claude Codeのデフォルトモデルであり、Claudeファミリーの中で**最も高性能なモデル**です。2026年1月からデフォルトモデルに採用されました。

以下の3つが主な特徴になります。

- **複雑な推論・論理的思考**が得意
- アーキテクチャ設計や大規模リファクタリングに適している
- 「深く考える」必要があるタスクで真価を発揮する

なお、API料金は1Mトークンあたり入力$15、出力$75となっています。

Opusはトークン消費が比較的大きく、サブスクリプションでの利用時では早く制限に達してしまう可能性があります。  

Max 20x であれば常に Opus でも問題ありませんが、Max 5x であればコストを意識してSonnetに切り替えることも検討してください。

トークン消費を抑える方法としては、モデル選択以外にもコンテキスト管理が重要です。

[Claude Codeのコンテキスト管理完全ガイド：/clear・/compact・/contextの使い分け](https://zenn.dev/tmasuyama1114/books/claude_code_basic/viewer/context-management)

### Sonnet（コストパフォーマンス重視）

Sonnetは、速度と品質のバランスに優れたモデルです。コストを抑えながら十分な品質を確保したい場面で活躍します。

Sonnetには以下の特徴があります。

- 速度と品質のバランスが優れている
- 日常的なコーディング、バグ修正、機能追加に最適
- コストパフォーマンスが良く、常用に適している

なお、**API料金は変動することがあるため注意が必要です**。現在は1Mトークンあたり入力$3、出力$15となっています。

Sonnetは「Opusほどの性能は不要だが、Haikuでは物足りない」という場面で選択するモデルです。

### Haiku（高速・低コストモデル）

Haikuは、**高速かつ低コスト**を重視したモデルです。

以下の3つが主な特徴になります。

- **レスポンスが非常に速い**
- 簡単なタスクであれば十分な品質
- コストを抑えたい場面に最適

API料金は1Mトークンあたり入力$1、出力$5となっているので、気軽に使えるのが魅力でしょう。

Haikuは、ファイル検索や簡単な確認作業など、高度な推論が不要な場面で活躍します。さらに、サブエージェントのモデルとして指定することで、全体のコストを抑えることもできるでしょう。

## モデル料金の比較

ここでは3つのモデルの料金差を表で整理します。

| モデル | 入力（/1M tokens） | 出力（/1M tokens） | 特徴 |
| --- | --- | --- | --- |
| Opus | $15 | $75 | デフォルト、最高性能 |
| Sonnet | $3 | $15 | バランス型、コスパ重視 |
| Haiku | $1 | $5 | 高速・低コスト |

「Opusは高い」と感じるかもしれませんが、デフォルトモデルとして採用されている理由は、その高い推論能力にあります。一方で、コストを抑えたい場合はSonnetへの切り替えを検討してみてください。

!

料金は変更される可能性があるため、注意が必要です。したがって、最新の料金は[Anthropic公式の料金ページ](https://www.anthropic.com/pricing)で確認してください。

## `/model`コマンドでの切り替え

セッション中にモデルを切り替えるには、`/model`コマンドを使用します。この方法を覚えておくと、状況に応じた柔軟な使い分けが可能になるでしょう。

### 基本的な使い方

まず、基本的なコマンドの使い方を説明しましょう。以下のコマンドを実行してください。

```
/model
```

このコマンドを実行すると、利用可能なモデルの一覧が表示されます。矢印キーで選択してEnterで確定するだけなので、直感的に操作できます。

```
> /model
────────────────────────────────────────────────────────────────────────────────
 Select model
 Switch between Claude models. Applies to this session and future Claude Code
 sessions. For other/previous model names, specify with --model.

 ❯ 1. Default (recommended)   Opus 4.5 · Most capable for complex work ✔
   2. Sonnet                  Sonnet 4.5 · Best for everyday tasks
   3. Haiku                   Haiku 4.5 · Fastest for quick answers

 Enter to confirm · Esc to exit
```

### タスクの途中でも切り替え可能

「調査はHaikuで進めて、実装に入ったらOpusやSonnetに切り替える」といった使い方が可能です。

```
# 調査フェーズ
/model → Haikuを選択
「このプロジェクトのディレクトリ構成を教えて」

# 実装フェーズ
/model → Opusを選択
「認証機能を実装して」

# テスト
/model → Sonnetを選択
「テストコードを生成して」
```

このように、タスクの性質に応じて柔軟に切り替えることで、コストを最適化できるでしょう。

`/model`コマンドの詳細な使い方やオプションについては、別の章で解説しています。

## タスク別モデル選択ガイド

どのモデルを使うべきか迷ったときの参考として、タスク別の推奨モデルを整理します。この目安を参考に、適切なモデルを選んでください。

### Opusを使うべきタスク

**Opus**は複雑な思考が必要なタスクに適しています。以下のような場面で活用してください。

- **新機能の設計・要件整理**
- **アーキテクチャの検討** （判断を誤ると手戻りが発生するため注意が必要です）
- **大規模リファクタリング**
- **難しいバグの調査**

### Sonnetを使うべきタスク

**Sonnet**は日常的な開発作業に最適なモデルです。以下のような場面で活用しましょう。

- **通常の実装** （機能追加やコード修正）
- **バグ修正** （原因が特定できているもの）
- **テストコード生成**
- **コードレビュー**
- **ドキュメント作成**

### Haikuを使うべきタスク

**Haiku**はシンプルで高速な処理が求められる場面で活躍します。以下のような場面で活用してください。

- **ファイル検索・確認** （「〇〇の処理はどこ？」といった質問）
- **簡単な質問** （「このエラーの意味は？」など）
- **コードの説明**

## まとめ

この記事では、Claude Codeで利用可能なモデルと選び方について解説しました。

ここで重要なポイントを振り返りましょう。

- **Opus**: デフォルトモデル、複雑な設計・推論に最適
- **Sonnet**: コストを抑えたいときの日常作業向け
- **Haiku**: 調査や簡単な作業向け

適切なモデル選択は、開発効率とコストのバランスを左右する重要な要素です。まずはデフォルトのOpusで始めて、コストが気になる場合はSonnetへの切り替えを検討してみてください。
