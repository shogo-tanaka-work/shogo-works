# Claude Codeの料金体系完全ガイド：ProとMaxの違いと料金比較

「Claude Codeを使いたいけど、毎月いくらかかるのかわからなくて不安...」と感じていませんか？

今回伝えたいことの要点は、Claude Codeの料金は「**サブスクリプション型**」と「**API従量課金型**」の2種類があり、**自分の使い方に合わせて選べる**ということです。

一言で言えば、定額で安心したいならサブスクリプション、柔軟にコストを最適化したいならAPI従量課金が適しています。

この記事では、Claude Codeの料金体系を両面から解説し、具体的な月額の目安やコスト管理のコツまでお伝えします。

## 忙しい人のために要約

この記事で押さえておきたいポイントは以下の通りです。

- Claude Codeにはサブスクリプション型（Pro/Max）とAPI従量課金型の2つの選択肢がある
- サブスクリプション型はPro（月額$20）、Max（月額$100〜）で定額利用が可能
- API従量課金では入力/出力トークンごとに課金される
- `/usage`コマンドでトークン使用量を随時確認できる

## Claude Codeを使う2つの選択肢

Claude Codeを利用するには、大きく分けて2つの方法があります。どちらが自分に合っているか、一緒に見ていきましょう。

### サブスクリプション型（Claude Pro / Max）

まず1つ目は、Claude.aiのアカウントでログインして利用する方法です。月額固定料金で、Claude CodeとWebインターフェースの両方を使えるのが特徴です。

#### **Pro プラン**（月額$20）

こちらは基本的なサブスクリプションプランで、以下の機能が含まれています。

- Claude Codeの利用
- 拡張思考（Extended Thinking）機能
- ファイル作成機能
- 一定のメッセージ制限あり

#### **Max プラン**（月額$100〜）

一方、より多くの利用量が必要な方向けの上位プランもあります。以下の機能が含まれています。

- 5倍（$100/月）または20倍（$200/月）の利用量オプション
- 会話をまたいだメモリ機能
- 優先アクセス

サブスクリプション型のメリットは、**コストの予測がしやすい**点にあります。

毎月の支出が固定されるため、「今月は使いすぎたかも...」という不安から解放されるのは大きいです。

また、Proプランの5倍か20倍の **どちらかを選択できる** 柔軟性もあります。

バリバリ開発する予定のときは Max 20x にアップグレードし、そこそこ開発しそうなときは Max 5x にダウングレードする、そしてその間はProプランを維持する、という使い分けができるのがメリットです。

柔軟に使い分けができるので、自分の使用量に合わせて切り替えるのがおすすめです。

### API従量課金型

次に2つ目は、Anthropic ConsoleでAPIキーを発行し、**使った分だけ支払う**方法です。使用量に応じて柔軟に課金されるのが最大の特徴です。

具体的には、以下のような特徴があります。

- 入力トークンと出力トークンごとに課金される
- メッセージ数の制限がない
- 月によってコストが変動する

「APIキーってどうやって取得するの？」と思う方もいるかもしれません。ただ、手順はシンプルです。まずAnthropic Consoleでアカウントを作成し、その後APIキーを発行するだけで利用を開始できます。

ログイン方法とアカウントの使い分けについては、別な記事で詳しく解説しています。

[Claude Codeのログインと認証設定：ProプランとAPI課金の違い](https://zenn.dev/tmasuyama1114/books/claude_code_basic/viewer/login-and-authentication)

## トークンとは何か

API課金の仕組みを理解するには、まず「トークン」の概念を押さえておく必要があります。

トークンとは、テキストを分割した最小単位のことです。Claudeを含む大規模言語モデル（LLM）は、テキストをトークン単位で処理する仕組みになっています。イメージとしては、文章を細かく切り分けた「かけら」のようなものです。

### トークン数の目安

英語の場合、1トークンはおおよそ4文字程度に相当します。一方、日本語では1文字が1〜2トークン程度となるケースが多いです。  

つまり、同じ内容を伝える場合でも日本語のほうがトークン数が増える傾向にあるわけです。これは日本語ユーザーとしては少し残念なポイントかもしれませんが、利便性を考えると仕方ない部分があります。

```
# 英語の例（約4トークン）
"Hello world"

# 日本語の例（約6-8トークン）
"こんにちは世界"
```

### 入力トークンと出力トークン

そして、API課金では、入力トークンと出力トークンが別々に計算されます。ここを理解しておくと、コスト感覚がつかめるようになります。

| 種類 | 説明 | 含まれるもの |
| --- | --- | --- |
| 入力トークン | Claudeに送るテキスト | プロンプト、ファイル内容、コンテキスト情報 |
| 出力トークン | Claudeが生成するテキスト | 回答、生成コード、説明文 |

**出力トークンのほうが単価が高い**設定になっています。なぜかというと、生成処理のほうが計算コストが高いからです。そのため、長いコードを生成するタスクでは、出力トークンのコストが積み上がりやすい点を意識しておくとよいでしょう。

## 各モデルの料金比較

Claude Codeでは3つのモデルを使い分けることができます。どのモデルをいつ使うかで、コストパフォーマンスが大きく変わってきます。

!

料金は2026年1月時点の情報です。変更される可能性があるため、最新の料金は[Anthropic公式サイト](https://claude.com/pricing)で確認してください。

### Sonnet 4.5（デフォルトモデル）

日常的な開発作業に最適なバランス型モデルです。私も普段はこのモデルをメインで使っています。

| 項目 | 料金（100万トークンあたり） |
| --- | --- |
| 入力トークン | $3 |
| 出力トークン | $15 |

コード生成、バグ修正、機能追加など、ほとんどの作業はSonnetで十分対応できます。ただし、入力が20万トークンを超える「ロングコンテキスト」リクエストの場合、料金が上昇する点には注意が必要です（入力$6、出力$22.50）。

### Opus 4.5（最高性能モデル）

次に、複雑な推論やアーキテクチャ設計、大規模リファクタリングなど、「深く考える」必要があるタスクに適したモデルです。

| 項目 | 料金（100万トークンあたり） |
| --- | --- |
| 入力トークン | $5 |
| 出力トークン | $25 |

常用するには料金が高めなので、計画立案や重要な意思決定の場面でのみ使用するのがおすすめです。ここぞという場面で頼れる存在です。

### Haiku 4.5（高速・低コストモデル）

最後に、最も高速かつ低コストなモデルで、簡単なタスクに適しています。

| 項目 | 料金（100万トークンあたり） |
| --- | --- |
| 入力トークン | $1 |
| 出力トークン | $5 |

ファイル検索や確認作業などの軽量なタスクにはHaikuが最適です。さらに、サブエージェントのモデルとしても活用できるので、**コスト最適化の強い味方**になってくれます。

各モデルの詳しい特徴と選び方については、別な記事で詳しく解説しています。

[Claude Codeで使えるモデル一覧：Opus・Sonnet・Haikuの選び方](https://zenn.dev/tmasuyama1114/books/claude_code_basic/viewer/available-models)

## サブスクリプション vs API、どちらを選ぶべきか

API での利用は柔軟であり、使った分だけというメリットがあります。

しかし、予測が大変であることや、利用料を常に気にする必要があるというデメリットが大きいです。

そのため、特にこだわりがなければ、サブスクリプション型がおすすめです。

実際、私もサブスクリプション型を選択しています。

## コスト管理の5つのヒント

Claude Codeを効率的に使いながらコスト（トークン使用量）を抑えるためのヒントを紹介します。

### 1. /usageコマンドで使用量を確認する

Claude Code上で`/usage`と入力すると、現在のセッションで消費したトークン数と料金を確認できます。

```
/usage
```

![](https://static.zenn.studio/user-upload/deployed-images/5003d0bc0326a96e75c9ae8e.png?sha=2d751bb14567316118beb803e9f392f8b6da40c5)

私も作業中にこまめにチェックする習慣をつけています。「あれ、今日は思ったより使ってるな」と気づけるだけで、コスト意識が高まります。

### 2. Haikuを活用する

すべてのタスクにSonnetやOpusを使う必要はありません。以下のような場面では、Haikuで十分対応できます。

- ファイルの検索・確認
- Git操作
- フォルダ・ファイルの整理

`/model`コマンドでセッション中にモデルを切り替えられるので、タスクに応じて使い分けてみてください。

![](https://static.zenn.studio/user-upload/deployed-images/13c534c557619b4aef1e1efb.png?sha=70a916b5a87ec3ff9cb78cdc3e0a92eb7dd0ad6a)

モデルの切り替え方法については、別な記事で詳しく解説しています。

[Claude Codeの/modelコマンドでモデルを切り替える：タスク別の最適選択](https://zenn.dev/tmasuyama1114/books/claude_code_basic/viewer/model-command)

### 3. コンテキストを適切に管理する

不要なコンテキストを引きずると、**毎回のリクエストでトークンを消費し続けて**しまいます。これは意外と見落としがちなポイントです。タスクが一段落したらコマンドでコンテキストを整理しましょう。

`/clear`でコンテキストを完全にクリアするか、`/compact`で会話を要約してコンテキストを圧縮できます。

会話を引き継いだ作業をしてほしい場合には `/compact`を使用し、新しいタスクを開始する場合には `/clear`を使用します。

コンテキスト管理の詳しい方法については、別な記事で詳しく解説しています。

[Claude Codeのコンテキスト管理完全ガイド：/clear・/compact・/contextの使い分け](https://zenn.dev/tmasuyama1114/books/claude_code_basic/viewer/context-management)
