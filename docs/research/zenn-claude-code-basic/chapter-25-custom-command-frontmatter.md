# Claude Codeカスタムコマンドのフロントマター設定：description・allowed-tools・model

カスタムコマンドを作ったものの、「もっと細かく動作を制御したい」と感じたことはないでしょうか。

**今回伝えたいことの要点は、YAMLフロントマターを使えばカスタムコマンドの動作を細かく制御できる、ということです**。一言で言えば、プロンプト本文だけでは実現できない「メタ設定」を追加できる仕組みです。

具体的には、使用できるツールの制限、専用モデルの指定、引数のヒント表示などが可能になります。「そんな設定ができるとは知らなかった」と思った方もいるかもしれませんが、使ってみると非常に便利ですよ。

そこでこの記事では、YAMLフロントマターの各設定項目と、実際の活用パターンを解説します。

カスタムコマンドの基本的な作り方については、別な記事で詳しく解説しています。

[Claude Codeのカスタムコマンドとは？スラッシュコマンドの作り方入門](https://zenn.dev/tmasuyama1114/books/claude_code_basic/viewer/what-is-custom-command)

## 忙しい人のために要約

この記事で解説する内容を、4つのポイントにまとめました。

- YAMLフロントマターは、Markdownファイルの先頭に `---` で囲んで記述するメタデータ領域
- `description` でコマンドの説明をサジェストに表示
- `allowed-tools` で使用可能なツールを制限
- `argument-hint` で引数の使い方をユーザーに提示

## YAMLフロントマターとは

YAMLフロントマターは、**Markdownファイルの先頭に記述するメタデータ領域**のことです。`---` で囲んだ中にYAML形式で設定を書くことで、カスタムコマンドの動作を制御できるわけです。

「YAMLとはどういうものだろう」と思った方もいるかもしれませんが、難しく考える必要はありません。見てみるのが一番わかりやすいので、基本的な構造を確認してみましょう。

```
---
description: コマンドの説明文
allowed-tools: Bash(git:*), Read, Edit
argument-hint: [ファイルパス]
---

ここにプロンプト本文を書く
```

なお、フロントマターは任意の設定なので、記述しなくてもMarkdownファイルの内容がそのままプロンプトとして使われます。ただ、フロントマターを活用することで、より柔軟で安全なカスタムコマンドを作成できるようになりますよ。

## description：コマンドの説明を表示する

`description` は、**コマンドの説明文を設定する項目**です。設定した説明は、`/` を入力した際のサジェスト一覧に表示されるので、コマンドの目的がひと目でわかるようになります。

具体例を見てみましょう。

```
---
description: Hello Worldを出力
---

ターミナルでHello Worldを出力してください。echoコマンドを使ってください。
```

この設定をしておくと、`/` を入力したときに「Hello Worldを出力」という説明が表示されます。どういったコマンドを実行するかが一目瞭然です。

```
─────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────
> /hello-world
─────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────
  /hello-world     Hello Worldを出力 (project)
```

一方で、`description` を省略した場合はプロンプト本文の最初の行が説明として使われます。

```
─────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────
> /hello-world
─────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────
  /hello-world     ターミナルでHello Worldを出力してください。echoコマンドを使ってください。 (project)
```

この例であればカスタムコマンド内の本文を読んでも分かるので問題ないですが、複雑なコマンドを実行する場合は明示的に `description` を設定しておくことをおすすめします。

## allowed-tools：使用可能なツールを制限する

`allowed-tools` は、**そのコマンド実行時に使用できるツールを制限する項目**です。セキュリティや意図しない操作の防止に役立つわけです。

「ツールを制限すると不便になるのでは？」と思うかもしれませんが、実はこれがカスタムコマンドの安全性を高めるポイントなのです。

### 基本的な書き方

まずは、基本的な記述例を見てみましょう。

```
---
allowed-tools: Read, Grep
---

プロジェクト内のエラーログを検索してください。
```

この例では、ファイル読み取り（`Read`）とテキスト検索（`Grep`）のみが許可されています。つまり、ファイルを編集したり、Bashコマンドを実行したりはできないわけです。

### よく使うツールの指定パターン

次に、よく使う指定パターンを表にまとめました。

| 指定 | 説明 |
| --- | --- |
| `Bash(git:*)` | git 関連のコマンドすべて |
| `Bash(npm:*)` | npm 関連のコマンドすべて |
| `Read` | ファイルの読み取り |
| `Edit` | ファイルの編集 |
| `Write` | ファイルの作成・上書き |
| `Glob` | ファイルパターン検索 |
| `Grep` | テキスト検索 |

### 制限のメリットと注意点

ツール制限の最大のメリットは、「コミット専用コマンドなのに、勝手にファイルを編集された」といった事故を防げる点でしょう。

コマンドの目的に応じて最小限のツールだけを許可することで、予期しない動作を抑制できるのです。

ただ、制限が厳しすぎるとコマンドが正常に動作しないこともあります。たとえば、コードレビューコマンドで `Read` を許可し忘れると、ファイルを読めずにレビューできませんよね。実際に動作確認しながら調整していくことが重要です。

最初はとりあえず全てのツールを許可しておき、実際に動作確認しながら必要なツールだけを制限していくのが良いでしょう。

## argument-hint：引数のヒントを表示する

`argument-hint` は、**コマンドの引数に関するヒントを設定する項目**です。

別な記事で解説する予定ですが、カスタムコマンドを実行する際に引数を入力することができます。  

たとえば、特定のファイルパスを指定したり、Pull Requestの番号を指定したりすることができます。

しかし、渡せる引数を忘れてしまうこともあるでしょう。そんな時に `argument-hint` を設定すると、引数の形式が表示されるので、使い方がわかりやすくなります。

具体例を見てみましょう。

```
---
description: ファイルをリネーム
argument-hint: [old-path] [new-path]
---

古いパスを新しいパスにリネームしてください。
```

この設定をすると、サジェストに `[old-path] [new-path]` と表示されるので、「2つのパスを引数として渡す必要があるんだな」とすぐに理解できます。

以下の例では `/rename` と入力したときに、`[old-path] [new-path]` と表示されています。この表示をもとに推測することになりますので、わかりやすいですね。

```
─────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────
> /rename  [old-path] [new-path]
─────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────
  /rename     ファイルをリネーム (project)
```

## 設定の組み合わせ例

ここまで個別の設定を見てきましたが、実際には複数の設定を組み合わせて使うことが多いです。

### 安全なコミットコマンド

フロントマター設定を活かした例を見てみましょう。

```
---
description: 変更をステージングしてコミット
allowed-tools: Bash(git add:*), Bash(git commit:*), Bash(git status:*)
---

以下の手順でコミットしてください。

1. git status で変更を確認
2. git add . で全ファイルをステージング
3. git commit でコミット
```

この例では、3つの設定項目を組み合わせています。

- `description`: サジェストで目的が分かる
- `allowed-tools`: Git操作のみに制限（ファイル編集を防止）

## まとめ

今回はYAMLフロントマターの設定項目を解説しました。

- `description`: コマンドの説明を表示
- `allowed-tools`: 使用可能なツールを制限
- `argument-hint`: 引数のヒントを表示

これらの設定を活用することで、カスタムコマンドの動作をより細かく制御できるようになります。
