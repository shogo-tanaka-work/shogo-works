# Claude Codeの最初のセッション：起動から基本操作まで

「Claude Codeをインストールしたけど、まず何をすればいいんだろう...」と迷っていませんか？

今回伝えたいことの要点は、**プロジェクトディレクトリで`claude`コマンドを実行するだけでセッションが始まり、Claudeが自動でファイルを読み込んでプロジェクトを理解してくれる**ということです。

この記事では、初回セッションの始め方から、効果的な使い方のコツまでを解説します。読み終わる頃には、Claude Codeとスムーズに対話を始められるようになっているはずです。

## 忙しい人のために要約

この記事のポイントは以下の通りです。

- プロジェクトディレクトリで`claude`コマンドを実行するとセッションが開始される
- Claude Codeは必要なファイルを自動で読み込むため、手動追加は不要
- 起動画面ではセッション情報、会話履歴、アップデート情報を確認できる
- `/help`でコマンド一覧、`/resume`で以前の会話を継続可能
- 最初はプロジェクトの概要を質問して、全体像を把握させるのがおすすめ

## プロジェクトディレクトリでClaude Codeを起動する

Claude Codeを使うには、まず作業したいプロジェクトのディレクトリに移動してから起動します。シンプルです。

```
cd /path/to/your/project
claude
```

たとえば、デスクトップの `test-project` フォルダに移動してから起動する場合は、次のようになります。

```
cd ~/Desktop/test-project
claude
```

この2行だけで準備は完了です。

Claude Codeはカレントディレクトリを作業対象として認識するので、**目的のプロジェクトディレクトリで起動する**ことがポイントになります。（正直、私も慣れるまでは間違ったディレクトリで起動してしまうことがありました）

## 起動画面の構成を理解する

Claude Codeを起動すると、ターミナルに対話型のインターフェースが表示されます。

```
──────────────────────────────────────────────────────────────────────────────────────
> 
──────────────────────────────────────────────────────────────────────────────────────
👆ここにプロンプトを入力します
```

起動画面には複数の重要な情報が含まれているため、それぞれ確認しておきましょう。

## 基本的なコマンドを覚える

Claude Codeの対話モードでは、スラッシュ（/）で始まるコマンドを使って様々な操作が可能です。

そこで、まず覚えておくべき**基本コマンド**を紹介しましょう。

| コマンド | 機能 |
| --- | --- |
| `/help` | 利用可能なコマンド一覧を表示 |
| `/resume` | 以前の会話を継続 |
| `/clear` | 会話履歴をクリア |
| `exit` または `Ctrl+C` | Claude Codeを終了 |

「どのコマンドがあるか分からない」と感じたら、まず`/help`を実行してみてください。すべてのコマンドと簡単な説明が表示されるので、困ったときの頼れる味方になります。

また、Claude Code自身の機能について質問することもできます。たとえば「Claude Code で会話履歴をクリアするにはどうすればいいですか？」と聞くと、`/clear`コマンドを使うことを教えてくれます。

基本コマンドの詳しい使い方については、別の記事で解説しています。

[Claude Codeの基本コマンド一覧：日常で使う主要コマンド](https://zenn.dev/tmasuyama1114/books/claude_code_basic/viewer/basic-commands)

以下は、実際に聞いてみた例です。

```
> 会話履歴をクリアするにはどうすればいいですか？

⏺ Claude Code では、会話履歴をクリアするには以下のコマンドを使用します：

  /clear

  このコマンドを実行すると、現在の会話履歴がクリアされ、新しいセッションが開始されます。

  その他のヘルプが必要な場合は、以下のコマンドで詳細情報を確認できます：

  /help
```

## 最初の質問を試してみる

Claude Codeを起動したら、まずプロジェクトについて質問してみましょう。**以下のような質問から始める**ことをおすすめします。

```
> このプロジェクトについて教えてください
```

この質問をすると、Claude Codeはプロジェクト内のファイルを自動的に読み込みます。そして、全体像を把握した上で回答してくれます。

ここで重要なのは、**ファイルを手動で追加する必要がない**という点です。Claude Codeは必要に応じてファイルを自動で読み込む機能を持っています。したがって、この点が単純なチャットボットとの大きな違いとなります。

ただし、ファイルやフォルダパスがあらかじめ分かっている場合には、具体的に指定してあげましょう。Claude Code による探索範囲が限定されることで、回答の精度が向上します。

ファイルを指定する場合は `@` につづけてファイルパス、またはファイル名を書いてあげましょう。

```
> @src/auth/login.ts を参考に、ログアウト機能を実装して
```

同じファイル名が複数ある場合は入力候補も表示されます。

なお、スクリーンショットを使ってUIを修正させることもできます。詳しくは別の記事で解説しています。

[Claude Codeでスクリーンショットを使ってUIを修正させる方法](https://zenn.dev/tmasuyama1114/books/claude_code_basic/viewer/screenshot-ui-fix)

## 日本語で質問しても大丈夫？

「英語で質問しないといけないのでは？」という疑問を持つ方もいるかもしれません。

結論から言うと、**Claude Codeは日本語にも対応しています**。日本語で質問すれば、日本語で回答が返ってくるので安心です。

```
> このプロジェクトは何をするもの？
```

ただし、Claude自体が英語で学習されているモデルなので、技術的な質問では英語の方が精度が高くなる場合もあります。とはいえ、日常的な開発作業では日本語で十分に実用的ですから、まずは日本語で試してみることをおすすめします。

## プロジェクトを理解させるコツ

Claude Codeを効果的に使うためには、**まずClaudeにプロジェクトを理解させる**ことが重要です。

いきなり「このバグを直して」と依頼したくなる気持ちはわかりますが、人間と同じく、まずは全体像を把握させることが重要です。

以下の順番で進める方が精度の高い結果を得られます。

1. **全体像を把握させる**: 「このプロジェクトの概要を教えて」
2. **関連ファイルを特定させる**: 「認証機能に関係するファイルはどれ？」
3. **具体的なタスクを依頼する**: 「ログイン機能のバグを修正して」

この「**探索→理解→実行**」という流れを意識してください。その結果、Claude Codeの出力品質が大幅に向上します。

人間が物事を理解するための基本的なプロセスと同じです。

## 会話を継続する方法を覚えておく

作業を中断してClaude Codeを終了しても、次回起動時に会話を継続できます。これは本当にありがたい機能です。

```
claude -c  # 直近の会話を継続（continueの略）
claude -r  # 過去の会話から選んで再開（resumeの略）
```

`-c`オプションは直近の会話を自動的に再開します。

一方、`-r`オプションを使うと、過去の会話一覧から選択して再開できます。

また、対話モード内では`/resume`コマンドでも同様の操作が可能です。したがって、状況に応じて使い分けてください。

!

会話履歴はローカルに保存されます。そのため、プライバシーに配慮しつつ過去のコンテキストを活用できる設計になっています。

セッション管理の詳細については、別の記事で解説しています。

## 終了方法に注意する

Claude Codeを終了するには、以下のいずれかの方法を使ってください。

```
> exit
```

または、キーボードショートカットの`Ctrl+C`でも終了できます。

ただし、複数回`Ctrl+C`を押すと強制終了になる場合があるので注意してください。通常は`exit`コマンドを使う方が確実です。

## まとめ

この記事では、Claude Codeの最初のセッションを始める方法を解説しました。要点を振り返りましょう。

- プロジェクトディレクトリで`claude`コマンドを実行して起動する
- `/help`で基本コマンドを確認できる
- まずはプロジェクトの概要を質問してClaudeに全体像を把握させる
- 日本語での質問にも対応している
- `exit`または`Ctrl+C`で終了できる

Claude Codeは必要なファイルを自動で読み込んでくれるため、手動でのファイル指定は不要です。難しく考えず、まずは気軽にプロジェクトについて質問してみてください。
