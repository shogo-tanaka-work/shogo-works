# MCPとは？Claude Codeで外部ツールと連携するModel Context Protocolを解説

「Claude Codeでデータベースを直接操作したい」「GitHubのIssueを自然言語で管理できたら便利なのに...」と感じたことはありませんか？

今回伝えたいことの要点は、**MCPを使えばClaude Codeの能力を外部ツールに拡張できる**ということです。一言で言えば、MCPはClaude Codeを「何でもできるアシスタント」に進化させるための仕組みなのです。

この記事では、MCPの基本概念から仕組み、実際に何ができるようになるのかまでを解説します。読み終わる頃には、MCPがどのような場面で役立つのかを理解できるようになるでしょう。

[https://docs.anthropic.com/en/docs/claude-code/mcp](https://docs.anthropic.com/en/docs/claude-code/mcp)

## 忙しい人のために要約

この記事の要点を5つにまとめました。

- MCPはAnthropicが策定したオープンな標準プロトコル
- Claude Codeと外部ツール（GitHub、データベース、Slackなど）を接続できる
- MCPサーバーを追加することで、Claude Codeの能力を拡張できる
- ローカルサーバーとリモートサーバーの2種類がある
- 一度設定すれば、自然言語で外部サービスを操作できる

## MCPとは

MCPは「Model Context Protocol」の略で、Anthropicが発表したオープンな標準プロトコルです。このプロトコルは、AIモデルと外部ツール・データソースを安全に接続するための「共通言語」として機能します。

「プロトコル」と聞くと難しそうに感じるかもしれません。イメージとしては「USB-C」に近いものがあるでしょう。つまり、USB-Cが様々なデバイスを1つの規格で接続できるように、MCPも様々な外部ツールを1つのプロトコルで統合する役割を果たしているのです。

### なぜMCPが必要なのか

Claude Code単体でファイルの読み書きやコマンド実行は可能です。ただ、以下のような操作にはClaude Code単体では対応していません。

対応していない操作の例を挙げてみましょう。

- データベースへの直接アクセス
- GitHubのIssueやPRの操作
- Slackへのメッセージ送信
- 外部APIの呼び出し
- ブラウザの自動操作

「こういう操作ができたらいいのに...」と感じたことがある方も多いのではないでしょうか。これらの操作を可能にするのがMCPです。MCPサーバーを追加すれば、Claude Codeの能力を大幅に拡張できるようになります。

!

一部の操作についてはそれぞれの CLI ツール（GitHub CLI など）を使えば、ターミナルからの操作と同じく Claude Code から操作することもできます。  

ただ、MCPを使うことで Claude Code が外部ツールとして使える機能を把握したり、より簡潔に操作できるようになるため、MCPを使うメリットがあります。

## MCPの仕組み

MCPは**クライアント-サーバー型**のアーキテクチャを採用しています。Claude CodeがMCPクライアントとして動作し、各種MCPサーバーに接続する形です。

MCPサーバーは、特定のサービスやツールとの橋渡しを行う役割を担っています。たとえば、GitHub MCPサーバーを使う場合、Claude CodeからのリクエストをGitHub APIに変換して実行してくれるわけです。

### サーバーの種類

MCPサーバーには**ローカルサーバー**と**リモートサーバー**の2種類があります。それぞれの特徴を表にまとめました。

| 種類 | 実行場所 | 設定方法 | メンテナンス |
| --- | --- | --- | --- |
| ローカル（stdio） | 自分のマシン | npxやDockerで起動 | 自分で管理 |
| リモート（HTTP） | ベンダーのサーバー | URLを追加するだけ | ベンダーが管理 |

ローカルサーバーは自分のマシンで実行されるため、細かいカスタマイズが可能です。一方で、リモートサーバーは設定が簡単です。また、ベンダーがアップデートを管理してくれるというメリットもあります。用途に応じて使い分けてください。

## MCPで何ができるようになるか

MCPを導入することで、様々な操作がClaude Code上で可能になります。

「具体的に何ができるの？」という疑問にお答えするため、いくつかの例を挙げてみます。

### 1. データベースへの直接アクセス

PostgreSQLやMySQLなどのデータベースに直接クエリを実行できるようになります。  

これが特に便利だと感じる方も多いのではないでしょうか。たとえば、以下のように指示してみてください。

```
> usersテーブルから最近登録した10人を取得して
```

すると、Claude Codeがスキーマを理解した上で、適切なSQLクエリを生成・実行してくれます。SQLを自分で書く必要がないため、開発効率が大幅に上がるでしょう。

一方で、データベースに対して直接クエリを実行することは、データベースに破壊的な変更を加えてしまう可能性があるため、注意が必要です。

MCPを使ってデータベースを操作する場合は、ローカルでの検証程度にとどめておきましょう。

### 2. GitHub連携

次に、Issue、PR（プルリクエスト）の作成・編集・レビューも可能になります。具体的には、以下のような操作に対応しています。

```
> PR #456の変更内容を確認して、改善点を提案して
> このバグについてIssueを作成して
```

### 3. コミュニケーションツール連携

さらに、SlackやDiscordへのメッセージ送信、チャンネルの履歴取得も可能になります。

```
> #generalチャンネルの今日のメッセージを要約して
> チームに進捗報告を投稿して
```

### 4. ブラウザ自動操作

また、Playwrightを使ったWebスクレイピング、E2Eテストも実行できます。

こういった操作は CLI では実現が難しく、かつ自然言語で操作できる方が効率的ですので、MCPを使うと便利です。

```
> このページのスクリーンショットを撮って
> ログインフォームを自動入力するスクリプトを作って
```

!

2026年1月現在では Claude Code の 「Claude in Chrome」機能を使えば、Chrome を操作することができます。

ただし、執筆時点では Chrome に限定されているため、クロスブラウザでの操作が必要な場合はまだ Playwright MCP を使う必要があります。

Playwright MCPとChrome DevTools MCPの使い分けについては、別の記事で実務経験をもとに詳しく解説しています。

### 5. 最新ドキュメントの取得

Context7などのMCPサーバーを使うと、最新のライブラリドキュメントを取得できるようになります。  

LLMの学習データは古いため、最新のAPIや使い方が不正確になることがあるんですよね。ただ、MCPを活用すればこの問題を解消できるでしょう。

```
> Next.js のApp Routerでミドルウェアを作って。use context7
```

## まとめ

この記事では、MCPの基本概念と仕組みについて解説しました。

MCPはAnthropicが策定したオープンな標準プロトコルで、Claude Codeと外部ツールを接続して能力を拡張できます。  

データベース、GitHub、Slackなど様々な連携が可能で、自然言語で操作できる点がポイントです。

MCPはClaude Codeの6つの拡張機能の1つです。他の拡張機能（CLAUDE.md、Skills、カスタムコマンドなど）との使い分けについては、別の記事で詳しく解説しています。
