# Claude Codeカスタムコマンドで$0/$1/$2を使って複数引数を扱う方法

「カスタムコマンドに複数の情報を渡したいけど、うまく個別に扱えない...」と困っていませんか？

今回伝えたいことの要点は、**位置引数（`$0`, `$1`, `$2`...）を使えば、引数を個別に受け取れる**ということです。

この記事では、位置引数の基本から実践的なコマンド例までを解説します。

## 忙しい人のために要約

この記事のポイントを簡潔にまとめます。

- `$0`, `$1`, `$2`で引数を個別に参照できる
- スペース区切りで自動分割される
- 引数が足りない場合は空文字列になる

## 位置引数の基本

**位置引数**とは、コマンドに渡された引数を順番に参照するための仕組みです。具体的には、`$0`で1番目、`$1`で2番目、`$2`で3番目の引数にアクセスできます。

では、簡単な例を見てみましょう。

```
# .claude/commands/rename.md

$0 を $1 にリネームしてください。
```

このコマンドを `/rename old-name.ts new-name.ts` と実行すると、以下のように展開されます。

- `$0` → `old-name.ts`
- `$1` → `new-name.ts`

つまり、「old-name.ts を new-name.ts にリネームしてください。」というプロンプトがClaudeに送られるわけです。

## 実践的なコマンド例

ここからは、位置引数を活用した実用的なコマンド例を紹介します。それぞれのパターンを理解することで、自分のワークフローに合わせたコマンドを作成できるようになります。

### ファイル参照と組み合わせる

次に、**2つのファイルを比較するパターン**を確認しましょう。なお、`@`記法と組み合わせることで、動的なファイル参照も可能になります。

```
# .claude/commands/compare.md
---
description: 2つのファイルを比較
argument-hint: [ファイル1] [ファイル2]
---

@$0 と @$1 を比較し、差分を説明してください。
特に重要な違いがあれば強調してください。
```

`/compare src/old.ts src/new.ts` と実行すると、両ファイルの内容がコンテキストに読み込まれます。その結果、Claudeから詳細な差分の説明を得られるわけです。

## 位置引数が役立つケース

位置引数が適しているのは、引数の役割が明確に分かれている場合です。

```
# .claude/commands/move.md

$0 を $1 ディレクトリに移動してください。
```

「何を」「どこへ」という2つの情報を明確に分離できるのが、位置引数の特徴です。

## 位置引数を使う際の注意点

ここで、位置引数を使う際に注意すべき点をいくつかお伝えします。

### 引数が足りない場合の挙動

**指定した位置に引数がない場合、その`$N`は空文字列に置き換わる**のです。具体的な例を見てみましょう。

```
# .claude/commands/greet.md

$0さん、$1の件についてお聞きします。
```

このコマンドを `/greet 田中` と実行すると（引数が1つだけ）、結果はこうなります。

```
田中さん、の件についてお聞きします。
```

`$1`が空文字列になるため、文として不自然な出力になってしまいます。そのため、プロンプト内でClaudeに判断を委ねるか、`argument-hint`で必要な引数を明示しておくとよいでしょう。

### スペースを含む引数には制約がある

位置引数はスペースで区切られるため、**スペースを含む値を1つの引数として渡すことが難しい**点に注意が必要です。クォートで囲んでもそのまま引数の一部として扱われる可能性があります。スペースを含むファイル名を扱う場合は、別の方法を検討してください。

## 応用コマンド例：PRレビュー準備

位置引数を活用した実用的なコマンド例を紹介します。

```
# .claude/commands/prepare-pr.md
---
description: PRレビュー用の情報を準備
argument-hint: [ブランチ名] [変更カテゴリ]
---

PR作成前の情報を準備してください。

- ブランチ: $0
- カテゴリ: $1 (feat/fix/refactor/docs)

以下を実行してください：
1. mainとの差分ファイル一覧を表示
2. 差分内容を分析
3. 簡潔なPR説明文を生成
```

このように、複数の異なるパラメータを扱うときに位置引数が活躍します。

## まとめ

位置引数（`$0`, `$1`, `$2` ...）を使えば、カスタムコマンドで引数を個別に受け取れます。複数のパラメータを持つコマンドを作る際に非常に便利です。ぜひ、自分のワークフローに合わせてコマンドを作ってみてください。
