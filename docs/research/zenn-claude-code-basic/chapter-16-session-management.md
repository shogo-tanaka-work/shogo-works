# Claude Codeのセッション管理：開始・終了・再開の完全ガイド

「Claude Codeを使い始めたけど、セッションってどう管理すればいいの？」「前回の続きをやりたいのに、やり方がわからない...」と感じていませんか？

今回伝えたいことの要点は、**セッション管理を理解すると、作業効率が大きく変わる**ということです。一言で言えば、「会話の続きを適切に管理できるようになる」のがゴールになります。

この記事では、セッションの開始・終了から、過去のセッション再開まで、一通りの操作方法を解説します。読み終わる頃には、状況に応じてセッションを使い分けられるようになっているはずです。

## 忙しい人のために要約

この記事の内容を5つのポイントにまとめました。

- **セッション**とは Claude Code との対話のまとまりのこと
- **`claude`** コマンドで新規セッション開始、**`/exit`** または `Ctrl+C`**を2回** で終了
- **`claude -c`** で直前のセッションを継続できる
- **`/resume`** で過去のセッション一覧から選択して再開できる
- **`/clear`** でコンテキストをリセットして新しいタスクに集中

## セッションとは

Claude Codeにおける**セッション**とは、一連の対話のまとまりを指します。

セッション内の**会話履歴は「コンテキスト」として保持される**仕組みになっています。つまり、「さっき話した内容を覚えていてくれる」状態になるわけです。

初めてClaude Codeを起動する方は、セッションの基本的な開始方法についても別な記事で解説しています。

[Claude Codeの最初のセッション：起動から基本操作まで](https://zenn.dev/tmasuyama1114/books/claude_code_basic/viewer/first-session)

例えば、「このファイルを修正して」と依頼したとしましょう。その後、先ほどの修正にミスがあったと気付いたとします。

そこで「さっきの修正を取り消して」と言えば、Claude Code は直前の変更内容を理解したうえで対応してくれます。これは、セッション内の文脈が保持されているからこそできることです。

また、各セッションには固有の**セッションID**が割り当てられています。そのため、後から特定のセッションを再開することも可能です。

## 新規セッションの開始

`claude`コマンドを実行するたびに、**新しいセッションが開始**されます。

ここで重要なのは、過去のセッションの情報は自動的には引き継がれないという点です。

「さっきの作業の続きをしたい」という場合は、次に説明する `--continue` オプションを使うことになります。

## 前回のセッションを継続する

一度 PC を閉じた後、「昨日の続きをやりたい」という場面はたまにあるでしょう。

そんなときに便利なのが `--continue`（短縮形: `-c`）オプションです。

```
# 直前のセッションを継続
claude --continue

# 短縮形でも同じ
claude -c
```

このオプションを使うと、**現在のディレクトリで最後に行ったセッション**が読み込まれます。

会話履歴がそのまま復元されるため、「昨日はどこまでやったっけ？」と確認する必要がなくなるのが嬉しいポイントです。

ただし、`--continue` は「**直前のセッション**」しか対象にできないという制限があります。

「昨日も作業したけど、今度は 3日前のセッションを再開したい」という場合は、次に説明する `/resume` コマンドを使ってください。

## 過去のセッション一覧から選択する

Claude Code を起動した後、`/resume` コマンドを実行すると、過去のセッション一覧が表示されます。

```
> /resume
```

一覧が表示されたら、矢印キーで目的のセッションを選択し、Enterキーで再開できます。

```
> /resume
──────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────

Resume

❯ READMEを書いてください
  2 seconds ago · 7 messages · -

  Create main.js with Hello and helloWorld function
  23 seconds ago · 18 messages · -
```

各セッションには作成日時や対話の冒頭内容が表示されるため、目的のセッションを見つけやすくなっています。

「先週やった認証機能の実装を再開したい」といった場面で、ぜひ活用してみてください。

セッション管理以外の便利なコマンドについては、別な記事で紹介しています。

[Claude Codeの基本コマンド一覧：日常で使う主要コマンド](https://zenn.dev/tmasuyama1114/books/claude_code_basic/viewer/basic-commands)

ただ、候補として表示されるのは会話の冒頭部分のみです。

これだと分かりづらいので `/rename xxxx` コマンドを使って、セッションの名前を変更することもできます。

```
> /rename READMEを書いていたセッション
```

これを実行すると、以下のように表示されます。

```
> /rename READMEを書いていたセッション
  ⎿  Session renamed to: READMEを書いていたセッション
```

こうしますと、`/resume` コマンドの一覧に表示される名前が変更されます。

一番上の（カーソルが当たっている）セッション候補を見ると、先ほど変更した名前が表示されていることがわかります。

```
Resume

❯ READMEを書いていたセッション
  18 seconds ago · 13 messages · -

  Create main.js with Hello and helloWorld function
  2 minutes ago · 18 messages · -

  A to show all projects · P to preview · R to rename · / to search · Esc to cancel
```

## 特定のセッションIDを指定して再開する

セッションIDがわかっている場合は、`--resume` オプションで直接指定できます。少し高度な使い方ですが、これを使うと特定のセッションを再開することができます。

```
claude --resume abc123def456
```

なお、**セッションIDは `/resume` コマンドの一覧画面で確認**できます。

チームでセッションIDを共有する使い方は想定されていません。一方で、自分用のメモとしてセッションIDを記録しておくと、特定の作業に素早く戻れるようになります。頻繁に参照するセッションがある場合は、メモしておくと便利です。

## `/clear` で新しいタスクに切り替える

ここまではセッションを再開したり、継続する方法を紹介してきました。

しかし、セッション（会話）が長くなるとコンテキストが大きくなり、トークン消費量が増加していきます。

そのため、基本的には出来るだけセッションを分けながら作業を進めることをおすすめします。

まったく別のタスクに移るときは、`/clear` コマンドを使いましょう。

人間で言うと一度これまでの会話を忘れて、新しいタスクに集中するようなイメージです。

話題を切り替える前に `/clear` を実行しておくと、Claude Code が過去の会話に引きずられることなく、新しいタスクに集中できるようになります。

`/clear` や `/compact` などのコンテキスト管理コマンドについては、別な記事で詳しく解説しています。

[Claude Codeのコンテキスト管理完全ガイド：/clear・/compact・/contextの使い分け](https://zenn.dev/tmasuyama1114/books/claude_code_basic/viewer/context-management)

## まとめ

この記事では、Claude Codeのセッション管理について解説しました。

セッション管理はちゃんと管理しようと思うと難しいですが、覚えておくと作業効率が変わってくるはずです。

慣れてきたら、`/resume` で過去のセッションを選んだり、ブランチごとにセッションを使い分けたりしてみてください。自分なりの運用スタイルが見つかると、より快適に開発を進められるようになりますよ。

セッション履歴を巻き戻して別の方向性を試したい場合は、`/rewind` コマンドも活用できます。

[Claude Codeの/rewindコマンドで過去に戻る：会話の巻き戻しテクニック](https://zenn.dev/tmasuyama1114/books/claude_code_basic/viewer/rewind-command)

また、対話的な使い方ではなくスクリプトからClaude Codeを呼び出したい場合は、ヘッドレスモードが便利です。

[Claude Codeのヘッドレスモード（-pオプション）でターミナルから非対話実行する方法](https://zenn.dev/tmasuyama1114/books/claude_code_basic/viewer/headless-mode)
