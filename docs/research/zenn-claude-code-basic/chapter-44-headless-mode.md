# Claude Codeのヘッドレスモード（-pオプション）でターミナルから非対話実行する方法

こんにちは、とまだです。

「Claude Codeをスクリプトから呼び出して、作業を自動化できないかな...」と思ったことはありませんか？

今回伝えたいことの要点は、`-p`**オプション**を使えばClaude Codeを非対話モードで実行でき、スクリプトからの呼び出しやパイプ連携ができるようになるということです。この機能は「ヘッドレスモード」と呼ばれています。

この記事では、ヘッドレスモードの基本的な使い方からカスタムコマンドの呼び出し方法まで解説します。

## 忙しい人のために要約

ポイントを4つにまとめておきます。

- `-p`オプションで**非対話実行**が可能
- カスタムコマンド（`/xxx`）もスクリプトから呼び出せる
- `--output-format`で出力形式を選択（textまたはjson）
- 標準入出力との**パイプ連携**に対応

## ヘッドレスモードとは

ヘッドレスモードは、**対話型インターフェースを使わずに**Claude Codeを実行する方法です。

通常のClaude Codeは、ターミナル上でリアルタイムに対話しながらコードを編集するのが基本です。一方、ヘッドレスモードでは**プロンプトを渡して結果を受け取る**というシンプルな使い方ができます。

!

「非対話モード」とも呼ばれます。UIなしで処理を完了させるため、バッチ処理との相性が良いのが特徴です。

## 基本的な使い方

ヘッドレスモードの使い方はシンプルです。`-p`（または`--print`）オプションでプロンプトを渡すだけで動作します。

```
claude -p "このプロジェクトのREADME.mdを読んで、概要を説明して"
```

このコマンドを実行すると、Claude Codeはプロンプトを処理し、結果を**標準出力に出力して終了**します。対話モードには入らないため、スクリプトからの呼び出しに最適なわけです。

### モデルを指定する

使用するモデルを指定したい場合は、`--model`オプションを使います。

```
claude -p "コードをレビューして" --model opus
```

指定できるモデル名は`opus`、`sonnet`などです。

### 許可するツールを制限する

「ファイルを勝手に書き換えられたら困る」と思った方もいるかもしれません。そこで使えるのが`--allowedTools`オプションです。

```
claude -p "コードを分析して問題点を報告して" --allowedTools "Read,Grep,Glob"
```

この例では、**ファイルの読み取りと検索のみを許可**しています。書き込みやコマンド実行は禁止されるため、意図しない変更を防げます。

## 出力形式の指定

用途に応じて出力形式を選べます。

### テキスト形式（デフォルト）

最もシンプルな形式から見ていきましょう。

```
claude -p "バグを見つけて報告して" --output-format text
```

人間が読みやすいプレーンテキスト形式で出力されます。ログに残す場合やそのまま確認する場合に向いているでしょう。

### JSON形式

プログラムで処理したい場合はJSON形式を選択します。

```
claude -p "コードの問題点を分析して" --output-format json
```

JSON形式では、応答内容に加えてメタデータも含まれます。セッションIDやコスト情報などを取得できるため、後続の処理でパースする場合に便利です。

```
{
  "type": "result",
  "subtype": "success",
  "result": "分析結果のテキスト...",
  "session_id": "abc123",
  "total_cost_usd": 0.003
}
```

## カスタムコマンドの呼び出し

ヘッドレスモードでは、**カスタムコマンド（スラッシュコマンド）もスクリプトから実行**できます。これはとても便利な機能です。

カスタムコマンドの作成方法については、別の記事で詳しく解説しています。

[Claude Codeのカスタムコマンドとは？スラッシュコマンドの作り方入門](https://zenn.dev/tmasuyama1114/books/claude_code_basic/viewer/what-is-custom-command)

### カスタムコマンドとは

`.claude/commands/`に配置したMarkdownファイルがスラッシュコマンドになります。

```
.claude/commands/
├── create-review.md     → /create-review
├── generate-docs.md     → /generate-docs
└── analyze-code.md      → /analyze-code
```

### スクリプトからの呼び出し

カスタムコマンドも`-p`オプションで実行できます。

```
claude -p "/create-review src/main.ts"
```

引数を渡す場合は、コマンド名の後にスペース区切りで指定します。コマンドファイル内では`$ARGUMENTS`や`$1`、`$2`として参照できます。

カスタムコマンドでの引数の扱い方については、別の記事で詳しく解説しています。

[Claude Codeカスタムコマンドで$ARGUMENTSを使って引数を受け取る方法](https://zenn.dev/tmasuyama1114/books/claude_code_basic/viewer/custom-command-arguments)

### 権限確認をスキップする

自動化スクリプトで使う場合、権限確認のプロンプトが邪魔になることがあります。`--dangerously-skip-permissions`オプションを使うと、すべての権限確認をスキップできます。

```
claude -p "/my-command" --dangerously-skip-permissions
```

!

このオプションは名前の通り**危険**です。すべてのツール実行が自動承認されるため、信頼できるコマンドでのみ使用してください。`--allowedTools`で必要なツールに制限することをおすすめします。

カスタムコマンドの詳細な設定方法については、別の記事で詳しく解説しています。

[Claude Codeカスタムコマンドのフロントマター設定：description・allowed-tools・model](https://zenn.dev/tmasuyama1114/books/claude_code_basic/viewer/custom-command-frontmatter)

## パイプ処理との組み合わせ

ヘッドレスモードは**Unixパイプと組み合わせて使える**という特徴もあります。標準入力からデータを受け取り、標準出力に結果を出力できます。

### ファイル内容を分析させる

ファイルの内容を直接パイプで渡せます。

```
cat error.log | claude -p "このエラーログの原因を分析して"
```

### コマンド出力を分析させる

他のコマンドの出力を分析させることもできます。

```
npm run test 2>&1 | claude -p "テスト結果を要約して"
```

### Git差分をレビューさせる

コミット前の変更内容をレビューさせるのも便利な使い方です。

```
git diff | claude -p "この変更内容をレビューして"
```

## その他のオプション

ヘッドレスモードで使える便利なオプションをいくつか紹介しておきます。

| オプション | 説明 |
| --- | --- |
| `--max-turns <n>` | エージェントループのターン数上限を設定 |
| `--continue`, `-c` | 直前の会話を継続 |
| `--resume <session-id>` | 特定のセッションを再開 |
| `--verbose` | 詳細ログを出力（デバッグ用） |
| `--disallowedTools` | 禁止するツールを指定 |

### 会話を継続する

`--continue`オプションを使うと、直前のセッションを継続できます。

```
claude -p "さっきのコードにテストを追加して" --continue
```

セッション管理の詳細については、別の記事で詳しく解説しています。

[Claude Codeのセッション管理：開始・終了・再開の完全ガイド](https://zenn.dev/tmasuyama1114/books/claude_code_basic/viewer/session-management)

## 注意点

ヘッドレスモードを使う際に知っておきたい点がいくつかあります。

### コンテキストの独立性

各呼び出しは独立したセッションとして扱われます。前回の実行結果を引き継ぎたい場合は、`--continue`や`--resume`オプションを使う必要があります。

### コスト

ヘッドレスモードでも通常どおりトークンが消費されます。大量のファイルを分析する場合や頻繁に実行する場合は、コストを意識しておくとよいでしょう。

一方で、「必要なファイルだけを対象にする」「`--allowedTools`で読み取り専用に制限する」といった工夫でコストを抑えることもできます。

## まとめ

この記事では、ヘッドレスモードの基本的な使い方を紹介しました。

- `-p`オプションで**非対話実行**が可能
- カスタムコマンドも`-p`で呼び出せる
- `--output-format`で**出力形式を制御**できる
- `--allowedTools`で**ツールを制限**するのが安全
- パイプ処理で他のコマンドと連携しやすい

まずは読み取り専用のツール（`Read,Grep,Glob`）に制限した状態で試してみるのがおすすめです。
