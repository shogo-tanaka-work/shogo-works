# Claude Codeでスクリーンショットを使ってUIを修正させる方法

「UIのバグを直してほしいけど、言葉で説明するのが難しい...」と感じたことはありませんか？

ボタンの位置がずれている、テキストが切れている——そういった視覚的な問題は、テキストだけで正確に伝えるのが意外と大変ですよね。

今回伝えたいことの要点は、**Claude Codeはスクリーンショットを直接入力できる**ということです。一言で言えば、「見せれば伝わる」のがこの機能の強みになります。

この記事では、スクリーンショットの撮り方から貼り付け方法、効果的な指示の出し方まで、実践的に解説していきます。読み終わる頃には、UIの問題を視覚的に伝えて素早く修正できるようになるでしょう。

## 忙しい人のために要約

この記事のポイントを5つにまとめました。

- Claude Codeは**スクリーンショットを入力として受け付ける**
- プロンプトに画像をペースト（`Cmd+V` / `Ctrl+V`）するだけで送信可能
- UIバグの修正、デザインカンプの実装、エラー画面の解決などに活用してください
- 具体的な説明を添えると、より正確な修正が得られやすくなる
- 大きな画像はコンテキストを消費するため、**必要な部分だけ切り取る**のがおすすめ

## 画像入力機能とは

Claude Codeは、マルチモーダルAIであるClaudeをベースとしているため、**画像を理解する能力があります**。ターミナル上で動作するCLIツールでありながら、スクリーンショットを入力として受け付け、その内容を解析してコード修正まで行えるという特徴を持っています。

Claude Codeの基本的な特徴や機能については、別な記事で詳しく解説しています。

[Claude Codeとは？AI駆動開発のメリット・特徴を徹底解説](https://zenn.dev/tmasuyama1114/books/claude_code_basic/viewer/what-is-claude-code)

一般的なCLIツールでは「画像を見せる」という操作は難しいものでしょう。しかし、Claude Codeではクリップボードから直接画像を貼り付けられます。そのため、言葉で説明しにくいUIの問題も、視覚的に伝えることが可能になります。

## スクリーンショットの撮り方

まず、スクリーンショットをクリップボードに保存する方法を確認しておきましょう。OS別の手順を説明します。

### Macの場合

Macでは、以下のショートカットでスクリーンショットをクリップボードに保存してください。

```
Cmd + Shift + 4
```

このショートカットを押すと、画面の一部を選択するカーソルが表示されます。続いて、修正したいUIの部分をドラッグして選択してください。選択が完了すると、その範囲がクリップボードに保存されます。

なお、ファイルに保存せずクリップボードに直接コピーされるため、すぐにClaude Codeに貼り付けできるという利点があります。

### Windowsの場合

Windowsでは、**Snipping Tool**を使用してください。

```
Win + Shift + S
```

このショートカットで画面の一部を選択でき、自動的にクリップボードに保存されます。

### Claude Codeへの貼り付け

スクリーンショットを撮影したら、Claude Codeのプロンプトで以下のように貼り付けてください。

- **Mac**: `Cmd + V`（うまくいかない場合は`Ctrl + V`）
- **Windows**: `Ctrl + V`

貼り付けると、画像が入力として認識されるのがわかります。入力画面にて、以下のように `[Image #1]` と表示されていればOKです。

```
──────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────
> [Image #1]
──────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────
```

あとはテキストで指示を追加してEnterを押すだけなので、操作は非常にシンプルです。

## 活用シーン

スクリーンショット機能は、さまざまな開発シーンで役立つ特徴があります。ここでは、具体的な活用例を4つ紹介します。

### UIバグの修正

最も多い使い方は、UIのバグを見せて修正を依頼するケースでしょう。

```
[Image #1]
このボタンのテキストが切れてしまっています。修正してください。
```

画像を見せることで、「ボタンのテキストが切れている」という状態を正確に伝えられます。そのため、言葉だけで説明するよりも、はるかに具体的な修正が得られやすくなります。

### デザインカンプの実装

次に、デザイナーから受け取ったデザインカンプを見せて、実装を依頼するケースを見てみましょう。

```
[Image #1]
このデザインと同じレイアウトを実装してください。
Tailwind CSSを使用しています。
```

Claude Codeはデザインの余白、配色、フォントサイズなどを画像から読み取り、それに近いコードを生成してくれます。

ただし、完璧な再現は難しい場合もあるので注意が必要です。ピクセル単位の精度を求める場合は、数値を明示的に指定するとよいでしょう。

### エラー画面の解決

さらに、エラーメッセージが表示された画面をそのまま見せて、解決策を求めることもできます。

```
[Image #1]
このエラーを解決してください。
```

エラーメッセージだけでなく、エラーが発生している画面全体のコンテキストも伝わるわけです。したがって、より適切な解決策が得られやすくなります。

### 他サイトのUI参考

また、参考にしたいウェブサイトのUIを見せて、類似のレイアウトを実装してもらうことも可能です。

```
[Image #1]
このようなカード型のレイアウトを実装してください。
```

ピクセル単位で正確に再現することはやはり難しいのですが、大まかなレイアウトを参考にすることは可能です。

## 効果的な指示の出し方

スクリーンショットを貼り付けるだけでも機能しますが、**より良い結果を得るためのコツ**が3つあります。順番に見ていきましょう。

### 具体的な説明を添える

まず、画像だけでなく、テキストで具体的な説明を添えることで精度が上がります。

```
[Image #1]
赤枠で囲った部分のマージンを20pxに変更してください。
```

特に複数の要素がある画面では、どの部分を修正したいのかを明確に伝えることが重要です。

### 期待する結果を明示する

次に、現状の問題点だけでなく、期待する結果も伝えると効果的でしょう。

```
[Image #1]
現在：テキストが1行で表示されている
期待：3行で表示し、はみ出す場合は省略記号（...）を表示
```

このように**Before/Afterを明確にする**ことで、意図した修正が得られやすくなるのです。

### 技術スタックを伝える

最後に、使用しているフレームワークやライブラリを伝えると、それに適したコードが生成されます。

```
[Image #1]
このレイアウトを修正してください。
技術スタック：Next.js, Tailwind CSS
```

Claude Codeはプロジェクト構造から技術スタックを推測できますが、明示的に伝えることでより確実になるという利点があります。

## 注意点

画像入力を使用する際には、**3つの注意点**があります。それぞれ確認しておきましょう。

### コンテキストの消費

大きな画像は**コンテキストウィンドウを多く消費**します。そのため、画面全体ではなく、必要な部分だけを切り取ることで、コンテキストの消費を抑えられるでしょう。

また、ピンポイントで画像を渡すことで、より修正箇所を特定しやすくなります。

人間が修正するとしても、大きな画像を雑に渡されるよりも、修正箇所の画像を具体的に渡される方が精度が高くなるのと同じです。

### 著作権への配慮

さらに、他サイトのデザインを参考にする場合は、著作権に配慮してください。以下のポイントを守ることをおすすめします。

- ロゴやアイコンをそのまま使用しない
- レイアウトパターンの参考にとどめる
- 必要に応じてオリジナルのデザインに調整する
