# Claude Codeのultrathinkで深く考えさせる：拡張思考モードの使い方

こんにちは、とまだです。

「Claude Codeにもっと深く考えさせたいけど、どうすればいいんだろう...」と感じたことはありませんか？

今回伝えたいことの要点は、`ultrathink`**というキーワードを使えば、Claude Codeに最大31,999トークンの思考予算を与えて深く考えさせられる**ということです。

この記事では、`ultrathink`キーワードの使い方と効果的な活用場面を解説します。

## 忙しい人のために要約

この記事で伝えたいポイントをまとめました。

- `ultrathink`で最大31,999トークンの思考予算が割り当てられる（出力予算から確保）
- `think`や`think harder`ではなく、`ultrathink`が必要
- 設計判断・根本原因分析・セキュリティレビューなど複雑なタスクに有効
- 思考トークンは次のターンに引き継がれないが、課金対象となる
- `MAX_THINKING_TOKENS`環境変数が設定されている場合、`ultrathink`は無効化される
- `/config`でグローバル設定も可能

## 拡張思考モード（Extended Thinking）とは

Claude Codeには、「**拡張思考モード**」という機能が搭載されています。これは、出力トークン予算の一部を確保して、複雑な問題を段階的に推論させる仕組みです。通常の応答よりも深い思考を行うため、複雑な問題に対してより質の高い回答を得られるのです。

通常、Claudeは質問を受けると比較的すぐに回答を生成しますよね。一方で、拡張思考モードを有効にすると、回答を出す前に内部で深い検討を行うわけです。この思考プロセスは「thinking」として可視化されるため、Claudeがどのように問題を分析しているか確認できます。

つまり、複雑な問題では深い思考が品質向上に貢献するということです。

## ultrathinkキーワードの使い方

Claude Codeで拡張思考を有効にするには、`ultrathink`というキーワードを使用してください。

```
> システム全体のアーキテクチャを見直して ultrathink
```

正しく認識できているときは `ultrathink` という単語が虹色で表示されているはずです。

![](https://static.zenn.studio/user-upload/deployed-images/c271af66007b9821536958a2.png?sha=ff665194c848f79a67875c1d9bb331c990de71d3)

!

**ultrathinkキーワードの重要な注意点**

1. `think`、`think harder`、`think more`などの表現は通常のプロンプト指示として処理されるため、拡張思考モードは有効になりません
2. `ultrathink`は思考予算の割り当てだけでなく、Claudeにより徹底的に推論するよう意味的にシグナルを送るため、タスクによっては必要以上に深く考えすぎる場合があります
3. `MAX_THINKING_TOKENS`環境変数が設定されている場合、その設定が優先され、`ultrathink`キーワードは機能しません

拡張思考を有効にするには`ultrathink`を使う必要がありますが、これらの特性を理解した上で使用してください。

## 思考トークンの仕組みと予算

拡張思考モードでは、**出力トークン予算から最大31,999トークンを思考用に確保**します。これは追加で与えられるものではなく、通常の出力に使えるトークンから割り当てられる点に注意が必要です。

簡単にいうと「たくさんのトークンを使って、深く考えてくれる」ということです。

なお、Sonnet 4.5やOpus 4.5では、デフォルトで拡張思考が有効になっています（2026年1月時点）。

各モデルの詳細な特徴については、別な記事で詳しく解説しています。

## 効果的な使いどころ

拡張思考モードは、すべてのタスクに使うべきではありません。トークン消費が増えるため、本当に必要な場面を見極めることが大切ですよね。では、どのような場面で効果を発揮するのでしょうか？代表的なケースを紹介します。

[Claude Codeのコンテキスト管理完全ガイド：/clear・/compact・/contextの使い分け](https://zenn.dev/tmasuyama1114/books/claude_code_basic/viewer/context-management)

### 設計の意思決定

まず、アーキテクチャの選択や技術スタックの決定など、**複数の選択肢を比較検討する場面**で威力を発揮します。設計に悩んだ経験がある方は多いのではないでしょうか。

```
> ultrathink このアプリケーションのバックエンドを設計して。
> スケーラビリティと保守性を重視して、最適なアーキテクチャを提案して
```

このように指示すると、Claudeは、REST vs GraphQL、モノリス vs マイクロサービスなど、様々な観点から検討した上で提案を行ってくれます。

### バグの根本原因分析

次に、表面的な症状だけでなく、**問題の本質を探りたいとき**にも有効です。また、見落としがちな原因候補も洗い出せるという特徴があります。

```
> ultrathink このエラーの根本原因を分析して。
> なぜこのタイミングでのみ発生するのか、考えられる原因をすべて検討して
```

### パフォーマンス最適化

さらに、**ボトルネックの特定と最適化戦略の検討**にも役立ちます。

```
> ultrathink このクエリのパフォーマンスを改善して。
> 現在の実装の問題点と、考えられる最適化手法を比較検討して
```

### セキュリティレビュー

加えて、**脆弱性の洗い出し**には多角的な視点からの分析が欠かせません。この場面でもultrathinkは力を発揮します。

```
> ultrathink このAPIエンドポイントのセキュリティをレビューして。
> 考えられる攻撃ベクトルと対策を網羅的に検討して
```

## グローバル設定で常に有効化する方法

「毎回ultrathinkと入力するのは面倒だな...」と感じる方もいるかもしれません。特定のプロジェクトや作業で常に拡張思考を使いたい場合は、`/config`コマンドでグローバル設定ができます。

```
/config
```

/configコマンドの詳細な使い方については、別な記事で詳しく解説しています。

[Claude Codeの基本コマンド一覧：日常で使う主要コマンド](https://zenn.dev/tmasuyama1114/books/claude_code_basic/viewer/basic-commands)

設定画面が開いたら、`Thinking mode` オプションを探して `false` から `true` に変更してください。

```
> /config
─────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────
 Settings:  Status   Config   Usage  (tab to cycle)

 Configure Claude Code preferences

   Auto-compact                              true
   Show tips                                 true
 ❯ Thinking mode                             false 👈 ここを true に変更
```

これにより、すべてのプロンプトで自動的に拡張思考が適用されるようになります。

ただ、常時有効化はトークン消費の大幅増加を招くことがあります。コストとのバランスを考慮して、プロジェクトの性質を見極めて設定するのがよいでしょう。

Claude のプランが Max 20x であれば常時有効化しても問題ないかもしれませんが、Pro プランや Max 5x プランの場合にはリミットに達しやすいので、必要な場面でのみ使うようにしましょう。

常時有効化の代わりに、タスクに応じてモデルを切り替えることでもコスト最適化が可能です。

[Claude Codeの/modelコマンドでモデルを切り替える：タスク別の最適選択](https://zenn.dev/tmasuyama1114/books/claude_code_basic/viewer/model-command)
