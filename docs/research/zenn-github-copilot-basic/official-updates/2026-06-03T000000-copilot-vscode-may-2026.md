# 2026-06-03 GitHub Copilot in VS Code 5月リリースまとめ

## 公式内容の日本語要約

VS Code 向け GitHub Copilot の5月分アップデートをまとめた changelog。エージェント運用の本格化（Agents window の Stable 昇格、リモートエージェントセッション、複数セッション並行）、企業向けの BYOK 拡充、ターミナルコマンドの AI リスク評価が柱。

## できるようになったこと

- **Agents window が Stable（preview）に昇格**: プロジェクト横断の agent-first 体験。変更レビューを高速化。
- **複数セッション並行実行**: ドラッグ / Alt+クリック / Open to the Side でエージェントセッションを横並びで同時実行。
- **リモートエージェントセッション（preview）**: SSH または Dev Tunnels 経由でリモートマシン上で実行。クライアント切断後もセッション継続（Agent Host Protocol への投資）。
- **BYOK 拡充**: 認証不要の分離環境向け「Air-gapped BYOK」、カスタムエンドポイントプロバイダー追加、モデルピッカー、実トークン使用量の可視化、コミットメッセージ生成等の「configurable utility models」。
- **コマンドリスク評価（experimental）**: ターミナルコマンドに AI 生成のリスク評価と説明を表示。パスワード等の機密は LLM へ非送信。
- ターミナル出力圧縮の対象拡大（テスト・ビルド・Docker など）。

## 影響範囲

- 対象ユーザー: VS Code で Copilot を使う開発者。エージェント運用・リモート開発・企業環境の利用者に効果大。
- 対象プラン: Copilot 各プラン（一部 Business/Enterprise 寄りの機能を含む）。
- API / UI / 管理者機能: VS Code 拡張（UI）、BYOK（設定）、Agent Host Protocol（リモート実行）。

## 教材化メモ

GitHub Copilot 教材の「エージェント運用」章に、Agents window の Stable 化・複数セッション並行・リモートエージェントセッションを追記。企業導入章には Air-gapped BYOK とコマンドリスク評価（機密非送信）をセキュリティ/ガバナンス論点として整理。
