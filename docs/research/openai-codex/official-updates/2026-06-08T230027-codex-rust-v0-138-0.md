# 2026-06-09 OpenAI Codex rust-v0.138.0

## 公式内容の日本語要約

OpenAI Codex の stable リリース rust-v0.138.0 が公開された。`/app` コマンドで現在の CLI スレッドを macOS / ネイティブ Windows の Codex Desktop へ引き渡せるようになり、Windows のワークスペース起動が手動プロンプトで止まらず Desktop に直接開けるようになった。ローカル画像添付・画像生成の保存先パスをモデルへ公開して後続編集を確実にし、reasoning effort 選択の柔軟化、アカウントトークン使用量の読み取り・PAT v2 認証対応、plugin 自動化の構造化出力強化なども入った。並行して alpha（rust-v0.138.0-alpha.7/8、rust-v0.139.0-alpha.1）も出ている。

## できるようになったこと

- **`/app` デスクトップハンドオフ**: 現在の CLI スレッドを macOS / ネイティブ Windows の Codex Desktop へ引き渡し。Windows のワークスペース起動は手動プロンプトで止まらず Desktop へ直接展開。
- **画像パスのモデル公開**: ローカル画像添付・スタンドアロン画像生成の保存先パスをモデルへ公開し、後続編集・ファイル参照を確実化。
- **reasoning effort 選択の柔軟化**: `Alt` バインドを持たない端末向けの fallback ショートカットを TUI に追加。モデルが提示する順序で effort レベルが流れるように。
- **アカウント連携・認証**: App-server 連携がアカウントのトークン使用量を読めるように。CLI / app-server フローで **v2 personal access token（PAT）** をサポート。
- **plugin 自動化の構造化出力**: add/remove・marketplace コマンドが `--json` 対応。plugin list JSON に marketplace source を含め、plugin 詳細にデフォルトプロンプト・リモート MCP サーバー・利用不可アプリテンプレートを公開。

## 影響範囲

- 対象ユーザー: Codex CLI 利用者（Desktop 連携は macOS / Windows）、plugin / app-server 連携を組む開発者
- 対象プラン: 一般（PAT v2 は認証フロー利用者）
- API / UI / 管理者機能: CLI / TUI、Codex Desktop ハンドオフ、app-server プロトコル、plugin 自動化（--json）

## 主な修正

- `/goal` ワークフローの予測可能性向上（`/goal edit` の複数行ペーストが早期送信されない、アイドル自動ターンが Plan モードに入らない、terminal turn 失敗後に goal が自動継続しない）。
- フォークしたスレッドがユーザー変更後のタイトルを保持。
- 起動の堅牢化（`/usr/bin/bash` 対応、Linux プロキシソケットパス短縮、期限切れ OAuth-backed MCP 資格情報の事前リフレッシュ）。
- リモート / シンボリックリンクのワークスペースで `AGENTS.md` の読み込みが正確に。

## 教材化メモ

Codex の「Desktop 連携」章に `/app` ハンドオフ（CLI ↔ Desktop、Windows 直接展開）を追記。「マルチモーダル」章に画像保存先パスのモデル公開を、「認証」章に PAT v2 対応を追加する。「plugin 自動化」章は `--json` 対応と plugin 詳細の構造化（marketplace source / デフォルトプロンプト / リモート MCP）を追記し、CI から plugin を機械的に扱う運用例を示す。rust-v0.137.0 の続きの stable として位置づけ、alpha（0.139 系）は件数言及に留める。
