# 2026-05-18 OpenAI Codex rust-v0.131.0（stable）

## 公式内容の日本語要約

Codex の stable リリース `rust-v0.131.0`。TUI・plugin・remote workflow・Python SDK の4領域を中心に大きく強化された。TUI は service-tier コマンドのデータ駆動化、blended token 使用量表示、権限/承認モード表示、有効な workspace root 表示、レスポンシブな Markdown テーブルなど、session 制御と表示が拡充。`@` メンションがファイル・ディレクトリ・plugin・skill を1つの picker で横断検索できるようになった。

plugin ワークフローは marketplace CLI コマンド、バージョン認識の共有、共有 checkout、shared-workspace バケットの明確化、**plugin hooks の既定有効化**を獲得。remote ワークフローは daemon 管理の `codex remote-control`、実行時 enable/disable API、status 読み取り、registry 連携/設定済み remote 環境をサポート。Python SDK は `openai-codex` / `openai_codex` へ移行し、runtime 生成型の固定、並行ターンルーティング、承認モード、統合テストを追加。診断用に `codex doctor`（runtime / auth / terminal / network / config / local state を横断）が追加された。

Bug Fix では Windows サンドボックス（deny-read ルール、scoped write root、PowerShell エッジケース）の堅牢化、権限昇格時の managed read 制限保持、app-server / local state 起動時の SQLite データ保持と fail-closed 化などが入った。レガシーな function 形式 `apply_patch` と after-tool-use hooks は既定安定化に伴い削除された。

## できるようになったこと

- TUI で blended token 使用量・権限/承認モード・有効 workspace root を確認でき、Markdown テーブルがレスポンシブ表示される。
- `@` メンションでファイル/ディレクトリ/plugin/skill を1つの picker から横断検索できる。
- plugin marketplace CLI コマンドが追加され、plugin hooks が既定で有効化された。
- `codex remote-control` が daemon 管理になり、実行時 enable/disable API・status 読み取り・registry 連携 remote 環境をサポート。
- Python SDK が `openai-codex` / `openai_codex` へ移行し、並行ターンルーティング・承認モードに対応。
- `codex doctor` で runtime / auth / terminal / network / config / local state を横断診断できる。
- Windows サンドボックス挙動が堅牢化された。

## 影響範囲

- 対象ユーザー: Codex CLI / Cloud / SDK 利用者、plugin・remote 運用チーム、Python で Codex を組み込む開発者、Windows 利用者
- 対象プラン: Codex 利用環境全般
- API / UI / 管理者機能: TUI / plugin marketplace / `codex remote-control` / Python SDK（`openai-codex`）/ `codex doctor`
- 注意: 旧 function 形式 `apply_patch` と after-tool-use hooks は削除（破壊的変更の可能性）。並行する alpha は `rust-v0.132.0-alpha.1` が窓内に1件。
