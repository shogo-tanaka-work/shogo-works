# 2026-07-08 Codex rust-v0.143.0 — リモートプラグイン既定有効化・OS標準プロキシ対応・Bedrock GPT-5.6追加

## 公式内容の日本語要約

リモートプラグインが既定で有効化され、npmマーケットプレイスソース対応やカタログ表示（ローカル/リモートのバージョン表示）が強化された。macOS/WindowsのシステムプロキシPAC/WPAD経由での認証・Responses API通信ルーティングに対応。`codex remote-control pair` コマンド追加（稼働中デーモンから手動ペアリングコードを生成）。Amazon Bedrock向けにGPT-5.6系（Sol/Terra/Luna）モデルを追加し、`max` reasoning effortを正式サポート。MCPツールが既定でtool searchを使用するようになり、ChatGPTホスト型MCPサーバーがセッション認証を明示利用可能に。app-serverクライアントが環境情報の取得、子スレッド一覧取得、任意ターンからの履歴フォークに対応。

バグ修正: Windows ConPTYの改行・バックスペース処理、サンドボックス資格情報の再試行、TUIの安全確認プロンプトの残留、exec-server一時オフライン時の復旧改善、リモートコントロールのトークン再試行ストーム防止、シャットダウン時のリアルタイム文字起こし末尾保持、WebSocketインクリメンタルリクエストの改善、GitHub APIレート制限によるインストーラ失敗の低減。

派生改善: hooks.jsonのトップレベルメタデータ検証を緩和。

セキュリティ関連: OpenSSL/Hono/fast-uri/quick-xml/crossbeam-epochを更新。

## できるようになったこと

- リモートプラグインの既定利用（npmマーケットプレイスソース対応）
- 企業プロキシ（PAC/WPAD）経由でのCodex利用
- Amazon Bedrock経由でのGPT-5.6系モデル利用、`max` reasoning effort
- MCPツールのtool search既定化、ChatGPTホスト型MCPサーバーのセッション認証利用
- `codex remote-control pair` による手動ペアリングコード生成

## 影響範囲

- 対象ユーザー: CLI/エージェント運用者全般。特にプラグイン運用チーム、企業プロキシ配下のユーザー、Bedrock利用者、MCPサーバー運用者、hooks.json利用者
- 対象プラン: 該当なし（CLIツール）
- API / UI / 管理者機能: CLI設定・plugin marketplace・remote-control・MCP

## 教材化メモ

Bedrock連携の広がり（Claude Fable 5に続きCodexも対応）は、マルチクラウドAIエージェント基盤比較教材の材料になる。企業プロキシ対応は導入障壁解消の実例として使える。
