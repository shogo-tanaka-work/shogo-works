# 2026-06-01 OpenAI Codex rust-v0.136.0

## 公式内容の日本語要約

OpenAI Codex の stable **rust-v0.136.0** が 2026-06-01 17:49 UTC（JST 06-02 02:49）に公開。前回 stable は 05-28 の v0.135.0。今回は TUI のリンク・テーブル表示改善、セッションのアーカイブ機能、app-server / リモート実行まわりの拡張、Windows sandbox の elevated プロビジョニング（alpha）、画像生成拡張、Python SDK ベータの整備、Bedrock 連携の改善が中心。

## できるようになったこと

- **セッションのアーカイブ**: TUI から `/archive`、CLI から `codex archive` / `codex unarchive`。アーカイブ済みセッションは復元するまで resume / fork から保護される。
- **TUI markdown のリンク・テーブル改善**: OSC 8 メタデータで web リンクをクリッカブルに維持。狭い表は読みやすい key/value レコード表示に切替（リンク先を失わない）。
- **app-server 統合の強化**: thread を初回ターンのページ付きで resume、MCP サーバーの詳細ステータス表示、`codex app-server --stdio` で stdio モード起動。
- **リモート実行の認証強化**: 承認済み OpenAI ホスト向けに `CODEX_API_KEY` 登録をサポート。remote-control の websocket は ChatGPT アクセストークンではなく短命のサーバートークンを使用。
- **Windows 管理者向け**: `codex sandbox setup --elevated` の alpha プロビジョニング経路、許可された Windows sandbox 実装の requirements サポート。
- **画像生成拡張**: feature-gate されたスタンドアロン画像生成拡張が、ネイティブの Codex 画像アーティファクト完了パイプラインを通って動作。
- **Python SDK ベータ**: 標準の `pip install openai-codex` 導線、quickstart / API リファレンス / FAQ / examples を整備。設定は公開名 `CodexConfig` に統一。`python-v*` タグで runtime とは独立にリリース可能。

## 主なバグ修正・堅牢化

- ChatGPT 認証が 5 分の失効前にトークンを更新。再利用された refresh token は generic な cloud error ではなく「再ログイン要求」として扱う。
- コマンド安全性の強化: `/diff` がリポジトリ提供の Git ヘルパー/フックを実行しない、非 Windows で PowerShell パーサ実行を回避、ブラウザ由来の exec-server websocket ハンドシェイクを拒否。
- sandbox コマンドが中断・Windows のネットワーク拒否後により確実にクリーンアップ。`deny` read ルールが safe-command / approval-bypass 経路でも維持。
- **Bedrock 認証が `AWS_REGION` / `AWS_DEFAULT_REGION` にフォールバック**。非対応の Bedrock GPT service tier は提示・送信しない。Bedrock カタログに **GPT-5.5** を追加、未対応 OSS エントリを削除、GPT モデルは default-tier のみ。
- MCP 依存を `rmcp` 1.7.0 に更新。組み込みツールのスキーマ説明（shell / Code Mode / MCP / image / goal / plan / multi-agent 等）に既定値・任意項目・範囲・enum を明記。

## 影響範囲

- 対象ユーザー: Codex CLI / app / app-server / Python SDK 利用者、Bedrock 経由で Codex を運用する組織、Windows 利用者。
- 対象プラン: Codex 利用可能なプラン全般。
- API / UI / 管理者機能: TUI（リンク・テーブル・アーカイブ）、CLI（`archive` / `app-server --stdio`）、リモート実行（`CODEX_API_KEY`）、Windows sandbox（管理者、alpha）、Python SDK（`pip install openai-codex`）。

## 教材化メモ

- Codex CLI / cli-features 教材に `/archive`・`codex archive` / `unarchive` のセッション保護を追記。長期運用での「過去セッション整理」手順として標準化。
- TUI の OSC 8 クリッカブルリンク・key/value テーブルは「ターミナル上での閲覧性改善」として cli-features に1段追記。
- リモート実行教材があれば `CODEX_API_KEY` 登録と短命サーバートークン化を「ChatGPT トークン依存からの脱却」として記述。
- Bedrock / クラウド経由運用の教材に `AWS_REGION` フォールバックと GPT-5.5 追加、default-tier 限定挙動を反映。
- codex-sdk 教材は `pip install openai-codex` ベータ導線と `CodexConfig` への統一を反映（旧 `AppServerConfig` 名は置換）。
