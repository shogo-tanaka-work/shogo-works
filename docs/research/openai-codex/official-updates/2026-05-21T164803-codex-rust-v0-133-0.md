# 2026-05-21 OpenAI Codex 0.133.0（stable）

## 公式内容の日本語要約

OpenAI は 2026-05-21 に Codex CLI の stable リリース **0.133.0**（`rust-v0.133.0`）を公開した。直前の stable 0.132.0（5/20）から、Goals 機能の experimental 卒業、権限プロファイルの管理機能強化、拡張（extensions）のライフサイクル観測の拡充が中心。同日に Codex アプリ `26.519`（"Appshots and Goal Mode Launch"）も公式 changelog に掲載され、Goal mode の experimental 卒業は CLI / アプリ共通のトピックになった。

## できるようになったこと

- **Goals が既定で有効化**され、experimental ではなくなった。専用ストレージに保存され、進行中のターンをまたいで進捗を追跡する。
- `codex remote-control` がフォアグラウンドコマンドのように動作し、準備完了まで待機してマシン状態を報告。明示的なデーモン風の `start` / `stop` も保持。
- **権限プロファイル**に list API・継承（inheritance）・managed な `requirements.toml` 対応・実行時リフレッシュが加わり、Windows サンドボックス統合も強化。
- プラグイン探索が、マーケットプレイス対応の list 出力・インストール済みバージョン表示・マーケットプレイスルートの可視化・リモートコレクション対応で検査しやすくなった。
- 拡張が観測できるライフサイクルイベントが増加（subagent の開始/停止、ツール実行、ターンのメタデータ、非同期の承認・ターン処理）。`SubagentStart` / `SubagentStop` フックが追加。
- （同日 Codex アプリ 26.519）macOS で Appshots が有効化、Goal mode が experimental 卒業、リモート computer use、マーケットプレイス経由のプラグイン共有、アプリ内ブラウザの注釈強化。

## 影響範囲

- 対象ユーザー: Codex CLI / アプリ利用者全般。特に Goals でタスク進捗を管理するユーザー、組織で権限プロファイルを統制する管理者、拡張・プラグイン開発者。
- 対象プラン: プラン制限の明記なし（CLI stable リリース）。
- API / UI / 管理者機能: CLI 機能 + app-server / 拡張 API。managed な `requirements.toml` は組織管理者向けの権限統制に関係。

## 教材化メモ

Goals が experimental 卒業し既定有効になった点は workflows-goals 教材の前提更新（「experimental」記述があれば削除）。権限プロファイルの list API・継承・`requirements.toml` 管理は approval-modes-sandbox / enterprise-admin 教材の恒久情報。`SubagentStart` / `SubagentStop` フックは hooks 教材へ追記候補。速報的なバージョン番号は持ち込まない。
