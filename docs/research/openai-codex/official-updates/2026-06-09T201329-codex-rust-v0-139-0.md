# 2026-06-09 OpenAI Codex rust-v0.139.0

## 公式内容の日本語要約

OpenAI Codex の stable リリース rust-v0.139.0 が公開された。code mode から（ネストした JavaScript ツール呼び出しを含め）単体の web search を直接呼べるようになり、検索結果をプレーンテキストで受け取れる。ツール／コネクタの入力スキーマが `oneOf` / `allOf` を保持し、大きなスキーマの圧縮時も浅い構造をより多く残すため、リッチな MCP ツールとの互換性が向上した。`codex doctor` の拡充、plugin marketplace のキャッシュ応答化なども入った。

## できるようになったこと

- **code mode から単体 web search を直接呼び出し**（ネストした JS ツール呼び出しからも可）。結果はプレーンテキストで取得。
- **MCP スキーマの `oneOf` / `allOf` 保持**。大きなスキーマの圧縮時も浅い構造をより多く維持し、リッチな MCP ツールとの互換性が向上。
- **`codex doctor`** がローカルレポートにエディタ・ページャの環境情報を含めるように（JSON 出力では生値をリダクト）。
- **plugin marketplace** が `codex plugin marketplace list --json` に各 marketplace source を含め、キャッシュ済みリモートカタログから先に返してからバックグラウンド更新。

## 影響範囲

- 対象ユーザー: Codex CLI 利用者、MCP ツール／プラグインを組む開発者
- 対象プラン: 一般
- API / UI / 管理者機能: CLI / TUI（code mode の web search、codex doctor）、MCP スキーマ互換、plugin marketplace 自動化

## 主な修正

- `codex resume --last "..."` / `codex fork --last "..."` が末尾引数を初期プロンプトとして扱うように（セッション ID 誤認の修正）。
- サブエージェントの MCP 起動警告が当該スレッドに留まり、親スレッドの重複アラート・起動スピナー停止を回避。
- 画像編集が会話履歴からの推測でなく参照された画像ファイルパスをそのまま使用。
- `/new` `/clear` `/fork` のスレッドリセット時にクラウド管理の要件・フィーチャーフラグを落とさない。
- サンドボックス実行が承認済みのエスカレーション判断を保持し、proxy-only ネットワーキングをより一貫して強制。
