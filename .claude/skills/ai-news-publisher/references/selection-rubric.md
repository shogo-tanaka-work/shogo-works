# AIニュース公開選別基準

## 判定カテゴリ

### A: 優先してAIニュース化

次のどれかに当てはまる場合。

- デフォルトモデル、主要モデル、フラッグシップ機能の変更
- 利用者の操作フローが変わる
- 企業導入・管理・権限・監査・セキュリティに影響する
- 料金、プラン、上限、モデル退役に関係する
- API / SDK / CLIの実装方法に影響する
- 既存教材の内容が古くなる
- 読者が「今どう使えばいいか」を知る価値が高い

### B: Knowledgeへ最小反映

次のような更新。

- 既存教材の前提、手順、料金、モデル一覧、対象プランが古くなる
- AIニュース記事だけでは、既存教材を読んだ人が誤解する
- 独立したKnowledge記事にすると速報寄りで薄いが、教材の正確性には関係する
- 管理者向け注意や関連リンクとして短く入れておきたい

Knowledgeへ入れるのは、ニュース本文の要約ではなく「恒久的に残すべき結論」です。

### C: 保留

次のような更新。

- 重要そうだが、公開先や読者像が未確定
- beta / previewで提供範囲が狭い
- 公式情報が短く、実務手順に落とし込むには不足
- 既存教材構成を変える必要がある
- ユーザーに方針確認した方がよい

### D: 見送り

次のような更新。

- 小規模バグ修正のみ
- release noteとしては重要だが、教材読者の学習価値が薄い
- 非公式情報または未確認情報
- 同じ内容がすでに教材へ反映済み
- 一時キャンペーン、イベント告知、採用広報など

## スコアリング目安

各項目を0から2で評価します。合計6点以上ならAIニュース化候補です。

| 観点 | 0 | 1 | 2 |
| --- | --- | --- | --- |
| 重要度 | 小さい | 一部ユーザーに重要 | 多くの読者に重要 |
| 持続性 | 一過性 | 数か月は有効 | 半年以上価値が残る |
| 実務影響 | ほぼなし | 使い方の補足になる | 手順や判断が変わる |
| 既存教材影響 | なし | 追記で改善 | 更新しないと古くなる |
| 公式情報の十分性 | 不足 | 最低限ある | 十分説明できる |

Knowledgeへ最小反映するかは、上記スコアとは別に次で判断します。

- 既存Knowledgeの記述が誤りになるか
- 既存Knowledgeで紹介している使い方・料金・モデル選定が変わるか
- ニュース記事へのリンクだけでは読者導線として弱いか
- 追記が数行から数段落で済むか

## 判断メモの書き方

各ニュースについて、短く次を残します。

- 判定: A / B / C / D
- 理由:
- 推奨アクション:
- 予定ファイル:
- 注意点:

## ありがちな判断

- 新モデル公開はAになりやすい。Knowledgeへはモデル一覧や使い分けだけ反映する。
- 管理者向けの新機能はAまたはB。読者が一般ユーザーだけならCもありうる。
- patch releaseは基本D。ただし重大な退役、権限、安全性、API破壊的変更を含む場合はB以上。
- 広告・収益化ニュースはAIニュース候補。既存教材カテゴリに入れる前に方針確認。
- 業界別agentテンプレートはAIニュース候補。汎用Knowledgeには設計思想だけ最小反映する。
