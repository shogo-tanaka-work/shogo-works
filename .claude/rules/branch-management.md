# ブランチ管理ルール

## 基本方針

- 通常の作業は `main` から作業ブランチを切って進める。
- 作業完了後は PR を作成し、`main` にマージしたら不要な作業ブランチを削除する。
- `worktree` は標準運用にしない。必要性が明確な場合だけ使う。

## 標準フロー

1. `main` を最新化する
2. `main` から作業ブランチを切る
3. 実装・検証・PR作成を行う
4. `main` にマージする
5. 不要になった local / remote branch を削除する

## worktree を使ってよい場合

次のいずれかに当てはまる場合のみ使う。

- 現在の作業ディレクトリに未コミット差分があり、別件を並行して安全に進める必要がある
- 2件以上の作業を同時並行で進める必要があり、同じ作業ツリー内での切り替えが不便
- 依存関係や生成物を分けて検証したい明確な理由がある

## worktree を使わない場合

次のような通常作業では、worktree を作らず作業ブランチだけで進める。

- 1件ずつ順番に進める記事作成
- 小さな修正や単発の更新
- `main` に未コミット差分がなく、並行作業も不要な場合

## cleanup ルール

- PR が merge 済み、または不要と判断した branch は放置しない。
- worktree を使った場合は、作業完了時に worktree と対応 branch の削除まで行う。
- branch / worktree が増えすぎた場合は、まず open PR、未コミット差分、未追跡ファイルの有無を確認してから整理する。

## 補足

- 迷ったら、まず worktree ではなく通常の作業ブランチを選ぶ。
- worktree は「安全のために常用するもの」ではなく、「並行作業が必要なときの例外手段」と考える。
