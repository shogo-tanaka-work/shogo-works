# テストルール

## TDDの手順（必ず守ること）
1. 失敗するテストを先に書く（Red）
2. テストが通る最小実装をする（Green）
3. リファクタリングする（Refactor）

## テストの方針
- 正常系・異常系の両方を必ずカバーする
- テスト名は「○○のとき、○○すること」の形式で書く
- モックは最小限に。実際の動作に近い形でテストする
- カバレッジは最低 80% を維持する

## ファイル配置
- ソース:  src/feature/xxx.ts
- テスト:  tests/feature/xxx.test.ts（src/ と同じ階層構造）

## テストの書き方（フレームワーク非依存の構造）
```
describe('機能名', () => {
  it('正常系: ○○のとき、○○すること', async () => {
    // Arrange（準備）
    // Act（実行）
    // Assert（検証）
  })

  it('異常系: ○○のとき、エラーを返すこと', async () => {
    // Arrange
    // Act
    // Assert
  })
})
```
