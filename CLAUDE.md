# 田中省伍のHP

## 最初に必ず読むファイル
- 仕様・技術スタック: docs/SPEC.md
- アーキテクチャ決定: docs/ARCH.md

## 技術スタック
※ docs/SPEC.md を参照。このファイルには書かない。
  プロジェクト開始時に SPEC.md を読んでスタックを把握すること。

## 開発ルール（必須・スタック共通）
1. 実装前に必ずプランモードで設計を確認すること
2. テストを先に書く（TDD）。テストなしの実装は禁止
3. 1ファイル1責務を守ること。肥大化したら分割する
4. 型定義は集約ファイルで管理。any / unknown の安易な使用禁止
5. エラーハンドリングを必ず実装する（握りつぶし禁止）
6. デバッグ用の出力はコミット前に必ず削除する
7. SPEC.md にない機能の先取り実装禁止

## 禁止事項
- テストなしのコミット
- 仕様外機能の実装
- main への直接プッシュ
- .env の内容をコードにハードコード

## 基本コマンド
```bash
npm run dev       # 開発サーバー起動
npm run build     # ビルド（astro check + astro build）
npm run check     # 型チェック（astro check）
npm run preview   # ビルド済みサイトのプレビュー
```
